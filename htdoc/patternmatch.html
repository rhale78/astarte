    • Match expressions

    You can use pattern matching to test whether a given value v has a particular form, and to bind identifiers to parts of v if v does have the desired form. For example, 
           Match ?h :: ?t = x.
    will only succeed if x is a nonempty list (so it has the form h::t for some h and t). It binds name h to the head of x and t to the tail of x. The right hand side of the match expression is called the target of the match. Statement 
           Match ?h :: ?t = [2,3,4]
    has target [2,3,4] and binds h = 2 and t = [3,4], but 
           Match ?h :: ?t = [].
    fails (with exception testX).

    Standard pattern forms

    Some standard pattern forms are as follows. Others can be created using pattern declarations.

    ?x
    This is called a pattern identifier. There must be no space between the question mark and the name (x in this case). When it is matched against a target, there are two possibilities.

    1. If x is currently unbound, then identifier x is bound to the target. Notice that the question mark is removed. So 
            Match ?x = t.
      has exactly the same meaning as 
            Let x = t.

    2. If x is already bound, then a test is done to see whether the current value of x is equal to the target. If so, then the match succeeds. If not, then the match fails with exception testX. For example, 
            Match (?x,?x) = (a,b).
      has the same effect as 
            Let x = a.
      {x == b}

      (See causing failure for a discussion of the meaning of {x == b}.) You can only do this if x was bound during this same pattern match. If you want to do an equality test with an identifier that was bound earlier, do not include a question mark. For example, 

            Match (x,?y) = (a,b).
      has the same effect as 
            {x == a}
      Let y = b.

    ?
    ? is an anonymous pattern identifier. Use it where you need to put something, but you don't need to do a binding. Example: Match (?a, ?) = p.

    x
    Any expression that does not contain any pattern identifiers is just evaluated. Its value is tested for equality with the target of the match. If the two are equal, then the match succeeds. If they are unequal, then the match fails with exception testX.

    Example: Match (?a, 4) = p.

    (?x,?y)
    This matches an ordered pair. It matches ?x against the left-hand component of the pair, and ?y against the right-hand component. You can use notation (x,y,z), etc., discussed in pairs.

    Example: Match (?a,?b,?c) = aTriple.

    ?x :: ?y
    This matches a nonempty list. It matches ?x against the head of the list, and ?y against the tail of the list. This pattern can also be written [?x & ?y]. The usual list abbreviations, such as [a,b,c], can be used in patterns.

    Example:
    Match ?h::?t = [1,2,3,4]. (sets h = 1, t = [2,3,4])
    Match [?a,?b & ?c] = [1,2,3,4]. (sets a = 1, b = 2, c = [3,4])

    [ ]
    This pattern only matches an empty list. It uses function nil? to perform the test, so it can be used for any species of list, even those that do not support equality tests.

    ?x +/ ?y
    This pattern matches a nonempty list. It matches ?x against the sublist obtained by deleting the last member of the target list, and matches ?y against the last member of the target list.

    Example: Match ?a +/ ?b = [1,2,3,4]. (binds a = [1,2,3], b = 4)

    ?x + c
    Pattern ?x + c matches a natural number that is greater than or equal to c. If the target is t, then it binds x so as to make equation x + c = t true.

    Example: Match ?n+1 = 4. (binds n = 3)

    c + ?x
    Same as ?x + c.

    ?x * c
    Pattern ?x * c matches a natural number that is divisible by c. If the target is t, then it binds x so as to make equation x*c = t true.

    Example: Match ?n*2 = 20. (binds n = 10)

    c * ?x
    Same as ?x * c.

    ?x _where_ c
    This matches ?x, but only when condition c is true.

    Example: Match ((?a,?b) _where_ a > b) = p.
    (succeeds when p is a pair of the form (x,y), with x > y)

    ?x _also_ ?y
    This matches both ?x and ?y against the same target.

    Example: Match (?a,?b) _also_ ?p = f(x).

    ?x _orelse_ ?y
    This matches ?x against the target. It that match fails, then it matches ?y against the target instead.

    escape(?x,t)
    This pattern ignores the actual target, and instead matches ?x against target t. This can be used, in combination with _orelse_, for defaults in pattern matching. For example, 
       Match ?h::?t _orelse_ escape((?h,?t), (0,[])) = x.
    tries to bind h to the head of x and t to the tail of list x. But if x is an empty list, it instead binds h = 0 and t = [].

    Orthogonality

    You can put any pattern in any place where you can put a pattern identifier. For example, any place where you might see ?x in a pattern, you can also put (?u,?v). So you can build complicated patterns. For example, 
          Match [?,(?x,?n+1) & ?] = [("four",4), ("five", 5), ("six", 6), ("seven", 7)].
    binds x to "five" and binds n to 4.
