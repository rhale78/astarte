LN3. Conversions Between Numeric Species

The following conversions are available. Some are shown with two different polymorphic types, the first for the case where you do not import math/math.ast, the second for the case where you do import that package.

+
: REAL`a -> REAL`b
: NUMBER`a -> NUMBER`b

+(x) is the same number as x, but cast to another numeric species.

Unary operator + can only be used to cast to a more general species, in the direction Natural -> Integer -> Rational -> Real -> Complex. It can perform such a cast underneath the wrapping of a secondary item.

~
: REAL`a -> Real

~(x) is the result of converting x to species Real. Here, x can be of any standard numeric species (Natural, Integer, Rational or Real).

Example: ~15 = 15.0

_as_
: (REAL`a, REAL`b) -> REAL`b
: (SEMIGROUP`a, SEMIGROUP`b) -> SEMIGROUP`b

x _as_ y is the same number as x, but cast to the species of y. Only conversions that preserve values are allowed, except that conversions to Real might round numbers. x _as_ y fails with exception conversionX if the conversion is impossible.

compatify
: (REAL`a, REAL`b) -> (REAL`c, REAL`c)
: (NUMBER`a, NUMBER`b) -> (NUMBER`c, NUMBER`c)

compatify(x,y) takes two numbers and forces their values to the most general of the two species. For example, compatify(2:Natural, 3:Rational) = (2:Rational, 3:Rational). Compatify is an irregular function, so the result species is checked at run time.

integer
: INTEGER`a -> Integer

integer(x) is the result of converting x to species Integer. x can be a natural number or an integer.

nabs
: INTEGER`a -> Natural

nabs(x) is the absolute value of x, of species Natural. x can be a natural number or an integer.

natconst
: Natural -> REAL`a
: Natural -> NUMBER`a

natconst(n) is the result of converting natural number n to an arbitrary numeric species. When you write natural number constant 2, you implicitly mean natconst(2).

natural
: INTEGER`a -> Natural

natural(x) is the result of converting x to species Natural. x can be a natural number or an integer. If x is negative, the conversion fails with exception conversionX.

ratconst
: Rational -> RFIELD`a
: Rational -> NFIELD`a

ratconst(x) is the result of converting rational number x to some numeric species. When you write constant 2.4, you implicitly mean ratconst(2.4).

rational
: REAL`a -> Rational

rational(x) is the result of converting x to species Rational. x can be any of the standard numeric species.
