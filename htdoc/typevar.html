    • Type variables

    A type variable stands for an unknown or unspecified or arbitrary species. Basic forms are as follows.
    G`a
    This is a variable that can take on any species in data abstraction G. Following the ` (the single left quote or grave accent character) is a name that can be any ordinary identifier. For example, ORDER`xyz is a variable that can stand for any species in ORDER.

    Two occurrences of the same variable must be the same species. So (REAL`x,REAL`x) can stand for (Natural, Natural) or (Integer, Integer), but not (Integer, Natural).

    The abstraction G in G`x is considered to be part of the name of the variable. For example, ORDER`a and EQ`a are different variables.

    `a
    `a abbreviates ANY`a. Note that, if you write both `a and ANY`a in a declaration, the two are the same variable.

    G`?
    This is an anonymous variable. Two occurrences of this variable are always considered different variables. You can also use this with other forms, such as `? and G``? (see below).

    Other variable forms

    The following forms of variables are for use with secondary species and for controlling type inference.

    G``a
    G``a is a variable that can only stand for a secondary species in genus G. For example, REAL``x might stand for REAL or INTEGER, but cannot stand for species Real.

    ``a
    ``a abbreviates ANY``a.

    G`*a
    G`*a is a variable that can only stand for a primary (non-secondary) species in genus G.

    `*a
    `*a abbreviates ANY`*a.

    G`~a
    Variable for G`~a is only used for determining sources of type restrictions. It is a variable that is like G`a, but it cannot be restricted. Use this form when you have a definition that is not as polymorphic as you think it should be to find out the cause of the loss of polymorphism. You can use the ~ modifier after `* or `` as well.

    When are variables the same?

    Two variables are considered to be the same if
    1. Neither is an anonymous variable, and
    2. They occur in the same declaration, and
    3. They have the same genus and the same name, and
    4. They have the same modifiers.
    So, For example, ORDER`a and EQ`a are considered different variables. EQ`a and EQ``a are also different variables.

    Renaming of variables

    The compiler does not attempt to keep your names for variables. It changes names at its own discretion. For example, if you define 
          Let f(?x: `elephant, ?y: `camel) = (y,x).
    the compiler will report 
          f: (`a,`b) -> (`b,`a)

    Variables that stand for families

    This section requires knowledge of parameterized data abstractions.

    Variables can stand for parameterized species as well as plain species. For example, COLLECTION is a parameterized data abstraction. COLLECTION`a stands for one of the families in COLLECTION. You can write COLLECTION`a(`b) for an arbitrary kind of collection holding an arbitrary species of value.

    All of the variable forms that are available for type variables are also available for family variables.
