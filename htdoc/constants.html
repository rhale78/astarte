E3. Constants

()
() is a special value called the hermit. This value is the only member of its species, called the unit species, since this species has only one value in it. The unit species is also called ().

24
Integer constants, using decimal notation, are allowed. An integer constant can have any numeric species.

When you write 2, the compiler substitutes natconst(2); when you write 234, the compiler substitutes natconst(234). You can add definitions to function natconst to generalize integer constants to other species. See natconst.

2.0
These are rational number constants. You must have at least one digit on each side of the decimal point, and can optionally have an exponent of 10, as in 2.5E40 or 2.5e40.

When you write 2.0, you implicitly mean ratconst(2.0). See ratconst.

'a'
This is a character constant. See characters

"abc"
This is a string constant. A string is a list of characters. String "abc" is the same thing as the list of three characters ['a', 'b', 'c'].

To include a double quote in a string constant, precede it with a backslash, as in "\"".

%"long string
%"constant
A long string constant begins with %". Successive lines whose first non-white characters are %" are concatenated together, each followed by a newline character, to form the long string constant. For example: 
      %" This is a long string constant.
      %" All consective lines that begin with %"
      %" form a single multi-line string.
If a line ends on \, no newline character will be added to the end of that line. So, while the above string constant has three newline characters in it, the following has none. 
      %" This is a long string constant.\
      %" All consective lines that begin with %"\
      %" form a single multi-line string.\
