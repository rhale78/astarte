E15. Threads

A thread is a separate flow of control. It is similar to a process. You can imagine that more than one processor is running a program simultaneously.

Astarte allows a program to run more than one thread at a time. Mechanisms are provided for creating threads (E17, L15), for terminating a thread (E16, L15) and for controlling and managing threads (L15).

The basic thread creation mechanism performs a fork. Where one thread enters an expression, two or more threads can come out. It is important to keep that in mind in reading some parts of the next two chapters of this manual.
