    • Compile errors

    If you get a compile error, always try to fix the first error first. Sometimes, that error will cause later errors. For example, if there is an error in the definition of function f, then f will not be defined, and later, when you try to use f, you will get an error complaining that f is not defined.

    Common errors are as follows.

  • An identifier being defined recursively is not defined in its own definition.
  • Species mismatch.
