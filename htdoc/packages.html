    • Packages

    Every file defines one package or one part of a package. A package starts with the word Package and end with %Package.

    There are three kinds of packages.

    1. A package that is not imported by any other package
    2. A package that is to be imported by another package, and that is entirely self-contained in a single file.
    3. A package that is broken into two files, consisting of an interface file and an implementation file.
    To import a package, see importing packages.

    Simple packages

    A simple package can import other packages, but cannot itself be imported by any other packages. It has the form
    Package name
      declarations
    %Package
    where name is the name of the package and declarations is a sequence of declarations. See example package for an example. This package should be put into a file with extension .ast, such as myfile.ast. Normally, the name of the package is the same as the name of the file, less its extension (.ast).

    Complete, importable packages

    A package that can be imported by another package can either be complete in one file or broken into two files. The form that is complete in one file looks like this.
    Package name
       export
       declarations
       implementation
       declarations
    %Package
    This package should be put into a file with extension .ast. All declarations are allowed in each part, but typically the export part contains expect declarations and the implementation part contains definitions that meet the expectations. See example package for an example.

    When declarations in the export part are read by another package (during an import) certain declarations are subject to special handling. See expect declarations.

    It is admissible to leave out the word implementation, and to put the entire package in the export part. That is not normally recommended, however. Only put in the export part what you want to tell everybody else about.

    Breaking up an importable package

    A package can be put into two files, one holding the interface (or export part) and another holding the implementation. The interface should be put into a file with extension .ast. It has the following form.
    Package name
       interface
       declarations
    %Package
    The implementation should be put into a file with the same base name as the interface, but with extension .asi. For example, if the interface is in file myfile.ast, then the implementation should be in file myfile.asi. The implementation has the form
    Package name
       implements "filename"
       declarations
    %Package
    where filename is the base name of the interface file (such as myfile). See example package for an example.

    Importing packages: basic imports

    To import a package, use an import declaration. The basic form is
    Import "filename".
    where filename is the path and base name of a file to import. For example, to import file myfile.ast in the current directory, use declaration 
      Import "myfile".
    The quotes around the file name are required. The import reads all declarations in the export or interface part of the package. To import standard package collect/search.ast, you write 
      Import "collect/search".

    Directories

    Normally, the compiler looks in the current directory and in the standard library directory (in that order) for packages to import.

    You can set environment variable AST_IMPORTS to a list of directories to search. This overrides the default. Use either colons or semicolons to separate the directories. You probably want to include "." in the list, since otherwise the current directory will not be searched.

    The current directory is always defined to be the directory that holds the file where the import declaration occurs.

    You can also use a directory declaration to choose directories. In the list of directories, you can use $ to stand for the directory where the standard library is stored. Declaration 

         Directory ".:./utils:$".
    indicates that directories ".", "./utils" and the standard library directory should be tried at each import, in that order. Use colons or semicolons to separate the directories.

    You can give names to directory strings. Use 

         Directory assortment1 = ".:./special".
    to define assortment1. Then 
         Directory assortment1.
    causes the directory search string to become ".:./special". Predefined names are standard = "$" and none = ".:$". So 
         Directory standard.
    means that subsequent imports must come from the standard library, until another directory declaration is given.

    Directory declarations that set the current search list only affect the current package. A directory declaration in an imported package is no longer in effect when the imported package is done, and the compiler returns to the current package. Declarations that define names, such as assortment1 above, are imported.

    Characteristics of imports

    When a package is imported, only its export or interface part is read. Any imports that are done by the imported package (such as myfile) within its export or interface part are implicitly done also by the importing package. Any imports done by the imported package's implementation are not visible to the importing package.

    Visibility of declarations is strictly controlled. For example, suppose that package A imports packages B and C, as follows. 

      Package A
    Import "B".
    Import "C".
    ...
  %Package
    When the export or interface part of package C is read, declarations that were made in package B (or declarations made in A before importing C) are not visible to the declarations within package C, and do not affect them.

    A package can be imported more than once. It does not interfere with itself. Importing a package twice is equivalent to importing it once.

    Cyclic imports (where a package imports itself directly or indirectly) are allowed as long as at least one of the imports is in the implementation part of a package. It is not allowed to do cyclic imports where all imports are in the interface, or export, part.

    Partial imports

    You can select items to import by listing them after the word Import, and following them by the word from. For example, to import only a, b and c from the package in file lib.ast, you write 
       Import a,b,c from "lib".
    Only the names that you have mentioned will be imported. This only applies to names of items (functions, numbers, etc.). All species, genera, etc. are imported, even if you do not mention them.
