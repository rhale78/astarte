    • Function expressions

    Functions are values in Astarte. You can give functions names, put them into variables, return them from other functions, etc.

    The most basic way to define a function is using an expression of the form (?x => e) where x is an identifier and e is an expression that can use x. The function that this expression denotes is the function f(x) = e. For example, 

          (?x => 2*x)
    is the function that returns twice its argument.

    In general, you can use an expression of the form (p => e) where p is a pattern and e is an expression. This is the same as (?x => Match p = x %Match (e)), where x is some new identifier. For example 

          ((?x,?y) => x + 2*y)
    is the function f defined by f(x,y) = x + 2y. (However, this function is anonymous; it is not called f or anything else.)

    Defining functions: single case

    Usually, you want to create a function and give it a name simulataneously. You can do that as follows. All of the forms shown in this and subsequent sections work equally well for let, define or relet statements, even if they are only shown for one of those kinds of statements.

    Let f p = e %Let
    You can define a function using a let statement. Definition 
      Let f p = e %Let
    is equivalent to 
      Let f = (p => e) %Let
    A let-statement cannot be recursive.

    Example: Let dbl(?x) = 2*x.

    Define f p = e %Define
    You can define a function using a define statement. The form shown is equivalent to 
       Define f = (p => e) %Define
    A define statement can be recursive. For example, the factorial function can be defined as 
        Define factorial(?n) =
      If n == 0 then 0 else n*factorial(n--1) %If
    %Define

    Let f p q = e %Let
    A curried function is a function that produces another function as its result. You can define a curried function using a let or define statement. The let statement shown above is equivalent to 
       Let f = (p => (q => e)) %Let
    You can use any number of parameters. For example, function compose such that compose f g x = f(g(x)) can be written as follows. 
         Let compose ?f ?g ?x = f(g(x)).
    This definition of compose is equivalent to 
         Let compose = (?f => (?g => (?x => f(g(x)) ))).
    So compose is the function composition function; (compose f g) is the function h defined by h(x) = f(g(x)).

    Defining functions: multiple cases

    If your function definition has several cases, you can use the form 
         Let f by choice-mode
       cases
     %Let
    or 
         Let f p by choice-mode
       cases
     %Let
    where cases is a sequence of cases, choice-mode tells how the cases are tried, and p is a pattern that is matched against the argument. For example, a function that takes the maximum of two values can be written as follows. 
         Let max by first
       case max(?a,?b) = a  when a >= b
       else max(?a,?b) = b
     %Let
    If you prefer, you can write it as follows, although it does not read as well. 
         Let max(?a,?b) by first
       case max(?) = a  when a >= b
       else max(?) = b
     %Let
    There are similar forms for define and relet statements.

    Choice modes

    The options for choice-mode are as follows.

    first
    Try the cases in the order written, choosing the first that is applicable. If you omit the choice-mode, the default is first.

    one
    Try the cases in an arbitrary order, choosing the first one that is encountered that is applicable. This is useful in telling a reader that the cases can be read independently of one another. When reading the third case, for example, you are not required to be aware that the third case will only be tried if the first two cases are not chosen. The third case might, in fact, be the first one tried.

    one mixed
    Try the cases in parallel, choosing the first one whose selection condition succeeds. The selection condition succeeds if the pattern match succeeds and the when condition, if present, is true.

    all
    Try all cases using separate threads, using a stream that produces the result of the first case first, etc. See streams.

    all mixed
    Try all cases using separate threads, using a mixed stream, and producing the results in an arbitrary order. See streams.

    Definition case forms

    The cases have the forms

    case f p = e when c
    Define f(p) = e, where p is a pattern. This definition is only used if p matches the input and condition c is true. You can omit the when c phrase.

    case f p q = e when c
    This is similar to the preceding case, but for a curried function. You can have any number of parts.

    else f p = e
    This can only used as a last case. This case will only be taken if no other cases apply. (A case applies if its pattern match succeeds and its when clause, if present, is true.)

    do f p => e
    When this case is encountered, pattern p is matched against the function argument, and expression e is evaluated. Expression e must have species (). If the match fails, then evaluation of the function fails. If the choice mode is one or one mixed or all mixed, then this can only be the first case.

    Any bindings done in a do-case are defined for all subsequent cases. When the choice mode is all, the do phrase is done by a thread that is common to all the cases that follow.

    There are curried do-forms, such as do f p q r => e, as well.

    Actions at the end of definitions with multiple cases

    As a final case in a definition with multiple cases, you can have a final action. It has the form 
          then p => s
    This action will be done after the function value has been computed. It will match pattern p against the result of the function, and then will evaluate statement s. For example, 
          Let max(?a,?b) by first
        case max ? = a when a >= b
        case max ? = b 
        then ?res => Writeln["max(", $a, ",", $b, ") = ", $res].
      %Let
    defines a function that takes the maximum of two values and prints an indication of what it did. It still returns the result.
