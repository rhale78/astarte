LC5. Conversion to and from Strings

  • Conversions to strings
  • Conversions from strings
  • Defining $

    • Conversion to strings

    The following operations are all available without any imports.

    $
    :`a -> String

    Most species have an operation $ that converts an item to a string. When $ is applied to a string, it adds quotes. So $("abc") = "\"abc\"". See the notes on defining $ below.

    Example: $(35) = "35" and $([1,2,3]) = "[1,2,3]".

    $'
    : `a -> String

    $'(x) is the same as $(x), but it puts commas in numbers every three digits.

    $$
    There is a formatted conversion available for some species. It is called $$. You generally write x $$ k, where k tells formatting information. For example, "abc" $$ 5 is string abc padded on the left with blanks to reach 5 characters. If the string already has k or more characters, then no padding is done; it is not shortened.

    Example: 3 $$ 5 = "     3".

    $$'
    x $$' n is the same as x $$ n, but it puts commas in numbers every three digits.

    _leftin_
    : (String,Natural) -> String

    s _leftin_ n is the result of padding string s on the right so that its length is n. If n already has n or more characters, then no padding is done.

    Example: "abc" _leftin_ 5 = "   abc".

    showTagged
    :`a -> String

    showTagged(x) is $x followed by typeof(x).

    Example: showTagged("abc") = "\"abc\":[Char]".

    view
    :`a -> String

    view(x) is similar to $(x), but it does not evaluate any unevaluated parts of x. Unevaluated parts are shown as "(lazy ...)" where ... tells what function is responsible for the evaluation. Use it to see what is happening in your program.

    Conversion from strings

    The following convert from a string to another type. You can also use the read and pull functions described in reading from strings. See collect/string.ast for details of the following.

    stringToNatural
    : String -> Natural

    Function stringToNatural converts a decimal numeral into a natural number.

    Example: stringToNatural("124") = 124

    hexToNatural
    : String -> Natural

    hexToNatural(s) is the number that hexadecimal numeral x represents.

    Example: hexToNatural("1A") = 26

    stringToInteger
    : String -> Integer

    Function stringToInteger converts a decimal numeral into an integer.

    Example: stringToInteger("-124") = -124

    stringToRational
    : String -> Rational

    Function stringToRational converts an Astarte-style constant to a rational number.

    Example: stringToRational("12.4") = 12.4

    stringToReal
    : String -> Real

    Function stringToReal converts an Astarte-style constant to a real number.

    Example: stringToReal("12.4") = 12.4

    Defining $

    When you define a new species, a definition of $ is provided for you automatically. The standard definition is fairly crude. If you declare 

        Species NewSpecies = something(Natural) | nothing.
    then $(something(2)) = "something(2)". You often want to use your own definition of $. To create your own, use option noDollar with the species declaration, as in 
        Species{noDollar} NewSpecies = something(Natural) | nothing.
    and then define $ in your own way.

    When you define $, you must be aware of function view. When view is called, it sets up context to call for lazy parts of items not to be evaluated, and then applies $. The definition of $ must be designed to check the context, and to suppress evaluation of lazy parts when appropriate. You can use the following tool when defining $.

    lazyDollar
    : `a -> String

    lazyDollar(x) is a string such as "(lazy)" if x is lazy AND the current thread is currently doing a view. Otherwise, lazyDollar(x) fails with exception testX.

    For example, to define $ for NewSpecies in such a way that $(something(3)) = "3" and $(nothing) = "none", you can write 

         Let $(?x) = 
         lazyDollar(x) 
       _butIfFail_
         Choose first matching x
           case something(?n) => $ n
           case nothing       => "none"
         %Choose
     %Let
