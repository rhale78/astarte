    • General rules

    You can create a new species using a species declaration. Each species declaration must be inside an extension, of the form 
          Extension
        ...
      %Exension
    Always use a name that starts with an upper case letter for a species.

    Species with a single constructor

    Declaration 
          Species N = c S.
    defines new species N to be represented by species S. For example 
          Extension
        Species Distance = distance(Real).
      %Extension
    creates species Distance, and indicates that each member of species Distance corresponds with, or is represented by, a real number. In general, declaration 
          Species N = c S.
    causes the following functions to be defined automatically.

    1. c: S -> N. This is the constructor, for building a member of the new species from its representation.

      In the distance example, you get a function called distance, of species Real -> Distance. So distance(5.0) is the distance that corresponds to real number 5.0. Think of it as 5.0 meters.

      You can use the constructor in patterns, as a way of finding out the representation of a value of species N.

      In the distance example, if you have defined d to be distance(3.5), then 

            Match distance(?x) = d.
      binds x to the real number 3.5.

    2. unc: N -> S. This is the inverse function of the constructor c, called the destructor function.

      In the distance example, this function is called undistance. For example, if d is distance(3.5), then undistance(d) is the real number 3.5.

    3. ==: (N,N) -> Boolean. Normally, a new species is automatically added to EQ, and a default definition of the equality operator == is created. This equality comparison considers two values to be equal if they have the same representation.

      In the distance example, two distances are considered equal if they are represented by the same real number.

      You can suppress adding the new species to EQ and the automatic definition of == using option noEqual, as in 

             Species{noEqual} N = c S.

    4. $: N -> String. This definition of $ just adds the name of the constructor c to the result of applying $ to the representation of a value.

      In the distance example, if d is distance(2.6), then $(d) is string "distance(2.6)".

      You can suppress automatic definition of $ by using option noDollar, as in 

             Species{noDollar} N = c S.

      To suppress both == and $, use both noEqual and noDollar, as in 

             Species{noEqual,noDollar} N = c S.

    5. pull: String -> (N, String). Function pull reads from a string that describes a member x of N from the front of a string, and returns (x,r), where r is what follows the description of x in the string.

      In the distance example, pull("distance(2.6) and more") = (distance(2.6), " and more").

      The forms that can be read by pull start with the name of a constructor and are followed by the representation value, possibly enclosed in parentheses. If desired, the entire value can be enclosed in parentheses. For example, pull("(distance 4.0) and more") = (distance(4.0), " and more").

      This definition of pull allow you to use Extract to read a value from a box. For example, 

          Assign b! := "distance(64) and more string".
    Extract [$(?d:Distance)] from b!.
      binds d to distance(64) and leaves box b! holding " and more string". Using either of the options noDollar or noEqual suppresses the automatic definition of pull as well as the definition of $ or ==. To suppress only the definition of pull, but not of $ or ==, use option noPull.

    Species with multiple constructors

    You can define a species with several constructors. Such a species has different kinds of members, identified by the constructor used. Just separate the parts by bars. For example, 
          Extension
        Species MyNewSpecies = con1(Real) | con2(Natural, Natural).
      %Extension
    defines species MyNewSpecies with two constructors, con1 and con2. Values constructed by different constructors are different values. A few of the members of species MyNewSpecies are con1(3.2), con1(4.66), con2(3, 4) and con2(5, 1).

    You can use the constructors in patterns. For example, suppose that m is con1(2.4). Then 

          Match con1(?x) = m.
    succeeds, and binds x to 2.4. But 
          Match con2(?x,?y) = m.
    fails (with exception testX), since m was not constructed using constructor con2.

    Inverses of functions con1 and con2 are provided. They are called uncon1 and uncon2. For example, uncon1(con1(2.0)) = 2.0. Expression uncon2(con1(2.0)) fails with exception testX.

    Testers con1? and con2? are also defined. They are used to test which constructor was used to create a particular item. For example, con1?(con1(2.5)) is true, but con1?(con2(4,6)) is false.

    Functions ==, $ and pull are defined in a similar way as for species with a single constructor.

    Options for species declarations

    1. You can omit the name of the constructor, if there is only one constructor. The constructor name defaults to the name of the species, with the first letter forced to lower case. So 
            Extension
        Species Distance = Real.
      %Extension
      is equivalent to 
            Extension
        Species Distance = distance(Real).
      %Extension

    2. You can omit the representation type. It defaults to (). When the representation type is (), the constructor is not defined as a function, but instead is defined as a member of the new species. For example, 
            Extension
        Species Color = red | blue | green.
      %Extension
      defines red: Color, blue: Color and green: Color. The testers red?, blue? and green? are defined as usual.

    3. You can place a vertical bar in front of the first alternative if desired. For example, the definition of species Color can be written 
            Extension
        Species Color = 
          | red 
          | blue 
          | green
        %Species
      %Extension

    4. You can include a mode in front of a constructor. The mode is added to (and overrides where appropriate) mode information from the species declaration. For example, 
        Species Book = {private} book(String).
      indicates that the constructor book should be marked private to the current package. If you use a mode, you cannot omit the constructor.

    Using operators and symbolic identifiers as constructors

    You can use binary operators as constructors. For example, you might define a species Expression as follows. 
          Extension
        Species Expression =
            Expression + Expression
            Expression * Expression
            - Expression
        %Species
      %Extension
    You get three constructors, +:(Expression, Expression) -> Expression, *: (Expression, Expression) -> Expression and -: Expression -> Expression.

    When you use a symbolic identifier as a constructor, the inverse of the constructor is given a name that is obtained from the constructor by adding /-. For example, you get +/-: Expression -> (Expression, Expression). (In the special case of identifier ::, the constructor inverse is called &/-.)

    Curried constructors

    A constructor can have a curried form. For example, declaration 
         Species Golf = golf Natural Real
    defines constructor golf to have species Natural -> Real -> Golf.

    The destructor produces all parts of the representation as a single tuple. For example, ungolf(golf 2 4.5) = (2, 4.5).

    The default $ and pull functions are defined so that, For example, $(golf 2 4.5) = "golf(2)(4.5)" and pull("golf 2 4.5 99") = (golf 2 4.5, " 99").

    Enumerated species

    If you define a species all of whose members are simply given by names, then you have defined an enumerated species. An example is species TrafficLightColor defined as follows. 
        Extension
      Species TrafficLightColor = red | amber | green.
    %Extension
    When you define an enumerated species, that species is automatically added to genus ENUMERATED. A few functions are also defined, illustrated here for species TrafficLightColor.

    rank
    : TrafficLightColor -> Natural

    The rank of a member of an enumerated species is its location in the definition list, where numbering is from 0. So rank(red) = 0, rank(amber) = 1 and rank(green) = 2

    unrank
    : Natural -> TrafficLightColor

    This is the functional inverse of rank. So unrank(0) = red, For example.

    trafficLightColor
    : Natural -> TrafficLightColor

    This is just another name for unrank.

    compare
    : (TrafficLightColor, TrafficLightColor) -> Comparison

    This compares two members, according to the rank ordering. For example, compare(red,green) = less. You also get comparisons <, >, <= and >= automatically from this.

    pred
    TrafficLightColor -> TrafficLightColor

    pred(x) is the predecessor of x. For example, pred(green) = amber. Expression pred(red) fails with exception domainX("predecessor of smallest").

    succ
    : TrafficLightColor -> TrafficLightColor.

    succ(x) is the successor if x. For example, succ(amber) = green. Expression succ(green) fails with exception domainX("successor of largest")

    _upto_
    : (TrafficLightColor, TrafficLightColor) -> [TrafficLightColor]

    This produces a list of contiguous members, in ascending order. For example, red _upto_ green = [red, amber, green].

    _downto_
    : (TrafficLightColor, TrafficLightColor) -> [TrafficLightColor]

    This produces a list of contiguous members, in descending order. For example, green _downto_ amber = [green, amber].

    smallest
    : TrafficLightColor

    This is red, the smallest traffic light color.

    largest
    : TrafficLightColor

    This is green, the largest traffic light color.

    All enumerated species are members of genus ENUMERATED and of genus RANKED. Here is part of the standard genus hierarchy, with smaller things to the right. 
                                                ___ ()
                                           /
                                          /____ Boolean
                                         /
        ORDER ___ RANKED _____ ENUMERATED _____ Comparison
                        \                \
                         \                \____ Char
                          \                \
                           \                \___BoxFlavor
                            \                \
                             \                \_Infinity
                              \
                               \_______________ Natural
    The only way to add a new species to ENUMERATED or RANKED is to define an enumerated species.

    Defining selectors

    In a species declaration, you can define selectors that return parts of a value. You do this by marking a component with s ~>. For example, 
       Species River = 
      (name   ~> String,
       length ~> Distance,
       flow   ~> Real
      )
   %Species
    defines three extra functions for you: name, length and flow. If you define r to be river("Nile", distance(1000), 10000) then name(r) = "Nile", length(r) = distance(1000) and flow(r) = 10000.

    You can defined selectors when there are multiple constructors. A selector that is applied to the wrong kind of object fails with exception testX. For example, if you define 

      Species A = a1(sel1~> Natural) | a2(sel2~> Integer, sel3~> Real).
    then sel1(a1(4)) = 4, but sel1(a2(6,5)) fails with exception testX, since selector sel1 can only select from a value that was constructed by constructor a1.

    You can include a mode in front of a selector. That mode augments (and replaces of appropriate) modes given with the species declaration. For example, 

       Species River = 
      ({protected} name   ~> String,
       {private}   length ~> Distance,
                   flow   ~> Real
      )
   %Species
    indicates that selector name is protected, length is private and flow is public.

    Expecting species

    You can make a promise that a species will be defined later. You typically do this in the interface to a package to indicate that a species is defined, but to hide the details of its definition. Declaration 
      Extension
     Expect species T.
  %Extension
    promises that species T will be defined. Use mode noEqual, as in 
      Extension
     Expect{noEqual} species T.
  %Extension
    if you do not intend to define equality for this species.

    Providing descriptions

    You can provide a description of a species in a declaration that defines or expects the species. The description is a string constant, and is put at the end of the declaration. Here are some examples. 
        Expect species Pelican
      %" Species pelican contains birds with large beaks.
    %Expect

    Species Pelican = Bird
      %" Species pelican contains birds with large beaks.
    %Species

    Adding species to abstractions

    When you create a species, you can add it to one or more abstractions. Just write isKindOf A,B,C after the name of the species to add it to abstractions A, B and C. For example, 
        Species Pelican isKindOf BIRD = SpecialBird.
