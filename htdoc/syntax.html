M4. General Lexical And Syntactic Issues

  • Comments

    • Comments

    1. A short comment starts at %% and ends at the end of a line.

    2. A long comment starts at any sequence of four or more consecutive < signs, and ends at any sequence of four or more consecutive > signs.

    3. You can place a sequence of four or more = signs any place a declaration can occur. It is a comment, and cannot occur inside a declaration. Use it to separate declarations. 
        %% This is a short comment

    <<<<<<<<<<<<<<<<<<<<<<<<<<<<
    This is a long comment.
    Notice that the arrows point to the
    noncomment part of the program.
    >>>>>>>>>>>>>>>>>>>>>>>>>>>>

    =============================
    Within a long comment you can temporarily exit the comment by using %>. From %> to the end of the line is not considered part of the comment -- it is read by the compiler. Some people like to program where most of the time they are writing comments, and they occasionally write code. For example, the definition of a function might go like this. 
        <<<<<<<<<<<<<<<<<<<<<<<<<<
    This function computes the product of the members of a list.
    Blah blah blah.
  
    %> Define prod by
    %>   case prod [] = 1
    %>   case prod (?h::?t) = h * prod(t)
    %> %Define

    Now we are back into a comment.

    Bracketing

    Most constructs begin with a word (such as If), and end with the same word, preceded by a percent sign (such as %If). There can be no space between % and If. By convention, words that have a matching end marker are capitalized.

    You can use a period as a wild-card end marker, matching an appropriate begin word. However, a period can only be used when it occurs on the same line as the matching begin word.

    For example, here is a definition of a power function. Notice the matching of begin and end words, and the periods, which stand for %Let at one place and %Value at others. 

      Define power(?x,?n) =
    If n == 0
      then Value 1.                    %% period stands for %Value
      else
        Let s = power(x,n--1).         %% period stands for %Let
        Value s*x.                     %% period stands for %Value
    %If
  %Define

    Semicolons

    All declarations and some expressions allow a special notation for chaining several of the same kind of thing together. A semicolon stands for %W W, for an appropriate word W. For example, 

        Define
      x = 1;
      y = 2
    %Define
    has the same effect as 
        Define x = 1.
    Define y = 2.
    since the semicolon stands for %Define Define. You can have a semicolon after the last part; it is ignored. So 
        Define
      x = 1;
      y = 2;
    %Define
    is acceptable.
