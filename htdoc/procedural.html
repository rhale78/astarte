    • Procedural programming is a form of imperative programming, where a program computes by performing actions, and a program contains commands to perform those actions.

    Locally imperative programming by name rebinding

    The simplest model of imperative programming is name rebinding. As a program runs, it changes the values that names are bound to.

    In Astarte, you can rebind a local name using Relet. For example, here is an implementation of the factorial function. 

      Define factorial ?n =
    Let r = 1.
    Let k = 0.
    While k < n do
      Relet k = k + 1.
      Relet r = r * k.
    %While
    Value r.
  %Define

    Procedures

    A procedure is a function that produces value (). In Astarte, a procedure is normally given a name that starts with an upper case letter. According to the syntactic conventions, when a procedure P is applied, a matching end marker %P (or a period that stands for it) must be provided at the end of the call. That must be done regardless the context in which the procedure call occurs. For example, here is a definition of a procedure that prints the factorials of the numbers from 1 to n. 
      Define PrintFactorials ?n. =
    Let k = 1.
    While k <= n do
      Writeln[$(factorial k)].
      Relet k = k + 1.
    %While
  %Define
    To print the factorials from 1 to 20, you write 
      PrintFactorials 20.

    Boxes

    An alternative to rebinding names is to use data items that have characteristics that can be changed. A box has a content that can be changed. The content of box b is written @b.

    You can create a box and give it a name using a Var-expression. Expression 

      Var x.
    creates a new box named x. The species of this box will be determined by type inference. Use Var x,y,z. to create three boxes.

    It is conventionl in Astarte to choose a name for a box that ends on an exclamation point, so that you can be reminded that it is a box.

    Locally imperative programming using boxes

    Boxes are data items, and can be passed among functions and procedures just like other data items. Suppose that b is a box, and you write P(b). Do you intend to pass the box b itself to procedure P, or do you actually want to pass the content of box b to P? The distinction is an important one. If the box is passed, then procedure P has access to that box, and can change its content. If only the content of the box is passed, then procedure P does not receive the box at all, and so cannot change its content.

    Procedural programming languages make that determination from context. For example, in C++ the determination is made by the kind of parameter that is being passed. If the parameter is a value parameter, then the content of the box is passed. If the parameter is a reference parameter, then the box itself is passed.

    Astarte does not make the determination of whether you mean a box or the content of the box. That is up to you. Expression f(b) always passes box b to f. Expression f(@b) passes the content @b of box b to f.

    The content of a box is changed by an assignment statement. The form is 

      Assign b := v.
    where v is the value to be put into the box. Assign is a procedure in Astarte, and statement Assign b := v %Assign passes the ordered pair (b,v) to that procedure. Like any other function or procedure, you must say exactly what should be passed to it. If you want to put 2 into box b, then you must pass the pair (b,2).

    Suppose that x and y are two boxes, and you want to copy the content of box y into box x. Then you must pass pair (x,@y) to the Assign procedure. So you write 

      Assign x := @y.

    Here is a definition of the factorial function using boxes to handle the imperative aspects. 

      Define factorial ?n =
    Var r!,k!.
    Assign r! := 1.
    Assign k! := 0.
    While @k! < @n! do
      Assign k! := @k! + 1.
      Assign r! := @r! * @k!.
    %While
    Value @r!.
  %Define
    To use boxes, be sure that you put @ in front of them when you want their contents.

    Arrays

    An array is, logically, a list of boxes. Normally, an array is stored in consecutive memory locations, which allows rapid subscripting, but that only affects how efficiently the boxes are accessed.

    You can create an array using a var expression. For example, 

      Var arr(n).
    creates an array of size n. The type of the boxes is determined by context. arr#k is the k-th box in this array. Keep in mind that arr#k is a box. To get the content of that box, use @(arr#k). Alternatively, you can get the entire array contents, as a list, using contents(arr).

    Here is an example that performs an insertion sort on an array. 

      Define 
    description
      %" Insert(arr,k) sorts arr(1..k) into ascending order, 
      %" under the supposition that arr(1..k-1) is already sorted.
      %" k must be positive.

    Insert(?arr,?k). =
      Let i = k--1.
      While i > 0 _and_ @(arr#i) > @(arr#(i+1)) do
         Swap(arr#i, arr#(i+1)).
         Relet i = i -- 1.
      %While
  %Define 

  Define
    description
      %" Sort(arr) sorts array arr into ascending order.

    Sort ?arr. =
      For ?i from 2 _upto_ length(arr) do
        Insert(arr,i).
      %For
  %Define

    Globally imperative programming

    Boxes can be put into data structures. An example of a data structure containing boxes is a hash table. A hash table presumes that a hash function is available that tells where in an array to put an item. If the hash function is called hashfun, then hashfun(k) _mod_ length(arr) tells where in array arr key k will be stored.

    A problem with hash tables is that two different keys can be stored at the same array index. This is called a collision. One solution is to store a list of items at each spot in the array. The list consists of pairs (k,v) consisting of a key and a value.

    Here is a partial implentation of hash tables. The support functions, such as removeKey, are by far easiest to write in a functional style, so we do that. 

      ===========================================================
  Exception hashTableX
    %" A lookup in a hash table has failed.
  %Exception
  ===========================================================
  Define 
    description
      %" newHashTable(n) returns a new hash table of size n.

    newHashTable(?n ) = arr |
      Var arr(n).
  %Define
  ===========================================================
  Define 
    description
      %" removeKey key lst returns the result of removing 
      %" the first pair (k,v) from lst where k = key.

    removeKey by first
      case removeKey ? []                    = []
      case removeKey ?key [(?key,?) & ?rest] = rest
      case removeKey ?key [?pr & ?rest]      = [pr & removeKey key rest] 
  %Define
  ===========================================================
  Define 
    description
      %" InsertIntoHashTable(arr,key,val,hashfun) modifies array
      %" arr by removing any existing pair (key,val'), and then adding
      %" pair (key,val).  
      %"
      %" Function hashfun is the hash function to use.

    InsertIntoHashTable(?arr,?key,?val,?hashfun). =
      Let index    = hashfun(key) _mod_ length(arr).
      Let oldEntry = @(arr #_ index).
      Let newEntry = (key,val) :: (removeKey key oldEntry)
      Assign arr#_index := newEntry.
  %Define
  ===========================================================
  Define 
    description
      %" RemoveFromHashTable(arr,key,hashfun) modifies array
      %" arr by removing any existing pair (key,val)
      %"
      %" Function hashfun is the hash function to use.

    RemoveFromHashTable(?arr,?key,?hashfun). =
      Let index    = hashfun(key) _mod_ length(arr).
      Let oldEntry = @(arr #_ index).
      Let newEntry = removeKey key oldEntry
      Assign arr#_index := newEntry.
  %Define
  ===========================================================
  Define 
    description
      %" lookupInHashTable(arr, key, hashfun) returns the value
      %" associated with key in hash table arr.  If there is
      %" no such key, it fails with exception hashTableX.
      %"
      %" Function hashfun is the hash function to use.  

    lookupInHashTable(?arr,?key,?hashfun) =
      Try
        Let index  = hashfun(key) _mod_ length(arr).
        Let result = assoc key  @(arr #_ index).
      then
        Value result.
      else
        fail hashTableX
      %Try
  %Define
  ===========================================================
    Data structures that contain boxes, and whose operations work by modifying the contents of those boxes, are called ephemeral data structures. Each time a modification is made, the former content of the data structure is lost. Ephemeral data structures can be quite efficient, but they need to be used with care. Be careful not to destroy something that you need!
