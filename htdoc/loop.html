E13. Loops

  • While loops
  • For loops
  • General loops
  • Examples of general loops
    • There are three kinds of loop.

    While loops

    Statement 
          While c do
        s
      %While
    repeatedly evaluates statement s until condition c is found to be true. If c is true initially, then s is not evaluated at all. For example, here is a factorial function written using a while loop. 
         Let factorial(?n) =
       Let result = 1.
       Let k = 0.           %% invariant: result = factorial(k)
       While k =/= n do
         Relet k = k + 1.
         Relet result = result * k.
       %While
       Value result.
     %Let

    The body of a while-loop (expression s in the general form) must have species (), and the while-loop itself also has species ().

    For loops

    A for loop has the form 
          For p from x do
        s
      %For
    Here, x is an expression that computes a list and p is a pattern. For each member m of list x, in order, pattern p is matched against m. If the match is successful, then statement s is evaluated. If the match is unsuccessul, then nothing is done for m, and the loop moves to the next value. Here is a factorial function written using a for loop. Expression (2 _upto_ n) produces list [2,3,...,n]. 
         Let factorial(?n) =
       Let result = 1.
       For ?k from 2 _upto_ n do
         Relet result = result * k.
       %For
       Value result.
     %Let

    The body of a for-loop (expression s in the general form) must have species (), and the for-loop itself also has species ().

    General loops

    Loop expressions provide a very general way of constructing loops. Unlike while loops and for loops, a loop expression can have a value different from (). Its form is 
         Loop p = i choice-mode
       cases
     %Loop
    where p is a pattern whose identifiers are the loop control variables and i is an initial value for those variables. When the loop starts, pattern p is matched against i. Then the cases are tried.

    Each case either indicates that the loop is finished (and tells the value of the loop expression) or indicates that the loop needs to go for another iteration, and indicates a new value for the loop control variables. Each time around the loop, pattern p is matched against the current value of the loop control variables. (If the match fails, then the loop fails.)

    You can omit p = i if you want a loop without control variables.

    choice-mode tells how the cases are tried. The choice modes are exactly as for choose expressions. When mode perhaps is used, the default else-case exits the loop with value ().

    If the choice mode is not perhaps, and there is no explicit else case, and none of the cases succeeds, then the loop fails. The exception is noCaseX for choice modes first, one and one mixed, and is testX for choice modes all and all mixed.

    The cases can have the following forms.

    exitcase a => v
    This case succeeds if a is true. When it is tried and succeeds, the loop exits, and the value of the loop expression is v.

    exitcase p => v when c
    This case is similar to the preceding form, but it is used when the choice-mode has a matching phrase. It succeeds if pattern p matches the target that is given after the word matching, and if c is true. Phrase when c can be omitted.

    exitcase do s => v
    Here, s must have species (). This case succeeds if evaluation of s does not fail.

    exitcase else => v
    This must be the last case. It is taken if no other cases are successful. It causes the loop to terminate, and the value of the loop expression to be v.

    exitcase a
    exitcase b => v
    Same as 
        exitcase a => v
    exitcase b => v
    You can have any number of cases.

    loopcase a => s continue w
    This case succeeds if a is true. When this case is tried and succeeds, statement s is evaluated. Then control passes to the top of the loop. Pattern p from the loop heading is matched against w, and the cases are tried again. Statement s can be omitted.

    loopcase a => s continue
    This form is used when p = i is omitted from the loop heading.

    loopcase p => s continue w when c
    This is similar to the preceding form, but is used with a matching phrase in the choice-mode.

    loopcase do a => s continue w
    This is similar to the preceding form, but succeeds if evaluation of a does not fail.

    loopcase else => s continue w
    This must be the last case. It is taken if no other cases are successful.

    loopcase a =>
    loopcase b => s continue w
    Same as 
         loopcase a => s continue w
     loopcase b => s continue w

    case a =>
       Subcases choice-mode
         cases
       %Subcases
    This case succeeds when a is true. When it succeeds, it tries the given subcases. If choice-mode is omitted after subcases, the choice mode is the same as for the enclosing loop or subcases, except that any matching phrase is omitted.

    case do a =>
       Subcases choice-mode
         cases
       %Subcases
    This is similar to the preceding form, but the case succeeds if evaluation of a does not fail. Expression a must have species ().

    do s
    When this case is encountered, s is evaluated. It must have species (). Any bindings done by s are visible in subsequent cases.

    do p => s
    This can only be used with a matching phrase. When this case is encountered, pattern p is matched against the target of the matches, and then statement s is evaluated.

    Examples of general loops

    Here is a loop that computes factorials. It uses two control variables, one to accumulate the result and one to keep track of where the computation is. Notice that no relets are required here. 
          Let factorial(?n) =
       
        %% Invariant: r = k! 

        Loop (?r,?k) = (1,0) first
          exitcase k == n => r             %% If k == n, the loop expression
                                           %% has value r.
          loopcase else   => continue (r*(k+1), k+1)
        %Loop
      %Let
    Here is a definition of a binary operator _in_, where x _in_ L is true if x is a member of list L. 
        Let ?x _in_ ?lst =
      Loop ?r = lst first matching r
        exitcase []       => false
        exitcase [x & ?]  => true
        loopcase [? & ?t] => continue t
      %Loop
    %Let
    Here is an implementation of library function findMemberAfter. This is a more involved loop, employing subcases. 
    Let findMemberAfter (?s,?sense) ?c ?lst =
  Loop (?n,?rest,?cc) = (0, lst, c) 
    first matching rest
      exitcase []   => fail testX
      case ?h :: ?t =>
        Subcases
	  exitcase (h _in_ s) == sense => (n, rest, cc)
	  loopcase else                => continue (n+1, t, h)
	%Subcases
  %Loop
%Let
