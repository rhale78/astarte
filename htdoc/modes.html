D2. Declaration Modes

  • Modes
  • Sharing modes

    • Modes

    Several kinds of declarations allow you to specify modes that influence how the declaration is processed. Modes are written in braces just after the word that introduces the declaration. For example, a let-declaration that overrides another declaration might be written as follows. 
         Let{override} myid = ... %Let
    You can use several modes. They are separated by commas, as in 
         Let{default,override} myid = ... %Let
    and can occur in any order.

    Sharing modes

    Several declarations of the same kind can be grouped together, with semicolons separating them. If you follow the right brace of the mode with a colon, then the mode applies across semicolons, to all following declarations in a group. For example, 
         Let {override}:
       myId1 = ...;
       myId2 = ...
     %Let
    is equivalent to 
         Let{override} myId1 = ... %Let
     Let{override} myId2 = ... %Let
    You can change the mode by providing a new mode with a colon following it. If you provide a mode without a colon, then that mode is added to the current mode (from the mode with a colon), and only applies to the current declaration. For example, 
         Let 
       {override}:

                 id1 = ...;
       {default} id2 = ...;
                 id3 = ...;

       {underride}:

                 id4 = ...;
     %Let
    uses the following modes.
    Identifier   Modes
    id1   override
    id2   override,default
    id3   override
    id4   underride
