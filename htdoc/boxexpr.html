    • Boxes

    A box is an item that has a content that can be changed. You can think of a box as a variable. When you store a new value in a variable, you change the content of the box.

    Expression <:a:> produces a new box whose initial content is a.

    A box can be empty, having no content at all. Expression <::> produces a new empty box.

    The species of a box

    A box containing a value of species T has species Box(T), which can be abbreviated <:T:>.

    Changing the content of a box

    You can put a new value into a box using an assign statement. Statement 

        Assign x := e.
    puts the value of expression e into box x.

    Getting the content of a box

    Most programming languages take the content of a box implicitly for you when the box occurs in a certain context. For example, in the programming language C, if you write x = y, you implicitly take the content of box y, and store that into box x. Astarte never takes the content of a box implicitly. You must do so explicitly by using operator @. To set box x to the content of box y, you write 

         Assign x := @y.
    Notice that you are taking the content of box y, but not of box x, since the assignment is into box x itself.

    By convention, boxes have names that begin with a lower case letter and end on an exclamation point. For example, there is a standard box called precision!.

    Most of the operations described here (those from lang/box.ast) are available without importing any packages. Those from other packages require that the other packages be imported. They are noted.

    Creating and naming boxes: Var expressions and declarations

    Var expressions and declarations can be used to create boxes and give them names. They can also be used to create arrays (lists of boxes). Here are some of the forms. More detail can be found in lang/box.ast.

    Var x.
    Bind x to a new empty box. If this form is used for a declaration, it must be possible to determine the species of x from other information, such as an assumption.

    Var x,y,z.
    Same as 
          Var x. 
      Var y. 
      Var z.

    Var x holding T.
    Same as Var x., but x has species <:T:>. This form can be used for expressions or declarations. In the case of declarations, species T must not be polymorphic.

    Var x,y,z holding T.
    Same as 
          Var x holding T. 
      Var y holding T. 
      Var z holding T.

    Var x := a.
    Bind x to a new box with initial content a.

    Var x holding T := a.
    Same as Var x := a, but x has species <:T:>.

    Var x,y,z holding T := a.
    Same as 
          Let newid = a.
      Var x holding T := newid. 
      Var y holding T := newid. 
      Var z holding T := newid.
    where newid is some new name.

    Var{shared} ...
    Forces a shared box to be chosen. (See box flavors.) This is the default.

    Var{nonshared} ...
    Forces a nonshared box to be chosen.

    Creating arrays using var expressions and declarations

    An array is a list of boxes, or a list of lists of boxes, etc. The following forms of var expressions and declarations can be used to create arrays.

    Var x(n) holding T %Var
    This binds x to an array of n new empty boxes. Each of those boxes has species <:T:>. Phrase "holding T" is optional for expressions, and only required for declarations if the species of the boxes cannot be determined by other means.

    Var x(*) holding T %Var
    This binds x to an infinite array of new empty boxes. The array is lazy; as much of it will be created as is used.

    Var x(n) holding T all := a %Var
    This binds x to an array of n new boxes, each with initial value a. You can use Var x(*) all := a %Var as well.

    Var x(n) holding T := L %Var
    This binds x to an array of n new boxes. The boxes are initialized from list L. The first box gets the first member of L, the second box gets the second member of L, etc.

    If L is shorter than x, then only a prefix of x is initialized. If L is longer than x, then only a prefix of L is used.

    This form can be used with infinite arrays as well as finite ones, and it is possible to initialize an infinite array from an infinite list.

    Var x(n,m) holding T %Var
    This form creates a two-dimensional array of n rows, each having m columns. If there is no initialization, the boxes are initially empty. If there is initialization, then the boxes are initialized as given.

    Any of the components can be infinitely long. For example, 

          Var a(*,*).
    creates a two dimensional array that is infinite in both dimensions.

    Any number of dimensions can be used. For example, 

          Var threedim(4,4,4) all := 0.
    creates a three dimensional array all of whose boxes initially hold 0.

    Flavors of boxes

    If you are unfamiliar with threads, then you can skip this part.

    There are two flavors of boxes, shared and nonshared.

    Shared boxes are shared by all threads. If one thread changes the content of a shared box, then that change is visible to all threads.

    Nonshared boxes are local to a single thread. When a thread forks into two threads, it also forks the nonshared boxes; each new thread gets a copy of all nonshared boxes. Any modifications to the content of a nonshared box made by one thread are invisible to the other thread.

    For example, 

      Try
    Var{shared} x := 2.
    Stream
      Assign x := 3.
    then
      ()
    %Stream
    Writeln[$x].
    {false}
  %Try
    prints 
      3
  3
    since the first thread changes the content of box x, and that change is visible to the second thread. However, 
      Try
    Var{nonshared} x := 2.
    Stream
      Assign x := 3.
    then
      ()
    %Stream
    Writeln[$x].
    {false}
  %Try
    prints 
      3
  2
    since the change done by the first thread to box x is invisible to the second thread.

    If a thread t gets access to a nonshared box that was created by another thread, then thread t sees that box as it was when the box was created. If the box was initially empty, then thread t sees it as empty. If the box was created with an initial value, then thread t sees it as holding that value, regardless of what another thread has done to the box.

    Species BoxFlavor has two members, shared and nonshared. You can get the flavor of a box using function flavor. That is, 

        flavor(b)
    is the flavor of box b.

    Creating boxes

    Var expressions and declarations are only one way to create boxes. You can also use expressions that produce, as their values, new boxes.

    The following functions produce boxes. All of them are standard.

    <: :>
    : <:`a:>

    <::> is a new, empty shared box.

    Example: Let b! = <::>.

    <:# #:>
    : <:`a:>

    <:# #:> is a new, empty shared box.

    Example: Let b! = <:# #:>.

    <:% %:>
    : <:`a:>

    <:% %:> is a new, empty nonshared box.

    Example: Let b! = <:% %:>.

    <:e:>
    : <:`a:>

    <:e:> is a new shared box whose initial content is the value of expression e.

    Example: Let b! = <:0:>.

    <:#e#:>
    : <:`a:>

    <:#e#:> is a new shared box whose initial content is the value of expression e.

    Example: Let b! = <:# 0 #:>.

    <:%e%:>
    : <:`a:>

    <:%e%:> is a new nonshared box whose initial content is the value of expression e.

    Example: Let b! = <:% 0 %:>.

    emptyBox
    : () -> <:`a:>

    emptyBox() is the same as <: :>.

    Example: Let b! = emptyBox().

    emptyNonsharedBox
    : () -> <:`a:>

    emptyNonsharedBox() is the same as <:% %:>.

    Example: Let b! = emptyNonsharedBox().

    emptySharedBox
    : () -> <:`a:>

    emptySharedBox() is the same as <:# #:>.

    Example: Let b! = emptySharedBox().

    box
    : BoxFlavor -> `a -> <:`a:>

    (box flav v) produces a new box holding initial content v. The box flavor is given by flav, which can be shared or nonshared.

    Example: Let b! = box shared 2.

    new
    : BoxFlavor -> <:`a:>

    new(flav) is a new empty box of flavor flav.

    nonsharedBox
    : `a -> <:`a:>

    nonsharedBox(v) is the same as <:%v%:>.

    Example: Let b! = nonsharedBox(2).

    sharedBox
    : `a -> <:`a:>

    sharedBox(v) is the same as <:#v#:>.

    Example: Let b! = sharedBox(2).

    Getting the content of a box, and checking for an empty box

    @
    : <:`a:> -> `a

    @(b) is the current content of box b. It fails with exception emptyBoxX if b is empty.

    Example: Let x = @b!.

    testAt
    : <:`a:> -> `a

    testAt(b) is the current content of box b. It fails with exception testX if b is empty.

    Example: Let x = testAt(b!).

    defaultAt
    : `a -> <:`a:> -> `a

    defaultAt d b is the current content of box b, if box b is nonempty, and is d if b is empty.

    Example: Let x = defaultAt 0 b!.

    optAt
    : <:`a:> -> Optional(`a)

    optAt(b) is value(x) if box b is nonempty and contains x, and is noValue if box b is empty. See optional for a discussion of value and noValue.

    Example: Let x = optAt(b!).

    emptyBox?
    : <:`a:> -> Boolean

    emptyBox?(b) is true just when box b is empty.

    Example: If emptyBox?(x!) then ...

    Modifying the content of a box

    Typically, you use Assign to change the content of a box. A few other operations are provided as well.

    Assign
    : (<:`a:>, `a) -> ()

    Assign b := v %Assign sets the content of box b to x. It also performs any actions that are associated with box b. See actions, below.

    Example: Assign b! := 4.

    Assign!
    : (<:`a:>, `a) -> ()

    Assign! b := v %Assign! sets the content of box b to x. It does not perform actions that are associated with box b. See actions, below.

    Example: Assign! b! := 4.

    AssignInit
    : (<:`a:>, `a) -> ()

    AssignInit b := v %AssignInit sets the initial content of box b to x. You generally do not use this directly. Use Var or <:i:>.

    Example: AssignInit b! := 4.

    EmptyBox
    : <:`a:> -> ()

    EmptyBox(b). makes b empty. It also performs actions associated with box b.

    Example: EmptyBox(x!).

    EmptyBox!
    : <:`a:> -> ()

    EmptyBox!(b). makes b empty, but does not perform any actions associated with the box.

    Example: EmptyBox!(x!).

    OptAssign
    : (<:`a:>, Optional(`a)) -> ()

    OptAssign b := v %OptAssign assigns b := x if v = value(x), and makes b empty if v is noValue. See optional for a discussion of value and noValue.

    Swap
    : (<:`a:>, <:`a:>) -> ()

    Swap(x,y) %Swap swaps the contents of boxes x and y. If either is empty, the other becomes empty.

    Example: Swap(b1!,b2!).

    AddTo
    : (<:REAL`a:>, REAL`a) -> ()

    Addto(b,x) %AddTo adds x to the content of box b.

    Example: AddTo (b!,2).

    Decrement
    : <:INTEGER`a:> -> ()

    Decrement(b) %Decrement subtracts 1 from the content of box b.

    Example: If box b! contains 5, and you do Decrement b! %Decrement, then b! holds 4 afterwards.

    Increment
    : <:INTEGER`a:> -> ()

    Increment(b) %Increment adds 1 to the content of box b.

    Example: If box b! contains 5, and you do Increment b! %Increment, then b! holds 6 afterwards.

    Pattern matching with boxes

    @!
    Pattern @! is used to put the result of a pattern match into a box rather than just giving the result a name. Statement 
        Match @!(x) = t %Match
    has the same effect as 
        Assign x := t %Assign

    Example: Match ( @!(x!), ?y ) = (2,3). puts 2 into box x!.

    <: :>, <:# #:>, <:% %:>
    Statement 
        Match <::> = t %Match
    succeeds only when t is an empty box. Pattern <::> matches any flavor of box. Patterns <:# #:> and <:% %:> are similar, but restrict the flavor of the box to shared and nonshared, respectively.

    Example: Match (<: :>, ?y) = (<: :>,2). succeeds.

    <:p:>, <:#p#:>, <:%p%:>
    Statement 
        Match <:p:> = t %Match
    succeeds only when t is a nonempty box, and it matches pattern p against the contents of that box. Pattern <:p:> will match any flavor of box. Patterns <:#p#:> and <:%p%:> are similar, but restrict the kind of box to shared and nonshared, respectively.

    Example: Match (<:?x:>, ?y) = (<:2:>,3). binds identifier x to 2.

    Miscellaneous operations on boxes

    $
    : <:`a:> -> String

    If b is a shared box, then $(b) is "shared n", where n is the number internally assigned to the box. If b is nonshared, then $(b) is "nonshared n". Be careful using this; the garbage collector reassigns internal box numbers, causing the result of $(b) to change. See FreezeBoxRanks, below.

    _eq_
    : (<:`a:>,<:`a:>) -> Boolean

    The equality operator == cannot be used with boxes. (This is because it is likely that there will be confusion as to whether x == y is asking whether x and y are the same box or whether their contents are the same.) To compare two boxes for equality, use operator _eq_. a _eq_ b is true if a and b are the same box.

    Example: If x! _eq_ y! then ...

    _neq_
    : (<:`a:>,<:`a:>) -> Boolean

    a _neq_ b is true if a and b are not the same box.

    Example: If x! _neq_ y! then ...

    rankBox
    : <:`a:> -> Natural

    rankBox(b) is a natural number that you can think of as the "address" of box b. Be careful: the rank of a box can change when the garbage collector runs. See FreezeBoxRanks.

    FreezeBoxRanks
    : Boolean -> ()

    FreezeBoxRanks(true). tells the system to suppress moving boxes to new locations, so that their ranks do not change. If you use this for a long interval, you cripple the garbage collector, and cause memory to become fragmented. FreezeBoxRanks(false). allows the garbage collector to move boxes.

    Actions associated with boxes

    OnAssign
    : (<:`a:>, (Optional(`a), Optional(`a)) -> ()) -> ()

    OnAssign attaches an action to a box, so that the action will be performed when the box has its contents changed. Statement 

        OnAssign(b,f) %OnAssign
    attaches action f to box b.

    Specifically, suppose that b currently contains value u, and Assign b := v %Assign is evaluated. Then f(value(u), value(v)) is evaluated after the assignment was done. If b was empty before the assignment, then f(noValue, value(v)) is evaluated. See optional for a discussion of value and noValue.

    When EmptyBox(b) %EmptyBox is done, if b is not already empty (and contains u), then f(value(u), noValue) is evaluated after making box b empty. If b was already empty, the action is not done.

    You can attach more than one action to a box. Each call to onAssign adds another function. The functions are evaluated in the order in which they were added.

    For example, suppose you write 

         Let zorkAction by

       case zorkAction(value ?a, value ?b) = 
         Writeln["I am changing zork! from ", $a, " to ", $b].

       case zorkAction(noValue, value ?b) =
         Writeln["I am changing zork! from empty to ", $b].

       case zorkAction(value ?a, noValue) =
         Writeln["I am emptying zork!"].

       else =>
	 Writeln["This should not happen"].
     %Let

     Var zork! holding Natural.
     OnAssign(zork!, zorkAction).
    Then if you do Assign zork! := 5., then expression zorkAction(noValue,value(5)) will be evaluated implicitly.

    Action functions must return () as their values.

    Polymorphism and boxes

    You will need to read about polymorphism to understand this section.

    Expression <::> returns a new, empty box of any desired species. So, if you declare 

          Let x = <::>.
    the declaration of x is polymorphic. Since polymorphism is just overloading in Astarte, you get infinitely many different boxes, one for each species. This can be confusing. If you store a value into one of those boxes named x, you do not alter another one. For example, doing 
      Assign x: <:Natural:> := 2.
  Assign x: <:Integer:> := 8.
  Writeln[$ @(x:<:Natural:>)].
  Writeln[$ @(x:<:Integer:>)].
    prints 
      2
  8
    since the two boxes called x (one of species <:Natural:>, the other of species <:Integer:>) are different boxes.

    To avoid this confusion, use var declarations. They do not allow polymorphism, to suppress this possible source of confusion.
