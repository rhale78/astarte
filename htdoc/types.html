    • Astarte is a typed language with a polymorphic type system. This page discusses a few important aspects of the type system, and provides links to other pages that give details.

    Species

    A specific type is called a species. Each data item or value has exactly one species. For example, the value true has species Boolean. We will occasionally use the word type for a species.

    Numeric species include Natural, Integer, Rational and Real. Since each value has just one species, there must be a separate value 0 for each of these species. 0:Natural is not the same as 0:Integer.

    Some symbols have more than one meaning, just as some words have more than one meaning. For example, the constant 0 can stand for 0:Natural, 0:Integer, 0:Rational or 0:Real. So it names four different values.

    Simple species

    A species is a set of items, or values. For example, species Natural contains values 0, 1, 2, ... There is a small collection of basic species for you to use, including numeric species, species Boolean for truth values, and a few others. The simple species that are provided are described in
  • T2. Standard species.
    • You can also create your own species. See
  • T15. Creating your own species.

    • Parameterized species

    For any species T you can form another species List(T) that consists of lists whose members have species T. For example, [true,false,false] is an item that has species List(Boolean). List is an example of a parameterized species, also called a family. Topics on parameterized species are
  • T3. Standard parameterized species,
  • T16. Creating your own parameterized species.

    • Complex Species

    You can build up species from simpler species. For example, given any two species A and B, you can form the species (A,B) consisting of ordered pairs whose left-hand member has species A and whose right-hand member has species B. Ordered pair (true,"frog") has species (Boolean, String).

    You can also create species of functions. For example, Boolean -> Boolean is a the species of a function that takes a parameter of species Boolean and produces a result of species Boolean. Ways of constructing complex species are discussed in

  • T4. Complex species.

    • Polymorphism

    Consider the function dup(x) = (x,x). This function should, ideally, be able to work on any species of item x. For example, dup(true) = (true,true) (where dup has species Boolean -> (Boolean,Boolean)) and dup("frog") = ("frog","frog") (where dup has species String -> (String,String)). Function dup is polymorphic. That is, it works on many different types of data.

    In a typed programming language, each function must have a type. What type should function dup have? If we decide that function dup takes a parameter of species Boolean, then we preclude using it on a string.

    The solution is to introduce type variables. A type variable stands for a unknown or arbitrary type. You can write a type variable in Astarte as a grave accent (`) followed by a name for the variable. Often, short variable names such as `a and `b are used. The polymorphic type of function dup is `a -> (`a,`a). That is, it is a function that takes a parameter of an arbitrary type T and produces a result of type (T,T).

    Generally, we use the word type for a polymorphic species, and species when there is no polymorphism. Polymorphism is discussed further in

  • T5. Polymorphism.

    • Type inference

    The compiler performs type inference. This means that you can write programs without including any type information at all, and you can begin programming without understanding the type system. The compiler can infer the information that you leave out.

    If you get a type error, the compiler will show you the types that mismatched. Unless you are familiar with the type system, this will not make much sense to you. If you do not understand the types, just look carefully at what you wrote; something is wrong with it. But see

  • T10. Understanding type errors
    • for hints on how to understand the error messages.

    Type inference does not always find the error at the line where it occurs. Sometimes, a mistake that you have made is consistent up to a point, and it is only later that the inconsistency becomes evident. You might want to put in some

  • E20. Assumptions or tags
    • to get a more accurate error report.

    For more details on type inference, see

  • T9. Type inference.
  • T24. Defaults

    • Data Abstractions

    A data abstraction can be thought of as a set of species. For example, there is a standard abstraction RRING that contains the species Integer, Rational and Real. We will also use the shorter term genus for a data abstraction.

    Data abstractions are used to limit the values that type variables can take on. If you write RRING`a, then you have a variable that can only be one of the species in RRING.

    The subtraction function (-), for example, is defined to be polymorphic, but to work only on species in RRING. The subtraction function itself has type (RRING`a,RRING`a) -> RRING`a. That is, it takes a pair of T's, where T must be one of the species in RRING, and produces a result of species T.

    Secondary species

    Astarte allows you to put off some type checking until run-time. This is done by having an item carry around its species with it, so that the run-time system can examine the species and determine what to do. This feature is discussed under
  • T20. Secondary species.

    • Irregular functions

    In order to use secondary species effectively, it must be possible to extract the intrinsic species (the species that an item carries with it) of an item. Irregular functions do that job. Read about
  • T21. Irregular functions
    • for how to create and use them.

    Constraints

    We have already seen that variables can be constrained as to which species they can take on. For example, variable RRING`a can only stand for a species in data abstraction RRING.

    Astarte provided another form of constraint on type variables, allowing you to relate the values of different variables to one another. This mechanism is used with secondary species, and is described in

  • T22. Constraints.

    Some standard operations have constraints in their polymorphic types. You can find out about them at
