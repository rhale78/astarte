    • Basic function application

    A function is a computational engine that takes one parameter and produces one result. In some cases, it also has side-effects; it can change things. Most functions, however, have no side effects. They only produce an answer.

    When you use, or run, a function, you apply it. The basic syntactic form for function application is juxtaposition. To apply a function to a value, you can just write the value after the function. That is, you juxtapose the function and its argument. For example, sqrt 4 is an expression that applies function sqrt to value 4. Its value is 2. Both the function and the argument can be arbitrarily complex expressions. For example, sqrt(2+2) is also the square root of 4.

    Associativity of juxtaposition

    Juxtaposition associates to the left. So 

      f a b
    is understood to mean 
      (f a) b
    You need to be careful about this. If your write 
      sqrt sqrt 4
    this is read as 
      (sqrt sqrt) 4
    which is asking to take the square root of the square root function. That makes no sense; you take the square root of a number, not of a function. So this results in a type error. Use parentheses to help. Expression 
      sqrt(sqrt 4)
    yields the square root of the square root of 4.

    Juxtaposition has higher precedence than binary operators. So sqrt 2 + 2 is understood to mean (sqrt 2) + 2. Use parentheses to get what you want.

    Precedence of binary operators

    Precedence and associativity of binary operators are as follows, from high to low.

    Operators(s) Associativity Precedence
    _opL9_ left high
    _opR9_, _o_ right
    _opL8_, #, ##, _as_, _eagerCutBy_ left
    _opR8_, ^ right
    _opL7_, *, /, //, _div_, _mod_ left
    _opR7_ right
    _opL6_, +, -, --, _monus_ left
    _opR6_ right
    _opL5_, $$, _upto_, _downto_, _leftin_ left
    _opR5_, :: right
    _opL4_, +/ left
    _opR4_, ++ right
    <, >, <=, >=, ==, =/=, _eq_, _neq_, _is_ none
    _opL3_ left
    _opR3_, _and_, _andf_ right
    _opL2_ left
    _opR2_, _or_, _orf_ right
    _opL1_, _where_, _also_, _orelse_, _orthen_, _orwith_ left
    _opR1_, ==>, _implies_ right
    _bar_, ||, | left
    _butIfFail_ left
    := right low

    Note: Symbols :=, |, _and_, _implies_ and _or_ are reserved. You cannot redefine their meanings or their precedence.

    Order of evaluation

    Although functions are often just given by names, they can be arbitrarily complex expressions. When a function is given by an expression, that expression must be evaluated to obtain the function before the function can be applied.

    The order of evaluation of the parts of expression (f a) or (f a b) is not specified. The compiler can choose to evaluate the individual parts from left to right or from right to left, or in any other order. Here are two possible ways to compute expression (f a).

    1. First evaluate f, then evaluate a, then apply function f to value a.

    2. First evaluate a, then evaluate f, then apply function f to value a.
    Because the order of evaluation is undefined, any identifiers that are defined in one of the parts are not visible in the other parts, since they might not have been computed yet. So the following is not allowed, since the name x is defined inside the function expression, and used in the argument. 
        (Let x = 1.5. Value sqrt.)(x)
    This problem can show up with binary operators. If you write 
        Let x = 1.5.
    x + x
    you might expect to have an expression whose value is 3.0. But juxtaposition has higher precedence than binary operators, so according to the rules of precedence, this is equivalent to (Let x = 1.5. x) + x. The second x is not allowed, since it might be evaluated before x is defined. To avoid this problem, either use parentheses or Value ... %Value. For example, write 
        Let x = 1.5.
    Value x + x.

    Other syntactic forms

    For convenience, there are some other syntactic forms that can be used for applying functions. They are nothing but conveniences of notation; the effect is exactly the same as what is achieved using juxtaposition.

    x | f
    x | f has the same meaning as f(x). Bar has lower precedence than juxtaposition, and associates to the left. You typically use it for doing a sequence of function applications. For example, (16 | sqrt | sqrt | -) produces -2, the same result as -(sqrt(sqrt 16)).

    P a %P
    A function name that starts with an upper case letter is called a procedure. When you apply a procedure, you must include a matching end-word or period. For example, standard procedure Writeln prints a list of strings, followed by a newline. The following two forms are equivalent. 
      Writeln ["Test"] %Writeln
  Writeln ["Test"].

    To use a procedure as an ordinary identifier, without special handling, enclose it in parentheses. See suppressing special handling for details on on treating procedure identifiers as ordinary identifiers.

    You can write P a,b,c %P. This stands for ((P) (a) (b) (c)). You can have any number of parts. Note the absence of parentheses. P (a,b,c) %P is not the same; it applies P to a single parameter that is an ordered triple, and is equivalent to ((P)(a,b,c)). For example, 

      FWriteln myoutfile, ["Test"] %FWriteln
    is equivalent to (FWriteln) myoutfile ["Test"].

    You can write P %P as an abbreviation for P() %P. For example, the following two lines are equivalent. 

      Randomize. 
  Randomize().

    You can use semicolons with procedures. Each semicolon starts another application of the same procedure. For example, Assign is a standard library procedure. Statement 

      Assign
    x := 1;
    y := 2
  %Assign
    is equivalent to statement 
      Assign x := 1.
  Assign y := 2.

    Ask x => f %Ask
    This notation is used to apply a function f to x, but to think of x as the agent that is carrying out the action performed by f. If f is not a function application, then Ask x => f %Ask stands for ((f)(x)). In general, Ask x => e %Ask places x just after the function that is being applied in e. Here are some example translations.

    Expression Stands for
    Ask x => f. f x
    Ask x => f y. f x y
    Ask x => f y z.   f x y z

    As you can see, the value being asked to perform an action is placed immediately after the action, and before any other parameters.

    x _is_ f
    x _is_ f is the same as f(x), but it requires that f be a predicate. That is, f must be a function whose return-type is Boolean. (See lang/boolean.ast) For example, n _is_ prime? is the same as prime?(n).

    op x
    Some identifiers are unary operators. They are just functions, but have different precedence and associativity rules than usual functions. Application of unary operators associates to the right, and has lower precedence than application of ordinary functions. For example, $ and - are unary operators. If you write $ f x, the compiler implicitly parenthesizes this as $ (f x). Similarly, -sqrt(x) means -(sqrt(x)).

    The standard unary operators are 

       $  +   -   /  //  ~  ^^  @  not
    Unary operators are discussed in more detail in lang/operator.ast. See operator for how to create a new unary operator.

    To use a unary operator without special syntactic processing, enclose it in parentheses. For example, ($) is the function whose name is $. See suppressing special handling for details on on treating unary operators as ordinary identifiers.

    a op b
    Some identifiers are binary operators. If op is a binary operator, then a op b abbreviates (op)(a,b). For example, +, -, * and / are binary operators. 2 + 3 abbreviates (+)(2,3).

    When a word is used as a binary operator, it is conventional (but not required) to start and end that word with an underscore. So _plus_ might be used as a binary operator.

    The precedence and associativity of standard operators is described above.

    To use a binary operator without special syntactic processing, enclose it in parentheses. For example, (+) is the addition function. See suppressing special handling for details on on treating binary operators as ordinary identifiers.

    Example: x*y means (*)(x,y)

    Creating new operators

    You can create a new unary operator called myop by giving declaration 
          Operator myop.
    Declaration 
          Operator newbinop(oldbinop).
    defines newbinop to be a binary operator with the same precedence and associativity as binary operator oldbinop. For example, to create a binary operator _plus_ with the same precedence and associativity as +, use 
          Operator _plus_(+).
    There are nine precedence levels, and two associativity choices for each level. Each level has a special operator name that describes it. Operator _opL1_ is at level 1, and associates to the left. Operator _opR1_ is at level 1 and associates to the right. This goes through _opL9_ and _opR9_. Level 9 has the highest precedence, level 1 the lowest precedence. In all levels, the L operator has higher precedence than the R operator.

    All operators have a precedence and associativity given by one of the eighteen possible values. For example, + has the same precedence and associativity as _opL6_. See above for a list of standard operators and their precedences and associativities.

    When is special meaning suppressed?

    You can suppress the special meaning of a procedure, unary operator or binary operator by enclosing it in parentheses. In general, special meaning is suppressed whenever the token that follows the operator or procedure is one of the following. 
      ) ] } ; , | = by
    Additionally, a binary or unary operator (but not a procedure) has special handling suppressed if it is followed by a period or end marker such as %Let.

    Tokens that follow a name are only recognized if there are not too many characters between tokens. If you put a large number of characters between a binary operator symbol and the token that follows it, then the symbol will be treated as a binary operator. The special case of a right parenthesis is always detected, regardless of how many characters intervene.
