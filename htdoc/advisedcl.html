D17. Advise Declarations

  • Advise declarations
  • Advisories
  • Examples

    • Advise declarations

    Advise declarations are used to advise the compiler on how to compile a package or declaration. An advise declaration has the form 
         Advise advisory %Advise
    Different kinds of advisories are listed below. After each, you can write one or all. If you write one, then the advisory holds only for the next declaration (not counting other intervening advisories). If you write all or nothing, then the advisory applies to all subsequent declarations, until it is replaced by another advisory.

    Advisories

    Available advisories are as follows.

    "string"
    If a listing is not being done at this point, then print string "string" into the error-message/listing file.

    overloads k
    Allow let/define declarations to do up to k different definitions.

    noshow
    Do not show results of type inference.

    notro
    Do not use tail-recursion improvement at all.

    Tail recursion improvement allows a tail recursive call to avoid creating a new stack frame. Instead, it just overwrites the stack frame of its caller. This can save a lot of memory, but it can also be confusing when you are debugging a program.

    notro f,g,h
    Do not use tail-recursion improvement when compiling calls to functions f, g and h. You can list as many or few functions as you like. The functions are given only by name.

    show
    Show the results of type inference.

    tro
    Use tail-recursion improvement, except for functions that are explicitly noted not to use tail recursion improvement via notro f,g,h.

    warn
    nowarn
    Turn on (off) all warnings.

    warn a,b,c
    nowarn a,b,c
    Turn on (off) selected warnings. You can list any number of warnings to turn on or off, separated by commas. The warnings are listed in the following cases.

    warn defaults
    nowarn defaults
    Warn (do not warn) about dangerous defaults that are done during type inference.

    warn indent
    nowarn indent
    Warn (do not warn) about suspicious indentations.

    warn missingDefinition
    nowarn missingDefinition
    Warn (do not warn) about missing definitions.

    warn notExhaustive
    nowarn notExaustive
    Warn (do not warn) about collections of patterns that are not exhaustive at pattern-matching choices.

    warn unused
    nowarn unused
    Warn (do not warn) about unused identifiers. Note: Any identifier that begins or ends with "unused" (or any case-variant of that, such as "Unused") will not be reported as unused at any time, regardless of whether this warning is turned on or off.

    Examples 

         Advise notro.
     Advise nowarn indent.
     Advise "Compiling package gopher.ast".
