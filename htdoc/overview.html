M3. Overview

  • Reading this documentation

    • Reading this documentation

    A simple program has the form 
       Package name
     declaration
     ...
     declaration
   %Package
    where name is the name of the package and declaration is a declaration. See this example package.

    To learn how to use Astarte from this manual, either follow the next links or select a topic from this page or the main page.

    In exploring this documentation, you will inevitably come across features that have not yet been described. Don't let that bother you. Look at the parts that you understand.

    Topics

    M4. General lexical and syntactic issues
    This page discusses what the syntax of the language looks like. You should read it before reading the other pages.

    M5. Notes for the new user
    This page gives some hints to new users that will help you stay out of traps. It is a good idea to read it.

    E1. Expressions
    Expressions compute values. There are also kinds of expressions whose value is void. They are called statements. Statements are a special kind of expression. Use this section to find out how to write expressions.

    T1. Type System
    The type system is polymorphic. You can write some programs without knowing anything about the type system, since the compiler will insert type information for you if you leave it out. This part of the manual describes how to use the type system when you need to.

    L1. Library
    The library section describes the primitive functions and types, as well as those that are provided in library packages. Even basic operators, such as +, are considered part of the library.

    D1. Declarations and packages
    Each Astarte file contains a package. This section describes the structure of a package, and how to link different packages together.

    At the top level, a package consists of a sequence of declarations. Declarations are used to do things such as

    Create new types
    Create functions
    Cause expressions to be evaluated
    Import other packages
