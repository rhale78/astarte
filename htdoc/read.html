LC6. Reading from Strings

  • Pattern matching with boxes
  • Pattern matching without boxes
  • Functions that involve boxes
  • Functions that do not involve boxes: Genera operations
    • Some operations are provided for extracting prefixes of strings and converting them to various species. You can use them directly as functions, or you can use pattern matching instead. Pattern matching is usually more pleasant.

    Pattern matching with boxes

    Pattern matching in strings is conveniently done using Extract, which works on arbitrary lists. Other forms for general lists can also be used with strings.

    The following pattern forms are available for strings. They are described in collect/string.ast and pattern/listpat.ast. Most of these require imports.

    Extract separated [...]
    Statement 
         Extract separated[p1,p2,...,pn] from b.
    is similar to 
         Extract[p1,p2,...,pn] from b.
    but it allows arbitrary white space between pattern targets, which Extract alone does not. Import collect/string.ast.

    Extract [$(p)]
    Statement 
         Extract [$(p)] from b.
    extracts a prefix that is an appropriate form according to the species of p, and matches pattern p against that prefix. For example, if box b contains string "453 9345" and n has species Natural, then 
          Extract[$(?n)] from b.
    binds n to 453, and leaves box b holding " 9345". Import collect/string.ast to use $ in patterns.

    Pattern $(?n) uses function pull to do the extraction. See pull below for details on what can be read.

    Extract [hex(p)]
    Statement 
         Extract [hex(p)] from b.
    extracts a prefix of @b that is a number in hexadecimal notation, and matches p against the number.

    Import collect/string.ast to use hex.

    Extract [theLine(p)]
    Statement 
         Extract[theLine(p)] from b.
    extracts one line from the front of the content of b, and matches pattern p against that line. The newline character is not part of the line. For example, if box b contains "abc\ndef", then 
         Extract[theLine(?x)] from b.
    binds x to "abc" and leaves b holding "def". Import collect/string.ast to use theLine.

    Extract [theString(p)]
    Statement 
         Extract[theString(p)] from b.
    first skips over white space, then extracts a maximal prefix that does not contain any white space characters. For example, if box b holds " abc def", then 
        Extract [theString(?s)] from b.
    binds s to "abc" and leaves b holding " def".

    If the first nonwhite character of the target is a double quote character, then this pattern gets an entire quoted string. For example, if box b holds " \" ab c\" def", then 

        Extract [theString(?s)] from b.
    binds s to " ab c" and leaves b holding " def".

    Import collect/string.ast to use theString.

    Extract [theId(p)]
    Statement 
         Extract[theId(p)] from b.
    first skips over white space, then extracts a maximal prefix that is an Astarte identifier. It gets the value of that identifier in the outer environment (of the same species as p) and matches pattern p against that value.

    Import collect/string.ast to used theId.

    Extract[white]
    Statement 
         Extract[white] from b.
    extracts all white space characters from the front of the content of b, and does nothing with them. Import collect/string.ast to use white.

    Extract [theWS(p)]
    Statement 
         Extract[theWS(p)] from b.
    extracts all white space characters from the front of the content of b, and matches pattern p against them. Import collect/string.ast to use theWS.

    Extract [theBalanced (lp,rp) p]
    Statement 
         Extract[theBalanced (lp,rp) p] from b.
    extracts a prefix of the target that is balanced with respect to left parenthesis lp and right parenthesis rp. For example, 
         Extract[theBalanced ('(',')') (?s)] from b.
    extracts a prefix, such as "(x(12)(14))", that is balanced with respect to parentheses.

    Pattern matching without boxes

    The pattern forms that can be used with Extract can also be used in matches using ++ to concatenate the pieces. For example, 
       Match $(?n:Natural) ++ theString(?s) ++ ?r = "42 kangaroo mice".
    succeeds, binding n = 42, s = "kangaroo" and r = " mice". You can also use function flatten in patterns, where flatten[a,b,c] has the same effect as a ++ b ++ c. So the preceding match can also be written 
       Match flatten[$(?n:Natural), theString(?s), ?r] = "42 kangaroo mice".
    Be careful: when you are not using Extract, an exact match is required. For example, match 
       Match $(?n:Natural) ++ theString(?s) = "42 kangaroo mice".
    fails, since the pattern cannot match the entire target string.

    You can use a variable to match what you want to leave alone. For example, 

       Match ?a ++ "kangaroo" ++ ?b = "42 kangaroo mice".
    binds a = "42 " and b = " mice".

    Functions that involve boxes

    The following functions are used to extract a prefix of a string in a box, leaving the rest in the box. They offer an alternative to pattern matching. See collect/string.ast for more detail.

    read
    : <:String:> -> `a

    Suppose that b is a box. Then read(b) extracts a prefix of the content of b, converts it to an appropriate species, and returns the result. It also binds box b to the remainder of its former content, just after the prefix that was extracted. The species that is returned depends on context. For example, if box b contains "244 a934", 

          read(b):Natural
    returns 244 and puts " a934" in box b.

    Read uses function pull to do its work. See pull.

    read
    : () -> `a

    read() is equivalent to read(stdin!). Import collect/string.ast to use read.

    readLine
    : <:String:> -> String

    readLine(b) extracts and returns a line from the front of the content of box b, and binds b to the remaining lines. The newline character is not in the returned string. Import collect/string.ast to use readLine.

    readLine
    : () -> String

    readLine() is equivalent to readLine(stdin!). Import collect/string.ast to use readLine.

    readString
    : <:String:> -> String

    readString(b) extracts and returns a string from the front of content of box b, and binds b to the remainder of the string. The string is obtained by skipping leading white space, then reading until the end of the string or white space is encountered. Import collect/string.ast to use readString.

    readString
    : () -> String

    readString() is equivalent to readString(stdin!). Import collect/string.ast to use readString.

    readUpToN
    : <:String:> -> Natural ->[`a]

    readUpToN b n extracts and returns the first n characters from the front of content of box b, and binds b to the remainder of the string. If there are fewer than n members in the list, then readUpToN reads them all. Import collect/string.ast to use readUpToN.

    readUpToN
    : () -> Natural -> String

    readUpToN() is equivalent to readUpToN(stdin!). Import collect/string.ast to use readUpToN.

    Functions that do not involve boxes: General operations

    The following functions are available for manipulating strings. See collect/string.ast for more detail.

    ++>
    : (String,String) -> String

    a ++> b = a ++ " " ++ b.

    Example: $(23) ++> $(44) = "23 44". If you use ++> in a pattern, it works in a way similar to ++, but it allows white space between the parts. For example, 

       Match "abc" ++> "def" = "abc  def"
    succeeds. Import collect/string.ast to use ++>.

    blank?
    : String -> Boolean

    blank?(s) is true if s contains only white space. Import collect/string.ast to use blank?

    breakIntoLines
    : String -> [String]

    breakIntoLines(s) is a list of the lines in string s.

    Example: breakIntoLines("abc\ndef\nghi\n") = ["abc", "def", "ghi"]. Import collect/string.ast to use breakIntoLines.

    expandTabs
    : String -> String

    expandTabs n s is the result of expanding tabs in string s, assuming that there is a tab stop every n characters. Import collect/string.ast to use expandTabs.

    filterComments
    : String -> String

    filterComments(s) is the result of removing all Astarte-style comments from string s. Only %% style comments are recognized, not <<<<<...>>>> style comments. Function filterComments is quasi-lazy, Import collect/string.ast to use filterComments.

    indentLines
    : String -> String

    indentLines n s is the result of inserting n blanks at the front of each line in string s. Import collect/string.ast to use indentLines.

    internString
    : String -> String

    internString(s) is the same string as s, but it refers to a copy of s that is stored in a hash table. Only one copy of any given string is stored in this hash table. Equality comparisons between two strings, both of which have been interned, is done by comparing pointers.

    Once a string has been put into the hash table, it is never removed. So only intern a string if you like the idea of keeping that string around.

    pull
    : String -> (`a,String)

    pull(s) is a pair (w,r) where w is obtained by extracting a prefix of s and converting it to an appropriate species, and r is the rest of s. If s is empty or has only white space, then pull(s) fails with exception listExhaustedX.

    The species of w, and the form of the string that determines w, depends on the species of pull. Standard implementations are as follows. All of these skip over any leading white space in the string before looking for the desired substring.

    Char
    The character must be in the form of an Astarte character constant, such as 'a' or '\n'. To pull just the first character of a string, use listToPair. See characters.

    Numeric
    Numbers should be in the form of Astarte numeric constants, possibly including an initial minus sign. See LN1, LN2, LN3, LN4.

    Lists
    A list is read in the form of a list constant, such as [2,3,4]. See lists.

    Strings
    A string must be in the form of an Astarte string constant, in double quotes. To read a string delimited by spaces instead of quotes, use pullString (or pattern theString).

    Ordered pairs
    An ordered pair, triple, etc. is written surrounded by parentheses, using commas as separators. For example, you can read (2,3,4) when pull has species String -> ((Natural, Integer, Natural), String). See ordered pairs.

    Default
    The default for pull is to read an identifier, and to return the value of that identifier. This, For example, can be used to read a function that has a name. Only identifiers that are defined in the outer scope can be used.

    pullId
    : String -> (`a,String)

    pullId(s) is a pair (w,r) where w is obtained by extracting an identifier from the front of string s and then looking up the value of that identifier, and r is the rest of s. The type of identifier w is determined by the codomain of pullId.

    pullLine
    : String -> (String,String)

    pullLine(s) is a pair (w,r) where w is the first line of string s, and r is the rest of s. So, if s contains a newline character, then s = w ++ "\n" ++ r. Import collect/string.ast to use pullLine.

    pullString
    : String -> (String,String)

    pullString(s) is a pair (w,r) where w is obtained by skipping any leading white space in string s, and then taking all characters up to white space, and r is the rest of s. So s = x ++ w ++ r, where x consists of leading white space.

    For example, 

      pullString(" abc def") = ("abc", " def")

    In the special case where the first nonwhite character of s is a double quote character, pullString removes an entire quoted string. For example, 

      pullString(" \"a horse and carriage\" and rest") = ("a horse and carriage", " and rest")

    If there is only white space in s, then pullString(s) fails with exception listExhaustedX.

    Import collect/string.ast to use pullString.

    pullHex
    : String -> (Natural,String)

    pullHex(s) skips leading white space in string s and then extracts a hexadecimal number from s. It returns pair (n,r) where n is the number extracted and r is the rest of s, after the number. Import collect/string.ast to use pullHex.

    pullWS
    : String -> (String,String)

    pullWS(s) is a pair (w,r) where w is the leading white space of string s and r is the rest. So s = w ++ r. Import collect/string.ast to use pullWS.

    skipWS
    : String -> String

    skipWS(s) is the result of removing all leading white space from string s.

    pullBalanced
    : (Char,Char) -> String -> (String,String)

    pullBalanced(lp,rp)(s) is a pair (b,r) where b is a prefix of s that is balanced with respect to left parenthesis lp and right parenthesis rp, and r is what follows b in s. Import collect/string.ast to use pullBalanced.

    toLower
    : String -> String

    toLower(s) is the result of converting all letters in s to lower case. Import collect/string.ast to use toLower.

    toUpper
    : String -> String

    toUpper(s) is the result of converting all letters in s to upper case. Import collect/string.ast to use toUpper.

    wrapLines
    : String -> String

    wrapLines n s is the result of inserting line breaks into string s so that no line has more than n characters in it. Import collect/string.ast to use wrapLines.

    wsExplode
    : String -> [String]

    wsExplode(x) is a list of the white-space separated substrings of string x.

    Example: wsExplode(" abc de fgh") = ["abc", "de", "fgh"].

    Parts that are surrounded by quotes are kept together, and the quotes are removed.

    Example: wsExplode(" \"abc de\" fgh") = ["abc de", "fgh"].

    Import collect/string.ast to use wsExplode.

    wsFlatten
    wsFlatten(x) is t: [String] -> String

    he concatenation of the members of list x, but with a space between consecutive members.

    Example: wsFlatten["abc", "def", "gh"] = "abc def gh". Import collect/string.ast to use wsFlatten.
