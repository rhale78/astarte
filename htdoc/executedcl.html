D3. Execute Declarations

  • Execute declarations
  • When are executes done?
  • Executes, failure and threads

    • Execute Declarations

    Declaration 
          Execute 
        s
      %Execute
    is used to ask for statement s to be evaluated. For example 
          Execute
        Let result = computeMyResult().
        Writeln["I got result ", $result].
      %Execute
    might be used to force something to be evaluated. There can be any number of executes in a program. They are done in the order in which they are encountered.

    When are executes done?

    The executes are all done after all definitions have been done. So if you write 
          Let{default} message = "first message".

      Execute
         Writeln[message].
      %Execute

      Let{override} message = "second message".
    what is printed is 
          second message

    Executes, failure and threads

    An execute is considered to have failed if no thread completes it successfully. If an execute fails, then computation is aborted.

    An execute is considered finished when all of its threads have finished. But at the end of the execute, all of the threads are joined together, and only one thread starts the next execute.

    If several threads complete the execute declaration successfully, then the contents of nonshared boxes are determined by their values in the first thread that finished. The nonshared boxes of all other threads are discarded.

    Typically, you want only one thread to finish an execute. You can do that by using a unique expression, as in 

        Execute
      Unique
        ...
      %Unique
    %Execute
