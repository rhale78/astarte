L6. Collections

The collection library allows you to manage lists, sets and tables. Package collect/collect.ast describes features of all collections. Specific kinds of collections and operations on them are described in the following sections.

    • Additionally, there is a parameterized abstraction COLLECTION, containing various kinds of collections, including lists and sets. The following operations are defined for all collections.

    allSatisfy
    : (`a -> Boolean) -> COLLECTION`c(`a) -> Boolean

    allSatisfy f c true if f(x) is true for every member x of collection c.

    bpartition
    : (`a -> Boolean) -> COLLECTION`c(`a) -> (COLLECTION`c(`a), COLLECTION`c(`a))

    bpartition f c is a pair (c1,c2) where c1 contains the members x of c that make f(x) true, and c2 contains the members x of c that make f(x) false.

    each
    : COLLECTION`c(`a) -> `a

    each(c) produces a stream of the members of collection c. For example, each[1,2,3] has the same effect as Stream 1 then 2 then 3 %Stream.

    eachMixed
    : COLLECTION`c(`a) -> `a

    eachMixed(c) produces a mixed (parallel) stream of the members of collection c. For example, eachMixed[1,2,3] has the same effect as Mix 1 then 2 then 3 %Mix.

    eagerToList
    : COLLECTION`c(`a) -> [`a]

    eagerToList(c) is a list of the members of collection c. This function is eager.

    emptyCollection
    : COLLECTION`c(`a)

    emptyCollection is an empty collection.

    emptyCollection?
    : COLLECTION`c(`a) -> Boolean

    emptyCollection?(c) is true if c is an empty collection.

    filter
    : (`a -> Boolean) -> COLLECTION`c(`a) -> COLLECTION`c(`a)

    filter f c is the collection of all members of x collection c such that f(x) is true.

    fold
    : (`a, (`a,`a) -> `a) -> COLLECTION`c(`a) -> `a

    fold (init,op) c is the same as fold(init,op)(toList c). See fold for lists.

    fromList
    : [`a] -> COLLECTION`c(`a)

    toList(x) is a collection (of a given kind) containing the members of list x.

    _in_
    : (EQ`a, COLLECTION`c(EQ`a)) -> Boolean

    x _in_ c is true if x is a member of collection c.

    map
    : (`a -> `b) -> (COLLECTION`c(`a) -> COLLECTION`c(`b))

    map f c is the collection that results from collection c by replacing each member x by f(x). For example, map (?x => x+x) [1,2,3] = [2,4,6].

    MapProc
    : (`a -> ()) -> COLLECTION`c(`a) -> ()

    MapProc f, c. applies function f to each member of collection c.

    mapSuccesses
    : (`a -> `b) -> (COLLECTION`c(`a) -> COLLECTION`c(`b))

    mapSuccesses f c is the same as map f c, but it skips over any members x where f(x) fails, keeping only those where f(x) succeeds.

    max
    : COLLECTION`c(ORDER`a) -> ORDER`a

    max(c) is the largest member of collection c. It fails with exception emptyListX on an empty collection.

    min
    : COLLECTION`c(ORDER`a) -> ORDER`a

    min(c) is the smallest member of collection c. It fails with exception emptyListX on an empty collection.

    partition
    : [`a -> Boolean] -> COLLECTION`c(`a) -> [COLLECTION`c(`a)]

    partition [f1, f2, ..., fn] c is a list [p1, p2, ..., pn+1] where each pi is a collection, and pi contains the members x of c that make f1(x), ..., fi-1(x) all false, but makes fi(x) true. The last collection, pn+1, consists of those members of c on which none of the functions f1, ..., fn returns true.

    select
    : (`a -> Boolean) -> COLLECTION`c(`a) -> `a

    select f c some member x of collection c such that f(x) is true. If no such member exists, select f c fails with exception testX.

    size
    : COLLECTION`c(`a) -> Natural

    size(c) is the number of members in collection c.

    sizeAtLeast
    : Natural -> COLLECTION`c(`a) -> Boolean

    sizeAtLeast n c is true if the size of c is at least n.

    someSatisfy
    : (`a -> Boolean) -> COLLECTION`c(`a) -> Boolean

    someSatisfy f c true if there exists a member x of collection c such that f(x) is true.

    toList
    : COLLECTION`c(`a) -> [`a]

    toList(c) is a list of the members of collection c. This function might or might not be lazy.
