LC10. Arrays and Structures Involving Boxes

  • Copying an array
  • Putting values in an array
  • Getting the nonempty boxes
  • Showing an array
    • An array is a list of boxes, or a list of lists of boxes (for a two-dimensional array), etc. A structure is a pair, triple, etc. of boxes. This section describes a few operations for use with arrays and structures. Package collect/boxes.ast defines genera BOXES, NEW and ARRAY, described under the standard genera. Those are used in describing the polymorphic types of some of these operations. Two useful species are BoxFlavor and CopyFlavor.

    Creating arrays and structures

    The most convenient way to create an array is to use a var expression or declaration. If you prefer to use functions to build arrays, the following are available only if package collect/boxes.ast is imported.

    array
    : BoxFlavor -> Natural -> [<:`a:>]

    (array flav n) is an array of n new empty boxes The flavor is given by flav. For example, array shared 10 produces an array of 10 new empty shared boxes. The array is packed.

    (array flav x) is an array of the same length as list x, initialized to the members of x. The first box in the array contains the member of list x, etc.

    arrayInit
    : BoxFlavor -> Natural -> [`a] -> [<:`a:>]

    (arrayInit flav n x) is a new array of n boxes initialized from list x. The first box holds the first member or x, the second box holds the second member of x, etc. If x has fewer than n members, then only a prefix of the array is initialized. If x has more than n members, then only a prefix of x is used.

    fullArray
    : BoxFlavor -> `a -> Natural -> [<:`a:>]

    (fullArray flav v n) is an array of n boxes of flavor flav, each holding initial value v.

    nonsharedArray
    : Natural -> [<:`a:>]

    nonsharedArray(n) is the same as (array nonshared n).

    sharedArray
    : Natural -> [<:`a:>]

    sharedArray(n) is the same as (array shared n).

    infiniteArray
    : BoxFlavor -> [<:`a:>]

    (infiniteArray flav) is an infinite array of new empty boxes The flavor is given by flav. The array is close to being packed; subscripting into it is fairly efficient.

    infiniteArrayInit
    : BoxFlavor -> () -> [`a] -> [<:`a:>]

    (infiniteArrayInit flav () x) is an infinite array of new boxes initialized from list x, in a style similar to arrayInit.

    fullInfiniteArray
    : BoxFlavor -> () -> `a -> [<:`a:>]

    (fullInfiniteArray flav v ()) is an infinite array of boxes, all initialized to hold value v.

    infiniteNonsharedArray
    : () -> [<:`a:>]

    infiniteNonsharedArray() is the same as (infiniteArray nonshared).

    infiniteSharedArray
    : () -> [<:`a:>]

    infiniteSharedArray() is the same as (infiniteArray shared).

    new
    : BoxFlavor -> NEW`a

    new flav is a newly created structure containing empty boxes of flavor flav.

    Indexing and taking the contents of an array

    @#
    : [<:`a:>] -> `a

    x @# k abbreviates @(x#k).

    @#_
    : [<:`a:>] -> `a

    x @#_ k abbreviates @(x#_k).

    contents
    : [<:`a:>] -> [`a]
    : [[<:`a:>]] -> [[`a]]
    : [[[<:`a:>]]] -> [[[`a]]]

    The contents of an array is obtained by replacing each box by its content. Function contents takes the content of a one, two or three dimensional array.

    You can use more general function contents!! to take the contents of an arbitrary dimension array, but it is not type-safe. If you need to use higher dimensional arrays, create a version of function contents by defining it to be the same as contents!!, but giving it a sane type. See collect/boxes.ast.

    optContents
    : [<:`a:>] -> [Optional(`a)]
    : [[<:`a:>]] -> [[Optional(`a)]]
    : [[[<:`a:>]]] -> [[[Optional(`a)]]]

    optContents(x) is obtained from x by replacing each nonempty box b by value(@b), and replacing each empty box by noValue. It is available for one, two and three-dimensional arrays.

    You can use more general function optContents!!, but it is not type safe.

    Copying an array

    copy
    : CopyFlavor -> `a -> `a
    : `a -> `a

    Expression (copy flav x) makes a copy of array x, using new boxes. The flavor of the new boxes is given by flav, which can be one of shared, nonshared or same. If flav is same, each box is made the same flavor as the box that it is a copy of.

    copy(x) is the same as copy same x.

    You can copy any structure consisting of lists and pairs, nested in an arbitrary way.

    Copy
    : (BOXES`a, BOXES`a) -> ()

    Statement 

      Copy x := y.
    copies array y into array x. The copy goes out to the smaller of the two array sizes, so it is not necessary for the two arrays to be the same size. For example, if x and y are created by 
       Var x(3) := [1,2,3].
   Var y(4).
    then 
       Copy y := x.
    assigns y#1 := 1, y#2 := 2 and y#3 := 3.

    Copy can copy arbitrarily nested lists and pairs. For example, it will work for a list of lists, or a list of pairs, or a pair of lists.

    Move
    : (BOXES`a, BOXES`a) -> ()

    Move is similar to Copy. However, it presumes that the two arrays (the one being copied from and the one being copied into) do not share any boxes. It is more efficient than Copy. (For generality, copy must get all contents from the source array before putting anything into the destination array. Move does not need to do that.)

    Putting values in an array

    Fill
    : ([<:`a:>], `a) -> ()
    : ([[<:`a:>]], `a) -> ()
    : ([[[<:`a:>]]], `a) -> ()

    Statement 

      Fill x := v.
    assigns v to each box in array x. Array x can have up to three dimensions. For more dimensions, you can use the more general but type-unsafe Fill!!. See collect/boxes.ast.

    Put
    : ([<:`a:>], [`a]) -> ()
    : ([[<:`a:>]], [[`a]]) -> ()
    : ([[[<:`a:>]]], [[[`a]]]) -> ()

    Statement 

      Put x := alist.
    stores the first member of list alist into the first box in array x, the second member of alist into the second box in array x, etc. If list alist is shorter than array x, then only a prefix of x is modified. If array x is shorter than list alist, then only a prefix of alist is used.

    Put can handle arrays of up to three dimensions. For higher dimensions use type-unsafe function Put!!, described in collect/boxes.ast.

    OptPut
    : ([<:`a:>], [Optional(`a)]) -> ()
    : ([[<:`a:>]], [[Optional(`a)]]) -> ()
    : ([[[<:`a:>]]], [[[Optional(`a)]]]) -> ()

    Statement 

      OptPut x := alist.
    is similar to Put x := alist %Put, but each member of alist is either value(v) (indicating that v should be put into the box) or is noValue (indicating that the box should be emptied).

    Array x can have up to three dimensions. For more dimension, use function OptPut!!, described in collect/boxes.ast.

    Getting the nonempty boxes

    nonemptyPrefix
    : [<:`a:>] -> [<:`a:>]

    nonemptyPrefix(x) is the longest prefix of x that consists entirely of nonempty boxes. It only handles linear arrays.

    Showing an array

    If you import collect/boxes.ast, the definition of $ will be changed so that arrays are shown in a more compact notation than if you do not import that package. For example, you might find that $(x) is "[shareds 20..22]" rather than [(shared 20), (shared 21), (shared 22)].
