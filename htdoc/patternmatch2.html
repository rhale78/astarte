D15. More on Pattern Matching

  • Pattern declarations
  • Expansion Time
  • Forcing pattern translations

    • Pattern declarations

    You can define your own kinds of patterns using a pattern declaration. The form is 
          Pattern p => t.
    where p is a pattern form and t is the translation of Match p = target. For example, 
          Pattern test(?n) => 
        Match ?n = untest(target).
      %Pattern
    indicates that 
          Match test(?q+1) = r.
    should be translated by the compiler to 
          Match ?q+1 = untest(r).
    Here are a rules for pattern declarations.

    1. The pattern being defined must be an application of a function. The pattern is associated with that function. For example, the pattern declaration above is associated with function test. You may use binary operator syntax and other syntactic abbreviations. For example, ?a + ?b abbreviates (+)(?a,?b), so is an application of function +.

    2. The pattern declaration applies to any match where the actual pattern has a form that looks like the form given in the pattern declaration. For example, 
            Pattern test(?a,f(?b)) =>
        Match ?a = left(target).
        Match ?b = unf(right(target)).
      %Pattern
      will only be used to translate a match of the form 
            Match test(A,f(B)) = C.
      for some expressions A, B and C.

    3. In the pattern of the pattern declaration, you can use pattern identifiers (such as ?x), anonymous pattern identifiers (?), identifiers (such as x), constants (such as 2), functions (such as f(...)) and tuples (such as (a,b)).
      ?x Stands for any pattern.
      ? Stands for ? (and only ?)
      x Stands for any pattern-identifier-free expression
      2 Stands for 2 (or 2.0).
      f(E) Stands for f(A) where E can stand for A.

      For example, 

          Pattern bird(5, n, ?x, "yes") => 
      Match ?x = feather(n).
    %Pattern
      will be only used for matches of the form 
          Match bird(5, A, B, "yes") = C.
      where A is an expression with no pattern variables in it, and B can be any pattern at all.

      Each pattern identifier or identifier must occur no more than once. So 

           Pattern test(?x,?x) =>
       Match ?x = untest(target)
     %Pattern
      is not allowed.

    4. If your pattern contains ?x, then your translation must define ?x using a pattern match.

    5. If you define any identifiers in the translation (for example, using let) then new names are chosen for those identifiers when the translation is done. For example, if you declare 
            Pattern f(?x,k) =>
        Let y = 2*k.
        Match ?x = y*y.
      %Pattern
      then statement 
            Match f(?z,10) = 50.
      is translated by the compiler to 
            Let newid = 2*10.
      Match ?z = newid*newid.
      where newid is some new identifier.

    6. Performing a translation at a pattern match requires doing substitutions. If a given expression will be put into the translation in only one place, then it is substituted directly. If the expression must be put in more than one place, then a name is given to its value, and that name is substituted instead. For example, declaration 
            Pattern g(?x,k) =>
        Match ?x = k + k + target + target.
      %Pattern
      causes pattern match 
            Match g(?y,h(n)) = sqrt(z).
      to be translated to 
            Let newid1 = sqrt(z).
      Let newid2 = h(n).
      Match ?y = newid2 + newid2 + newid1 + newid1.
      If an expression ?x occurs twice, then the expression that matches it is copied to two places.

    Expansion time

    The body of a pattern declaration typically contains other match expressions. Those match expressions are not expanded until the rule is used. As a consequence, any rules that are defined between the point where a given rule is defined and that rule is used will have an effect on that rule.

    Forcing pattern translations

    Statement 
          Match f(x) = y.
    stands for 
          {f(x) == y}
    since expression f(x) has no pattern identifiers. Suppose that you want to force a pattern declaration for function f to be used instead of computing f(x) and testing to see if it is the same as y. You can do that in two ways. One is to tag f as a pattern function in the match, as follows. 
          Match (pattern f)(x) = y.
    The other is to use a pattern assumption. Declaration 
          Assume f: pattern.
    indicates that certain occurrences of f should implicitly be marked for pattern match translation. Specifically, the occurrences of f that are marked are those that occur in patterns, but do not occur within some other expression that calls for direct computation. So, For example, f will be marked for pattern matching in f(x,g(y)), but not in g(f(x)), if function g is not marked for pattern matching.
