    • Species Boolean and Comparison

    Species Boolean and Comparison are enumerated species. Their members are as follows.

    Species Members
    Boolean false   true
    Comparison equal   greater   less

    The following operations can be used to convert between Boolean or Comparison and natural numbers.

    boolean
    : Natural -> Boolean

    boolean(0) = false and boolean(1) = true.

    comparison
    : Natural -> Comparison

    Function comparison converts from a natural number to species Comparison (the species of the result of function compare). comparison(0) = equal, comparison(1) = greater and comparison(2) = less.

    rank
    : RANKED`a -> Natural

    Function rank converts from an enumerated species to a natural number. rank(false) = 0, rank(true) = 1, rank(equal) = 0, rank(greater) = 1 and rank(less) = 2.

    Comparisons

    Comparisons are described in more detail in lang/compare.ast. They are standard, and do not require an import. All species in genus ORDER support order comparisons such as < and >. Species in genus EQ support equality and inequality comparisons.

    ==
    : (EQ`a, EQ`a) -> Boolean

    x == y is true just when x and y are the same data item. Equality tests are available for most standard species. Functions and boxes are exceptions. No equality test is available for functions because equality of functions is undecidable. No equality test is available for boxes to prevent confusion. For boxes, use _eq_.

    When you create a new species, a definition of == will be provided for you. The standard definition says that two values are equal if they have equal representations. You can suppress this by using option noEqual. For example, to define your own equality checker, you would use something like this. 

        Species{noEqual} MySpecies = Natural.

    =/=
    : (EQ`a, EQ`a) -> Boolean

    x =/= y if x and y are not equal. This is defined as the negation of ==, and should not be redefined. Redefine == instead.

    ==?
    : (EQ`a, EQ`b) -> Boolean

    When you write x == y, the compiler assumes that x and y must have the same (extrinsic) species. x ==? y is similar to x == y, but it allows the possibility that x and y have different species.

    In most cases, two values of different species are not equal. The numeric species are special. Two numeric values can be equal even when they have different species. For example, 2 ==? 2.0 is true.

    =/=?
    : (EQ`a, EQ`b) -> Boolean

    x =/=? y is the same as not(x ==? y).

    _eq_, _neq_
    : (<:`a:>, <:`a:>) -> Boolean

    These are the equality and inequality tests for boxes (boxes). Operators == and =/= are not defined for boxes because it is likely that their meaning will be misunderstood. x _eq_ y is true if x and y are the same box, not if they have the same content.

    compare
    : (ORDER`a, ORDER`a) -> Comparison

    compare(x,y) compares x and y, and produces value less if x < y, value equal if x == y and value greater if x > y.

    Operations <, <=, > and >= all use compare to perform their tests. If you need to add a new ordered species, define compare for it, and add your species to genus ORDER. Operations <, >, <= and >= will be defined automatically in terms of compare.

    Operation == does not use compare. It must be written separately.

    Example: compare(2,5) = less.

    <
    : (ORDER`a, ORDER`a) -> Boolean

    x < y if x is less than y.

    <=
    : (ORDER`a, ORDER`a) -> Boolean

    x <= y if x is less than or equal to y.

    >
    : (ORDER`a, ORDER`a) -> Boolean

    x > y if x is greater than y.

    >=
    : (ORDER`a, ORDER`a) -> Boolean

    x >= y if x is greater than or equal to y.

    Maximum and minimum

    max
    : [ORDER`a] -> ORDER`a

    max(x) is the largest value in list x.

    Example: max[2,4,6,3] = 6.

    min
    : [ORDER`a] -> ORDER`a

    min(x) is the smallest value in list x.

    Example: min[2,4,6,3] = 2.

    maxmin
    : ((`a,`a) -> Boolean) -> [`a] -> `a

    (maxmin cmp x) is an extremal value in list x, where the nature of the extreme is determined by comparison function cmp. For example, maxmin(>=) x is the maximum value in list x, and maxmin(<=) x is the minimum value in list x. The comparison function should be a total order.

    Example: maxmin(>=)[2,4,6,3] = 6.

    Boolean operations

    Boolean operations are described in lang/boolean.ast. That package does not need to be imported.

    _and_
    x _and_ y abbreviates If x then y else false %If. It is a short-circuited "and" operation.

    _andf_
    : (Boolean, Boolean) -> Boolean

    x _andf_ y is the "and" of boolean values x and y. _andf_ is a function, and is not short-circuited.

    _or_
    x _or_ y abbreviates If x then true else y %If. It is a short-circuited "or" operation.

    _orf_
    : (Boolean, Boolean) -> Boolean

    x _orf_ y is the "or" of boolean values x and y. _orf_ is a function, and is not short-circuited.

    _implies_
    x _implies_ y abbreviates If x then y else true %If. It is a short-circuited "implies" operation.

    ==>
    : (Boolean, Boolean) -> Boolean

    x ==> y is true if either x is false or y is true. ==> is a function, and is not short-circuited.

    not
    : Boolean -> Boolean

    not(true) = false and not(false) = true.

    Predicates

    A predicate is a function that returns a boolean result. Most predicates in the library have names that end on a question mark. For example, predicate prime? returns true on a prime number and false on a nonprime number.

    You can apply a predicate just as any other function, or you can use the standard operator _is_. Expressions prime?(n) and n _is_ prime? have the same meaning. So you might write something like this. 

        If n _is_ prime? then
      ...
    else
      ...
    %If
