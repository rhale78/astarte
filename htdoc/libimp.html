Library Implementation

The library consists of a combination of built-in and defined items. The built-in operations are described in packages in the lang directory, and in a few other directories. The defined operations are defined by packages in other directories, with a few exceptions that are found in the lang directory. Packages that define items are typically broken into two parts, an interface (with extension .ast) and an implementation (with extension .asi). See the interface for descriptions and type information. See the implementation for the actual definition.
  • standard.ast These things are imported by every package.

  • standard.asi Some of the definitions made here are simple and instructive to look at. A few, however, make use of some items that only this package is allowed to use. Do not try to make sense out of those.

    Support for object-oriented programming

  • object.ast This package provides the top class Object for object-oriented programming, and implements basic operations that all objects shared.

  • object.asi Here are the implementations of the shared operations.

    Collections

  • collect/boxes.ast These operations are used for creating and manipulating lists and products of boxes.

  • collect/boxes.asi These are the definitions.
  • collect/bitvec.ast This package defines infinite bit vectors and operations on them. Near the beginning are descriptions of built-in functions, such as nbvAnd, that you probably are not interested in.

  • collect/bitvec.asi The implementation of infinite bit vectors is done via primitives that work on integers. This implementation package just packages the primitives for use on species Bitvec.
  • collect/collect.ast This package describes an abstract concept of a collection. Examples of collections are lists and sets.

  • collect/collect.asi Most of the operations on collections are virtual. A few are defined here.
  • collect/denseset.ast This package implements sets as bit vectors. See collect/set.ast.

  • collect/denseset.asi Here the set operations are defined in terms of operations on bit vectors.
  • collect/findlist.ast This package defines some functions that search lists for substrings.

  • collect/findlist.asi The implementions here use the Knuth/Morris/Pratt and Aho/Corasick algorithms for searching lists.
  • collect/funarr.ast This package provides an efficient implementations of functional arrays, which behave like arrays, but have persistent operations.

  • collect/funarr.asi Here are the definitions.
  • collect/linset.ast This package implements sets as lists.

  • collect/linset.asi Here set operations using lists are defined. See collect/set.ast.
  • collect/lintbl.ast This package implements tables as lists. See collect/table.ast.
  • collect/list.ast This package describes built-in list operations.
  • collect/listfun.ast This package provides a few general functions for working with lists.

  • collect/listfun.asi Here are the implementations. The style is direct (one equation) rather than a more elegant one with multiple equations. This package was written when the multiple equation style was not very efficiently implemented.
  • collect/pair.ast This package provides a few utilities for working with ordered pairs.
  • collect/search.ast This packages provides operations that search lists, and closely related operations.

  • collect/search.asi The implementations here are mostly given in two styles: an elegant equational style (in a comment) and a more efficient style.
  • collect/set.ast This package describes the abstract concept of a set. Most of the operations are virtual, but a few are defined here.

  • collect/set.asi Here are implementations of those set operations defined at the abstract level.
  • collect/sort.ast This package provides general functions for sorting lists.

  • collect/sort.asi The sort implementation uses mergesort.
  • collect/sparsset.ast This package implements sets as balanced binary search trees. See collect/set.ast and collect/srchtree.ast.

  • collect/sparsset.asi Here is the implementation of binary search trees.
  • collect/sparstbl.ast This package implements tables as balanced binary search trees. See collect/table.ast.
  • collect/srchtree.ast This packages provides binary search trees.

  • collect/srchtree.asi Here is the implementation of binary search trees. The trees are weight-balanced.
  • collect/string.ast This package provides operations on strings. It also provides some patterns for doing pattern matching in strings.

  • collect/string.asi Here is the implementation of the string operations.
  • collect/table.ast This package defines an abstract concept of a table. Most of the functions are virtual, but a few are defined here.

  • collect/table.asi Here the operations are defined that can be defined at the abstract level.
  • collect/matrix.ast These operations are useful for two-dimensional lists.

  • collect/matrix.asi Here are the definitions.

    Language description packages

  • lang/boolean.ast This package describes the built-in Boolean operations.
  • lang/box.ast This package describes built-in box operations, as well as Var expressions.
  • lang/bracket.ast This package discusses the syntax and bracketing.
  • lang/char.ast This package describes built-in operations on characters, and implements a few operations.

  • lang/char.asi Here are the implementations of the character operations that are not built-in.
  • lang/choose.ast This package describes the choice constructs.
  • lang/compare.ast This package describes comparisons, such as equality and ordering comparisons.
  • lang/except.ast This package describes exceptions and failure.
  • lang/expect.ast This package describes expectations and anticipations.
  • lang/function.ast This package describes functions and built-in operations on functions.
  • lang/hermit.ast This package describes () and what it does.
  • lang/hierarch.ast This package shows the standard genus/species hierarchy.
  • lang/import.ast This package describes imports.
  • lang/irreg.ast This package describes irregular functions and wrapped items.
  • lang/lazy.ast This package describes delayed evaluation.
  • lang/let.ast This package describes let and define expressions.
  • lang/loop.ast This package describes loop constructs.
  • lang/misc.ast This package describes some built-in concepts that do not seem to fit anywhere else.
  • lang/operator.ast This package describes binary and unary operators, and has a precedence table.
  • lang/optional.ast This package describes values that have a special "noValue" item that indicates no value is present.
  • lang/pattern.ast This package describes pattern matching.
  • lang/rank.ast This package describes ranked and enumerated species (which are similar to natural numbers or finite ranges of natural numbers), and operations on them.
  • lang/stream.ast This package describes facilities for backtracking and for multiple threads.

    Logic programming support

  • logic/symbol.ast This package provides the concept of a symbol, or an atom.
  • logic/unify.ast This package implements unification.
  • logic/unknown.ast This package describes the built-in concept of an unknown.
  • logic/utils.ast This package provides a few utilities that are useful for logic programming.

  • logic/utils.asi Here are the implementations of the utilities.

    Miscellaneous

  • misc/random.ast This package provides a pseudo-random number generator.

  • misc/random.asi The pseudo-random number generator is based on the difficulty of taking square roots mod a product of two distinct primes.
  • misc/trace.ast This package provides a few functions that are useful in tracing computations.

    Numeric operations

  • math/complex.ast This package provides some operations on complex numbers.

  • math/complex.asi Here is the implementation.
  • math/math.ast This package provides a few mathematical functions, such as exp and ln.

  • math/math.asi Here are the implementations.
  • math/modular.ast This package provides some operations that use modular arithmetic.

  • math/modular.asi Here are the implementations.
  • math/number.ast This package generalizes the standard numeric genera to allow other numeric species, such as complex numbers. It creates several genera, and some virtual functions for them.

  • math/number.asi Here, a few general operations on numbers are defined.
  • math/prime.ast This package provides a test for primality and a factoring function.

  • math/prime.asi The prime number tester is Monte Carlo, and the factoring algorithm is Pollard's.
  • math/stdnum.ast This package described built-in numeric operations.

    Pattern matching support

  • pattern/listpat.ast This package defines list patterns that involve taking sublists or breaking apart lists. It also provides a few support operations for doing the breakup of the lists.

  • pattern/listpat.asi Here are implementations of some of the support operations.
  • pattern/pattern.ast Here first class patterns are defined.

  • pattern/pattern.asi Here are implementations of helper functions for first class patterns.
  • pattern/stringp.ast This package provides first class patterns for pattern matching in strings.

  • pattern/stringp.asi Here is the implementation of the first class string patterns.

    System operations

  • system/io.ast This package describes built-in operations for input from files and output to files. A few new operations are defined here as well.
  • system/system.ast This package describes some functions that interact with the operating system, and creates some more pleasant interfaces to some of the built-in functions. It also creates the notion of a file, and provides some operations on file names.

  • system/system.asi Here the operations are implemented.

    Creating threads and communicating among threads

  • thread/channel.ast This package provides communication channels for communicating between threads.

  • thread/channel.asi Here, the channel operations are implemented.
  • thread/semaphor.ast This package provides semaphores for synchronizing threads.

  • thread/semaphor.asi Here, semaphores are implemented.
  • thread/sync.ast This package provides an alternative synchronization mechanism for threads.

  • thread/sync.asi Here is the implementation.
  • thread/thread.ast This package provides support for creating and controling threads.

  • thread/thread.asi Here is the implementation.
