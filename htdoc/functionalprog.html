    • Functional programming

    Functional programming has the following characteristics.

    1. Data consists only of values that do not change. There is no concept of a variable that can be changed. All data are constants. This is close to the way mathematicans model things. It has the advantage that it is not possible for a programmer to make the error of changing something in one part of a program that another part of the program needs to remain constant.

    2. Computation is performed by functions that compute new values from old. The functions are written using equations. The equations can be checked mathematically for correctness.

    3. Functional programming is done at a conceptual level, with most of the details of computation handled by the language implementation. This tends to make programs short and simple. Debugging time is minimal.

    This page describes basics of how to do functional programming in Astarte. Ideally, before reading this, you should be familiar with the most basic notions covered in the following sections. You can read on and come back to these if necessary.

    Functions with a single defining equation

    If you have a function that can be described conveniently with a single equation, then just write that equation. For example, the function sqr(x) = x2 can be defined as 
          Define sqr(?x) = x^2.
    Notice that the parameter x must have a question mark in front of it in the heading (sqr(?x)) but not in the body (x^2).

    A function that takes the third member of a list can be written as 

          Define third(?x) = head(tail(tail(x))).

    Functions with more than one equation

    You can write a function with several defining equations. Each equation can have a proviso that tells under what circumstances it is true. Here are some examples.

    A function that takes the maximum of two values: 

          Define max by
        case max(?x,?y) = x      when x >= y
        else max(?,?y)  = y
      %Define
    A parameter without a name (?) is one that you don't use.

    Here is a function that returns the sign of a number. (The sign of a positive number is 1, the sign of a negative number is -1 and the sign of 0 is 0.) 

          Define sign by
        case sign(?x) = 1       when x > 0
        case sign(?x) = 0       when x == 0
        else sign(?)  = -1
      %Define

    Pattern matching

    A convenient way to break up a parameter is by pattern matching. The pattern match solves a simple equation. For example, an alternative definition of the function that computes the third member of a list is 
          Define third [?a, ?b, ?c & ?r] = c.
    Expression [x,y,z & s] creates a list that begins [x,y,z,...], with suffix (the ...) given by list s. When you compute third([1,2,3,4,5]), you must solve a simple equation
    [a,b,c & r] = [1,2,3,4,5]
    The solution is clearly a = 1, b = 2, c = 3 and r = [4,5]. The definition of function third then says that the answer is c (=3).

    In the pattern, the question marks decorate the things that are being solved for. Sometimes, you need to solve for something but do not really care what it is. You can use plain question mark for such things. The definition of third can be written as follows. 

          Define third [?, ?, ?c & ?] = c.

    Recursion

    You can use recursion in definitions.

    Example: the factorial function. 

          Define factorial by
        case factorial (0)    = 1
        case factorial (?n+1) = (n+1) * factorial(n)
      %Define
    Notice that pattern ?n+1 has been used. To compute factorial(4) requires solving the equation n+1 = 4, yielding n = 3.

    Example: a function that takes the sum of the members of a list. (So sum[2,3,4] = 9.) 

          Define sum by
        case sum []       = 0
        case sum (?h::?t) = h + sum(t)
      %Define
    Example: a function that reverses a list. (So rev[2,3,4] = [4,3,2].) 
           Define rev by
         case rev []       = []
         case rev (?h::?t) = rev(t) ++ [h]
       %Define

    Higher order functions

    A function can take functions as parameters. A function that works on other functions is called a higher order function.

    Here is a function that takes a function f and a list [a,b,c,d] and produces list [f(a), f(b), f(c), f(d)]. It does a similar thing for arbitrary length lists. 

          Define map by
        case map(?f,[])      = []
        case map(?f, ?h::?t) = f(h) :: map(f,t)
      %Define
    So, for example, suppose that function sqr is defined to square its parameter, as was done above. Then map(sqr,[1,2,3,4,5]) = [1,4,9,16,25].

    Curried functions

    Not only can a function take other functions as parameters, but it can produce a function as its result. Imagine a function add such that add(x) is a function. Specifically, (add(x))(y) = x + y. That is, add(x) is a function that adds x to its parameter.

    In Astarte, function application (juxtaposition) associates to the left. If you write 

     add x y
    you mean the same thing as 
     (add x) y
     So add x y = x + y. You can write a function to take its parameters in this way. For example, you can write 
        Define add ?x ?y = x + y.
    It looks like function add takes two parameters, and it does. But it takes them one at a time, and it is acceptable to give only one. For example, add(1) is a function that adds one to its parameter. To write a function called inc such that inc(x) = x + 1, you can write 
        Let inc = add 1.
    Notice that inc is a function, since add(1) is a function.

    When a function is designed to take its parameters one at a time, it is called a curried function. Curried functions have the advantage that they can be given partial information.

    The map function written above is not curried. There is a map function in the library that is curried. A simple curried map function can be written as follows. 

        Define map by
      case map ?  []       = []
      case map ?f (?h::?t) = f(h) :: map f t
    %Define
    The advantage of a curried map function is that you can fix the function f, without deciding on what the list will be. For example, 
        Let incAll = map (add 1).
    defines function incAll, where incAll[a,b,c,d] = [a+1, b+1, c+1, d+1].

    A useful higher order function is rfold. It performs an accumulation. You give it an initial value and an operator. Examples are

    rfold(0,+) [a,b,c,d] = a + b + c + d + 0
    rfold(1,*) [a,b,c,d] = a * b * c * d * 1
    In general, you give a function of two arguments as the operation. The operator associates to the right. (Hence, the r in rfold.) So
    rfold(i,f)[a,b,c] = f(a,f(b,f(c,i)))
    Here is a definition of rfold. 
        Define rfold by
      case rfold(?i,?) []        = i
      case rfold(?i,?f) (?h::?t) = f(h, rfold(i,f) t)
    %Define

    List comprehensions

    A list comprehension is a convenient way of describing a list.

    Expression [:e | generate {test} :] produces the list of all values of e that are obtained from values generated by the generate part and that pass the test of the test part. For example, function prime? returns true just when its parameter is prime. List comprehension 

        [: x | Let x = each(1 _upto_ 20).  {prime?(n)} :]
    produces list [2,3,5,7,11,13,17,19], the primes up to 20. The generate part typically selects members of a list using each. The test is enclosed in braces. It can be omitted if no filtering is to be done.

    The expression that tells what to put in the list can be any kind of expression. For example, to produce the list of squares of numbers from 1 to 10, you can write 

        [: x^2 | Let x = each(1 _upto_ 10). :]
    This produces list [1,4,9,16,25,36,49,64,81,100].

    The map function was written above. The idea is that map f  [a,b,c] = [f(a), f(b), f(c)]. It is very easy to write map using a list comprehension. 

        Define map ?f ?x = [: f(y) | Let y = each x. :].
    Here is an implementation of the quicksort sorting algorithm using list comprehensions. 
        Define quicksort by
      case quicksort [] = []

      case quicksort (?pivot :: ?t) =
           quicksort [: x | Let x = each t. {x <= pivot} :]
        ++ [pivot]
        ++ quicksort [: x | Let x = each t. {x > pivot} :]
    %Define
    You can create a higher order version of quicksort that takes a comparison function as a parameter. With this version, quicksort(<) x sorts into ascending order, but quicksort(>) x sorts into descending order. Here is such a higher-order function. 
        Define quicksort by
      case quicksort ? [] = []

      case quicksort ?cmp (?pivot :: ?t) =
           quicksort cmp [: x | Let x = each t. {cmp(x,pivot)} :]
        ++ [pivot]
        ++ quicksort cmp [: x | Let x = each t. {not cmp(x,pivot)} :]
    %Define
    Notice how useful it is that this is curried. If you want to define a function called sort that sorts into ascending order, you only need to write 
        Define sort = quicksort(<).

    Making lazy functions

    By default, functions are evaluated in an eager fashion. This means that the parameter of a function is evaluated before starting to compute the function. For example, if you define sqr(x) to be x*x, then evaluating sqr(10 + 10) will start by evaluating 10+10 = 20, and then will evaluate 20*20 = 400.

    You can ask for evaluation to be delayed until the value is needed. One way to do that is to use form (:e:), which creates a deferred version of expression e. So, you can write 

        Define lazySqr(?x) = (:x*x:).
    to create a function that does the same thing as function sqr, but that does not do the multiplication until the value is needed. If you write lazySqr(lazySqr(10+10)), then 10+10 is still computed before starting the inner lazySqr, but the multiplication 20*20 is deferred. Similarly, the outer lazySqr does not perform its multiplication. Only when the result of this expression is needed is the evaluation performed.

    This kind of thing can be used to create lists that are infinitely long. Definition 

        Define nats(?n) = (:n :: nats(n+1):).
    defines function nats so that nats(n) = [n,n+1,n+2,...]. The list that is produced is infinitely long, but only as much of the list as is ever looked at will be computed.

    Another way to accomplish the same thing is to add phrase await then to a definition by cases. So function nats can be defined as 

        Define nats by 
      Await then
        case nats(?n) = n :: nats(n+1)
      %Await
    %Define
    Here is a version of the map function that only evaluates as much of the result list as is needed. 
        Define map by 
      Await then
        case map ?  []       = []
        case map ?f (?h::?t) = f(h) :: map f t
      %Await
    %Define
    Yet another approach to defining a lazy function is to use a list comprehension. List comprehensions are implicitly lazy, and only compute as much of their lists as is looked at. So definition 
        Define map ?f ?x = [: f(y) | Let y = each x. :].
    also defines a lazy map function.

    Polymorphism

    So far, we have ignored type information. The compiler will infer type information in a most general way. Consider the function 
         Define idf(?x) = x.
    This function is polymorphic. It can take any kind of item, and produces the same kind of item. Its polymorphic type is `a -> `a. Here, `a stands for any type at all, but it must be the same type at every occurrence. The arrow -> indicates a function. So polymorphic type `a -> `a indicates a function that takes any type of item and produces the same type of item as its result.

    Here is some notation that is used for writing types.

    Natural
    This is the type (or species) Natural. You can use names for basic species.

    A -> B
    This is the type of a function that takes a parameter of type A and produces a result of type B. Type A is called the domain of the function type, and B is called the codomain of the function type.

    (A,B)
    This is the type or ordered pairs where the first member has type A and the second member has type B.

    [A]
    This is the type of lists whose members have type A.

    REAL`a
    REAL is a data abstraction. It can be thought of as a set of species. Abstraction REAL contains species Natural, Integer, Rational and Real. REAL`a is a variable that can be any of the members of REAL.

    Another example of a data abstraction is ORDER. It contains all species that allow comparisons such as < and >. A variable that stands for any species in ORDER might be ORDER`v.

    What comes after the ` is any normal (not symbolic) identifier. So you can have a type variable REAL`mynum. You can leave the abstraction off, as in `a, for a type variable that can stand for any species at all.

    Here are some functions that were defined above, along with their polymorphic types. (Note that factorial is not polymorphic.)

    Function Type
    factorial Natural -> Natural
    sqr REAL`a -> REAL`a
    third [`a] -> `a
    max (ORDER`a,ORDER`a) -> ORDER`a
    sum [REAL`a] -> REAL`a
    rev [`a] -> [`a]
    add REAL`a -> (REAL`a -> REAL`a)
    map (`a -> `b) -> ([`a] -> [`b])    (this is for the curried definition)
    rfold (`a, (`b,`a) -> `a) -> ([`b] -> `a)
