M5. A Note to New Users

  • Types (Species)
  • Subtracting natural numbers
  • Arithmetic
  • Names that start with a capital letter
    • There are some aspects of Astarte that tend to catch novices by surprise. This page calls attention to some.

    Types (Species)

    Astarte is a typed language. However, it appears to be untyped, since you do not need to put explicit type information into your program. The compiler determines type information for you. Be sure that your definitions make sense from a point of view of types, or they will not compile. If you get a type error that you do not understand, try reading understanding type errors.

    Subtracting natural numbers

    The library uses type Natural (natural numbers, including 0, but not including any negative numbers) for quite a few things. For example, if you use x#i to select the i-th member of list x, then i must be a natural number.

    When subtracting natural numbers, you are supposed to use operator --, not operator -. x -- y is defined to be |x - y|. That way, you cannot get a negative answer. If you use ordinary subtraction (-) on natural numbers, you will get a type error, because subtraction is not defined on natural numbers.

    Arithmetic

    Astarte does not allow mixing of numeric types in expressions. For example, if n has type Natural and i has type Integer, then n+i is not an admissible expression. One way around this is to use a unary +. If n is any kind of number, then +n is the same number possibly converted to another type. For example, you can write (+n) + i

    Operations are typically polymorphic, allowing several different types. The available types of numeric operations are noted in the documentation. For example, the addition function can have the following types. (T,T) -> T indicates that it takes a pair of things of type T, and produces something of type T. 

          +: (Natural, Natural) -> Natural
      +: (Integer, Integer) -> Integer
      +: (Rational, Rational) -> Rational
      +: (Real, Real) -> Real
    As you can see, there is no definition of type (Natural, Integer) -> Integer. You cannot add things of different types.

    Exponentiation is available, but it is a little tricky. The method employed depends on the type of the exponent. There are three different exponentiation operators.

    1. x^n is x to the n-th power, where n has type Natural. x can be any kind of number.

    2. x^/i is x to the i-th power, where i has type Integer. This allows negative powers. x must be of type Rational or Real, or some other type that is closed under division.

    3. x^~r is x to the r-th power, where r has type Real. The algorithm uses logarithms, and requires that x is not negative.
    For reasons that are difficult to explain here, there is no exponentation operator where the exponent has type Rational.

    Names that start with a capital letter

    Names that start with a capital letter are used to name types and data abstractions, and also to name procedures, which are functions that are run for their side effects, and do not return values. You must be careful how you use names that begin with a capital letter. Don't use names that begin with a capital letter to name your own things until you understand them.

    Each time you use a procedure P, you must include an end marker %P after the use. For example, you write 

         Writeln ["Hello"] %Writeln
    to run procedure Writeln on list ["Hello"]. The end marker can be replaced with a period if it occurs on the same line as the name of the procedure. For example, you can write 
         Writeln ["Hello"].

    There is more detail about procedures in Procedures.
