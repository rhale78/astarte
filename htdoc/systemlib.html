L11. System Functions

  • Introduction
  • Miscellaneous operations
  • File names
  • File properies
  • Creating and removing files

    • Introduction

    The following are described in detail in system/system.ast. Import that package to use these functions, and see that package for details that are not discussed here.

    Several functions require file names. Use a / or a \ to separate directories. For example, you can write /home/myself or \home\myself. But be careful if you use \ in a string constant; you must write "\\home\\myself".

    You can create an item of species File that holds information about a system file. For example, file("mydir/myfile.txt") is a File item that records information about system file myfile/myfile.txt (found relative to the current directory). Some operations require that a file be given as an item of species File. Some other operations can take either a string that names a file or a File item.

    Miscellaneous operations

    ChangeDirectory
    : File -> ()
    : String -> ()

    ChangeDirectory(d). changes the current directory to d. The parameter d can be a string or a File object.
    Example: ChangeDirectory("/local/lib").

    commandLine
    : [String]

    commandLine is the command line that invoked this program, as a list of strings. Only those parts that follow the name of the program being interpreted are included, plus the program name itself. For example, if you use command 

          astr -t myprog -x elephant
    then commandLine will be ["myprog", "-x", "elephant"].

    dateTime
    : () -> String

    dateTime() is a string that describes the current date and time.

    dirList
    : File -> [File]

    dirList(d) is a list of all members of directory d, including files and directories.

    file
    : String -> File

    file(name) is the File item corresponding to the system file whose name is given. If not absolute, it is taken from the current directory.

    fileExists?
    : String -> Boolean

    fileExists?(name) is true if there is a file on the system called name, and false otherwise. A directory is considered a kind of file for this test.

    getCwd
    : () -> File
    : () -> String

    getCwd() returns the current working directory, either as a string or as a File item.

    getEnv
    : String -> String

    getEnv(s) is the value of environment variable s.

    secondsSinceTimeZero
    : () -> Natural

    secondsSinceTimeZero() returns the number of seconds that have elapsed since some reference time.

    sysEnviron
    : [(String,String)]

    sysEnviron is a list of all of the environment variable bindings. Each binding is a pair (v,b), where v is the name of a variable and b is the value of that variable.

    system
    : String -> String -> String
    : [String] -> String -> String

    (system cmd in) runs system command cmd. The standard input of cmd is string in. The returned value is the standard output produced by cmd.

    cmd can either be a single string or a list of strings. The command is not processed by a shell in either case. If you want a shell to process it, use a shell directly.

    Examples:
    system ["ls", "-l"]  ""
    system "ls -l"  ""

    File names

    /
    : (File,String) -> File

    d/s is a File item for file s in directory d. Here, s must be a string, and d must be a File item that describes a directory.

    basename
    : File -> String

    basename(f) is the name of file f, without a directory or an extension. For example, basename(file "a/b/fil.txt") = "fil".

    direct
    : File -> File

    direct(f) is the directory part of file f. For example, if f is file "a/b/c.txt", then direct(file "a/b/c.txt") = file "a/b".

    ext
    : File -> String

    ext(f) is the extension of file f. It is the part of the file name of f that comes after the last period. For example, the extension of file "fil.txt" is "txt".

    filename
    : File -> String

    filename(f) is the name of the file described by File item f, without the directory. For example, filename(file "a/b/c.txt") = "c.txt".

    pathname
    : File -> String

    pathname(f) is the full path name of the file described by File item f.

    File properties

    directory?
    : File -> Boolean

    directory?(f) is true just when f is a directory.

    executable?
    : File -> Boolean

    executable?(f) is true if file f is executable. Parameter f must be of species File.

    group
    : File -> Natural

    group(f) is the group of file f.

    modTime
    : File -> Natural

    modTime(f) is the modification time the file described by File item f.

    owner
    : File -> Natural

    owner(f) is the owner of the file described by File item f.

    readable?
    : File -> Boolean

    readable?(f) is true if the file described by file item f is readable.

    size
    : File -> Natural

    size(f) is the size, in bytes, of the file described by file item f.

    UpdateStatus
    : File -> ()

    When information about a file, such as whether it is a directory, or its size, is requested, the system is asked about all status information, and that information is memoized. Later questions are answered from the memoized information. UpdateStatus(f) %UpdateStatus indicates that the status information of File item f should be reread from the system.

    writable?
    : File -> Boolean

    writable?(f) is true if the file described by File item f is writable.

    Creating and removing files

    MakeDirectory
    : String -> ()

    MakeDirectory(s). creates a directory called s in the current directory.

    RemoveDirectory
    : File -> ()
    : String -> ()

    RemoveDirectory(d). destroys directory d. Directory d must be empty. Parameter d either be a string or a File item.

    RemoveFile
    : File -> ()
    : String -> ()

    RemoveFile(s). destroys file s. Parameter s either be a string or a File item.

    RenameFile
    : (File, String) -> ()
    : (String, String) -> ()

    RenameFile(s,t). renames file s to have new name t. Parameter s can either be a string or a File item.
