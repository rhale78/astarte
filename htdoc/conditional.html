E12. Conditionals

  • Simple conditionals
  • Choose expressions
  • Examples
    • Conditionals allow you to perform tests, and to make decisions based on the outcomes of those tests. You can use simple conditionals or the choose construct, which combines many kinds of tests in a single framework. Details on conditionals not covered here can be found in lang/choose.ast.

    Simple conditionals

    If a then b else c %If
    This expression has the same value as b if a evaluates to true, and the same value as c if a evaluates to false. Expression b will only be evaluated if a is true, and expression c will only be evaluated if a is false.

    You can omit either the "then b" part or the "else c" part. Each of b and c defaults to () if it is omitted. () means "do nothing".

    Examples:
    If z == 0 then 0 else 1/z %If
    If x > y then Relet x = y. %If
    If x < y else Relet x = y. %If

    Try a then b else c %Try
    Understanding this requires knowledge of failure. Other options for it are discussed with failure.

    This expression has the same value as b if evaluation of a succeeds, and the same value as c if evaluation of a fails. Expression a must have species (). When a succeeds, c is not evaluated. When a fails, b is not evaluated.

    Within b, you can use identifier bindings done in a, but such bindings cannot be used in c.

    Within c, you can use expression exception as the exception that occurred when evaluation of a failed.

    You can omit either the "then b" part or the "else c" part or both. Each of b and c defaults to () if it is omitted. () means "do nothing".

    Warning. If the failure of a is trapped, then evaluation will cease, and c will not be evaluated. See failure.

    Examples: 

         Try f(z) 
       then Writeln["All went well"].
       else Writeln["Exception ", $exception, " occurred"]. 
     %Try

     Try 
       f(z) 
     %Try

    Choose expressions

    Choose expressions offer a way to write very general multi-way choices. A choose expression has the form 
          Choose choice-mode
        cases
      %Choose
    where cases is a sequence of cases and choice-mode tells how the cases are tried.     Examples of choose expressions are given below. The possibilities for choice-mode are as follows.

    first
    Try the cases in the order written, choosing the first that is applicable. If you omit the choice-mode, the default is first.

    perhaps
    Same as first, but causes a default else case to be added that does nothing, if there is no else case. That is, else () is added.

    one
    Try the cases in an arbitrary order, choosing the first one that is encountered that is applicable.

    one mixed
    Try the cases in parallel, choosing the first one that is found to be applicable.

    all
    Try all cases using separate threads, using a stream that produces the result of the first case first, etc.

    all mixed
    Try all cases using separate threads, using a mixed stream, and producing the results in an arbitrary order.

    first matching e
    perhaps matching e
    ...
    Each case gives a pattern that is matched against the value of expression e. The case succeeds if the pattern match succeeds.
    The cases can have the following forms.

    case a => v
    If a is true, and this case is tried, then the value of the choose expression is v. (This kind of case is used without a matching phrase in the mode.)

    case p => v when c
    This case is used when a matching phrase is part of the choice mode. Here, p is a pattern and c is a condition. This case succeeds if pattern p matches the target given after the word matching, and condition c is true. Phrase when c can be omitted.

    case do a => v
    Here, a must have species (). If evaluation of a is successful, the case succeeds, and the value of the choose-expression is v.

    case a
    case b => v
    You can attach any number of cases to a value. This is the same as

       case a => v
       case b => v

    else => v
    This can only be the last case. If none of the other cases is successful, then the value of the choose expression is v.

    If there is no else phrase, and there are no successful cases, then the choose expression fails. For choice modes first, one and one mixed, the exception is noCaseX. For choice modes all and all mixed, the exception is testX. Choice mode perhaps is different. When the choice mode is perhaps, there is no else case, and none of the cases is successful, the value is ().

    Exception noCaseX is normally trapped. So your program will abort if it does not provide an applicable case when using modes first, one or one mixed.

    do s
    When this case is encountered, statement s is evaluated. Any bindings done by s are available in all subsequent cases. When the choice mode is all or all mixed, the same thread does s for each of the remaining cases, forking after s.

    If the choice mode is one, one mixed or all mixed then do s can only be the first case.

    do p => s
    This can only be used with a matching phrase. When this case is encountered, pattern p is matched against the target of the matches, and then statement s is evaluated. Bindings done are available in subsequent cases.

    case a =>
       Subcases choice-mode
         cases
       %Subcases
    If a is true (when there is no matching phrase in the mode) or if a is a pattern that matches the target (when there is a matching phrase in the mode) then the cases in the list of subcases are tried.

    You can omit the choice mode after subcases. It defaults to the same mode as the choose expression or subcases part that contains it, except that any matching phrase is omitted.

    You can insert a do phrase before Subcases. It is done before the subcases are tried. For example, you can write 

        case a =>
      do s
      Subcases
        ...
      %Subcases

    case do s =>
       Subcases choice-mode
         cases
       %Subcases
    This is similar to the preceding form but the cases in the list of subcases are tried if evaluation of s succeeds. Expression s must be a statement. That is, it must have type ().

    Examples

    Here are two examples of choose expressions. Expression member(v,x) is true if v occurs in list x. Notice that the member function can also be defined using a by phrase and definition by cases. We use a choose expression here for illustration of choose expressions. 
        Define member(?v,?x) =
      Choose first matching x
        case []       => false
        case [v & ?]  => true
        case [? & ?t] => member(v,t)
      %Choose
    %Define

    Define member(?v,?x) =
      Choose first
        case nil? x       => false
        case head(x) == v => true
        else              => member(v, tail x)
      %Choose
    %Define
    Here is an implementation of function mapSuccesses using a choose expression. MapSuccesses is defined so that mapSuccesses f [a,b,c,d] = [f(a), f(b), f(c), f(d)], but any cases where f fails are left out. 
          Define mapSuccesses ?f ?x =
        Choose 
          case nil? x => []

          do Match [?h & ?t] = x.

          case do Let v = f(h). => [v & mapSuccesses f t]

          else => mapSuccesses f t
        %Choose
      %Define
