    • Numeric Species

    Natural
    Species Natural contains natural numbers 0,1,2,... The size of a natural number is limited only by the overall amount of memory that your computer has, not by the size of a machine word. So you can use large natural numbers.

    Natural number constants are written in decimal notation.

    Integer
    Species Integer contains integers ..., -2, -1, 0, 1, 2, ... The size of an integer is limited only by the overall amount of memory that your computer has, not by the size of a machine word.

    Integer constants are written in decimal notation, possibly with a sign.

    Rational
    Species Rational contains rational numbers, stored exactly as ratios of integers.

    Rational constants are written in decimal notation, with or without a decimal point, and with an optional exponent of ten (indicated by E or e). If a decimal point is used, there must be at least one digit on either side of the decimal point. Examples are 23, 1.23, 1E-40 and 2.3e55.

    Real
    Species Real contains real numbers, stored approximately. The precision of operations is controlled by the content of box precision!. If precision! holds k, then built-in operations are carried out to at least k decimal digits of precision. For example, to get 30 digits of precision, you should say 
        Assign precision! := 30.
    The precision starts at normalPrecision, which can be set on the command line. By default, normalPrecision is the amount of precision offered by the machine double-precision floating point arithmetic. You can use normalPrecision in your program.

    Real constants are written in decimal notation, with or without a decimal point, and with an optional exponent of ten (indicated by E or e). If a decimal point is used, there must be at least one digit on either side of the decimal point. Examples are 23, 1.23, 1E-40 and 2.3e+55.

    Complex
    Species Complex contains complex numbers, represented approximately. You must import math/complex.ast to use complex numbers.

    The precision of operations is controlled by the content of box precision!. If precision! holds k, then built-in operations are carried out to at least k decimal digits of precision.

    All of the operations that follow have a single polymorphic type. However, for clarity, in most cases several types are shown.

    Basic arithmetic operations

    The first type of each operation is the polymorphic type, provided math/math.ast has been imported. Below that are specific species.

    +
    : (SEMIGROUP`a, SEMIGROUP`a) -> SEMIGROUP`a
    : (Natural, Natural) -> Natural
    : (Integer, Integer) -> Integer
    : (Rational, Rational) -> Rational
    : (Real, Real) -> Real
    : (Complex, Complex) -> Complex

    n + m is the sum of n and m.

    Example: 2+3 = 5

    - (binary)
    : (GROUP`a, GROUP`a) -> GROUP`a
    : (Integer, Integer) -> Integer
    : (Rational, Rational) -> Rational
    : (Real, Real) -> Real
    : (Complex, Complex) -> Complex

    n - m is the difference of n and m.

    Example: 2-3 = -1

    - (unary)
    : GROUP`a -> GROUP`a
    : Integer -> Integer
    : Rational -> Rational
    : Real -> Real
    : Complex -> Complex

    -n is the negation of n.

    --
    : (REAL`a, REAL`a) -> REAL`a
    : (Natural, Natural) -> Natural
    : (Integer, Integer) -> Integer
    : (Rational, Rational) -> Rational
    : (Real, Real) -> Real
    : (Complex, Complex) -> Complex

    n -- m is |n-m|.

    Example: 5--2 = 3 and 2--5 = 3

    _monus_
    : (Natural, Natural) -> Natural

    n _monus_ m is max(0, n-m).

    Example: 5 _monus_ 2 = 3 and 2 _monus_ 5 = 0

    *
    : (SEMIRING`a, SEMIRING`a) -> SEMIRING`a
    : (Natural, Natural) -> Natural
    : (Integer, Integer) -> Integer
    : (Rational, Rational) -> Rational
    : (Real, Real) -> Real
    : (Complex, Complex) -> Complex

    n * m is the product of n and m.

    Example: 5 * 2 = 10

    / (binary)
    : (FIELD`a, FIELD`a) -> FIELD`a
    : (Rational, Rational) -> Rational
    : (Real, Real) -> Real
    : (Complex, Complex) -> Complex

    x / y is the quotient of x and y.

    Example: 5 / 2 = 2.5

    / (unary)
    : FIELD`a -> FIELD`a
    : Rational -> Rational
    : Real -> Real
    : Complex -> Complex

    /x is the reciprocal of x.

    Example: / 2 = 0.5

    // (binary)
    : (INTEGER`a, INTEGER`b) -> Rational
    n // m is the quotient of n and m. Here, n and m must be integers or natural numbers.

    Example: 2 // 4 = 0.5

    // (unary)
    : INTEGER`a -> Rational

    //n is the reciprocal of n. Here, n must be an integer or natural number.

    Example: // 2 = 0.5

    _mod_
    : (INTEGER`a, Natural) -> Natural
    : (Natural, Natural) -> Natural
    : (Integer, Natural) -> Natural

    n _mod_ m is the remainder when you divide n by m. See divide, below.

    Example: 53 _mod_ 10 = 3

    _div_
    : (INTEGER`a, Natural) -> INTEGER`a
    : (Natural, Natural) -> Natural
    : (Integer, Natural) -> Integer

    n _div_ m is the (natural number) quotient when you divide n by m. See divide, below.

    Example: 53 _div_ 10 = 5

    divide
    : (INTEGER`a, Natural) -> (INTEGER`a, Natural)
    : (Natural, Natural) -> (Natural, Natural)
    : (Integer, Natural) -> (Integer, Natural)

    divide(n,m) = (n _div_ m, n _mod_ m) = (q,r), where q and r satisfy 0 <= r < m and q*m + r = n.

    Example: divide(53, 10) = (5,3) and divide(-53, 10) = (-6, 7)

    ^
    : (SEMIRING`a, Natural) -> SEMIRING`a
    : (Natural, Natural) -> Natural
    : (Integer, Natural) -> Integer
    : (Rational, Natural) -> Rational
    : (Real, Natural) -> Real
    : (Complex, Natural) -> Real

    x ^ n is x to the power n. You must import math/math.ast to use this.

    Example: 10^3 = 1000

    ^/
    : (FIELD`a, Integer) -> FIELD`a
    : (Rational, Integer) -> Rational
    : (Real, Integer) -> Real
    : (Complex, Integer) -> Real

    x ^/ i is x to the power i. Here, i must be of species Integer. You must import math/math.ast to use this.

    Example: (2)^(-2) = 0.25

    ^~
    : (APPROX`a,APPROX`a) -> APPROX`a
    : (Real, Real) -> Real
    : (Complex, Complex) -> Complex

    x ^~ y is x to the power y. (More precisely it returns the principal value of x to the power y. For example, 4^~(0.5) = 2, not -2.) You must import math/math.ast to use this.

    Example: (-2.5)^2 = 6.25 and 4.0^0.5 = 2.0 (approximately)

    Conversion to and from strings

    These are special cases of more general polymorphic functions. For example, $ and $' have polymorphic type `a -> String.

    $
    : Natural -> String
    : Integer -> String

    $(n) is the decimal representation of n. If box putCommasInNumbers! holds true, then commas will be inserted every 3 digits.

    Example: $(14) = "14"

    If you had said 

        Assign putCommasInNumbers! := true.
    then you would find that $(1234567) = "1,234,567"

    $
    : Rational -> String

    $(x) is a string that shows x as a fraction. If box putCommasInNumbers! holds true, then commas will be inserted every 3 digits.

    Example: $(14.1) = "141/10". If the rational number is an integer, it is shown as an integer. So $(14.0) = "14".

    The fraction is always shown in reduced form. So $(6//4) = "3/2".

    $
    : Real -> String

    $(x) is a string that is a decimal representation of x. The number is shown in fixed point format for numbers that look good in that format, and in exponential format for very large of very small numbers. The number of digits is determined by the current precision, but trailing zeros will not be shown. If box putCommasInNumbers! holds true, then commas will be inserted every 3 digits.

    Example: $(14.10) = "14.1"

    $
    : Complex -> String

    $(z) is a string that shows z.

    Example: $(complex(2.2,4.0)) = "(2.2+i*4)"

    $'
    : Natural -> String
    : Integer -> String
    : Rational -> String
    : Real -> String

    $'(n) is similar to $(n), but commas are inserted every three digits (on the left side of the decimal point, if there is one.)

    Example: $'(140000) = "140,000"

    $$
    : (Natural, Natural) -> String
    : (Integer, Natural) -> String
    : (Rational, Natural) -> String

    n $$ k is similar to $(n), but is padded on the left with blanks if necessary to make k characters. Of $(n) has more than k characters, then there is no padding, and no characters are removed.

    Example:
    14 $$ 4 = "   14"
    14 $$ 1 = "14"
    14.1:Rational $$ 8 = "   141/10"

    $$
    : (Real, Natural) -> String

    x $$ k is a string that shows the value of x, in exponential format, using a total of k characters. The exponent is three digits, but will use more if necessary. The exponent is adjusted so that there is one digit is to the left of the decimal point.

    Example: 14.12 $$ 12 = " 1.4120E+001"

    $$
    : (Complex, Natural) -> String

    z $$ k is a string that shows complex number z, but uses k characters for each of the real and imaginary parts.

    Example: complex(2.2,4.0) $$ 12 = "( 2.2000E+000+ i* 4.0000E+000)"

    $$'
    : (Natural, Natural) -> String
    : (Integer, Natural) -> String
    : (Rational, Natural) -> String
    : (Real, Natural) -> String
    : (Complex, Natural) -> String

    x $$' k is the same as x $$ k, but the number has commas every three digits from the right. The commas are counted in the number of characters.

    Example: 1000 $$' 6 = " 1,000"

    $$
    : (Rational, Natural, Natural) -> String
    : (Real, Natural, Natural) -> String

    x $$ (n,k) is the (approximate) decimal representation of x, using at least n total characters, with k characters to the right of the decimal place.

    Example: 14.1 $$ (8,2) = "    14.10"

    $$
    : (Complex, Natural, Natural) -> String

    z $$ (n,k) is the (approximate) decimal representation of z, using at least n total characters, with k characters to the right of the decimal place, for each of the real and imaginary parts.

    Example: complex(2.2,4.0) $$ (8,2) = "(     2.20+ i*     4.00)"

    $$'
    : (Rational, Natural, Natural) -> String
    : (Real, Natural, Natural) -> String
    : (Complex, Natural, Natural) -> String

    x $$' (n,k) is the similar to x $$ (n,k), but the number has commas every three digits from the right. Parameter n is the total number of characters to use, including commas.

    Example: 1234567.1234 $$' (15,4) = " 1,234,567.1234"

    $$
    : (Rational, Natural, Natural, Natural) -> String
    : (Real, Natural, Natural, Natural) -> String

    x $$ (n,k,e) is the (approximate) decimal representation of x, in exponential format, using at least n total characters, with k characters to the right of the decimal place, and using e characters for the exponent (not including the sign or the E).

    Example: 14.1 $$ (12,2,3) = "    1.41E+001"

    $$
    : (Complex, Natural, Natural, Natural) -> String

    z $$ (n,k,e) is the (approximate) decimal representation of z, in exponential format, using at least n total characters, with k characters to the right of the decimal place, and using e characters for the exponent, for each of the real and imaginary parts.

    Example: 14.1 $$ (12,2,3) = "(    2.20E+001+ i*    4.00E+000)"

    $$'
    : (Rational, Natural, Natural, Natural) -> String
    : (Real, Natural, Natural, Natural) -> String
    : (Complex, Natural, Natural, Natural) -> String

    x $$' (n,k,e) is identical to x $$ (n,k,e).

    naturalToHex
    : (Natural, Natural) -> String

    naturalToHex(n, k) is the hexadecimal representation of n, in at least k characters, padded with blanks on the left if necessary. If more then k digits are necessary to write the number, more will be used.

    Example: naturalToHex(26, 0) = "1A"

    naturalToBinary
    : (Natural, Natural) -> String

    naturalToBinary(n, k) is the binary representation of n, in at least k characters, padded with blanks on the left if necessary. If more than digits are necessary to write the number, more will be used.

    Example: naturalToBinary(26, 0) = "11010"

    stringToNatural
    : String -> Natural

    stringToNatural(s) is the number that decimal numeral x represents.

    Example: stringToNatural("14") = 14

    stringToInteger
    : String -> Integer

    stringToInteger(s) is the number that decimal numeral x represents.

    Example: stringToInteger("14") = 14

    stringToRational
    : String -> Rational

    stringToRational(s) is the number that decimal numeral x represents.

    Example: stringToRational("14.1") = 14.1

    stringToReal
    : String -> Real

    stringToReal(s) is the number that decimal numeral x represents.

    Example: stringToReal("14.1") = 14.1

    hexToNatural
    : String -> Natural

    hexToNatural(s) is the number that hexadecimal numeral x represents.

    Example: hexToNatural("1A") = 26

    binaryToNatural
    : String -> Natural

    binaryToNatural(s) is the number that binary numeral x represents.

    Example: octalToNatural("11010") = 26

    pull
    : String -> (Natural,String)
    : String -> (Integer,String)
    : String -> (Rational, String)
    : String -> (Real, String)

    pull(s) extracts a decimal number from the front of string s and returns an ordered pair (n,r) where n is the number and r is the suffix of s that follows the number. For example, pull("23 345") = (23, " 345") and pull("123.5E10 abcd") = (123.5E10, " abcd").

    Any initial white space is skipped over.

    Extract
    A convenient way to get a number from a string is to put the string in a box and used Extract. For example, if box b contains "999 abc" then 
      Extract [$(?n)] from b.
    binds n to 999 and leaves box b holding " abc".

    Mathematical operations

    You must import math/math.ast to use the following.

    cos
    : APPROX`a -> APPROX`a
    : Real -> Real
    : Complex -> Complex

    cos(x) is the (approximate) cosine of x, where x is in radians.

    exp
    : APPROX`a -> APPROX`a
    : Real -> Real
    : Complex -> Complex

    exp(x) is (approximately) e to the x.

    invtan
    : Real -> Real

    invtan(x) is (approximately) the inverse tangent of x (in radians).

    ln
    : APPROX`a -> APPROX`a
    : Real -> Real
    : Complex -> Complex

    ln(x) is (approximately) the natural logarithm of x.

    log
    : APPROX`a -> APPROX`a -> APPROX`a
    : Real -> Real -> Real
    : Complex -> Complex -> Complex

    log b x is (approximately) the logarithm to base b of x.

    magnitude
    : Complex -> Real

    magnitude(z) is (approximately) the magnitude of z.

    pi
    : () -> Real

    pi() is (approximately) pi.

    sin
    : APPROX`a -> APPROX`a
    : Real -> Real
    : Complex -> Complex

    sin(x) is the (approximate) sine of x where x is in radians.

    sqrt
    : APPROX`a -> APPROX`a
    : Real -> Real
    : Complex -> Complex

    sqrt(x) is (approximately) the (principal) square root of x.

    tan
    : APPROX`a -> APPROX`a
    : Real -> Real
    : Complex -> Complex

    tan(x) is the (approximate) tangent of x, where x is in radians.

    Miscelleneous operations

    ~
    : REAL`a -> Real

    ~(x) is the result of converting x to species Real. Here, x can be of any standard numeric species. If x already has species Real, then x is rounded to the current precision.

    Example: ~15 = 15.0

    abs
    : RRING`a -> RRING`a
    : Integer -> Integer
    : Rational -> Rational
    : Real -> Real

    abs(x) is the absolute value of x.

    Example: abs(-15) = 15

    absCeiling
    : RFIELD`a -> Natural
    : Rational -> Natural
    : Real -> Natural

    absCeiling(x) is the absolute value of the ceiling of x. See ceiling, below.

    absFloor
    : RFIELD`a -> Natural
    : Rational -> Natural
    : Real -> Natural

    absFloor(x) is the absolute value of the floor of x. See floor, below.

    ceiling
    : RFIELD`a -> Integer
    : Rational -> Integer
    : Real -> Integer

    ceiling(x) is the smallest integer that is not smaller than x.

    Example: ceiling(-15.8) = -15

    complex
    : (Real, Real) -> Complex

    complex(r,s) is complex number with real part r and imaginary part s.

    complex
    : REAL`a -> Complex
    : Complex -> Complex

    complex(x) is the number x converted to species Complex. This operation is overloaded.

    denominator
    : Rational -> Natural

    denominator(x) is the denominator of the reduced form of fraction x. The denominator is of species Natural.

    Example: denominator(2//3) = 3

    factors
    : Natural -> [Natural]

    factors(n) is list of the prime factors of n, in nondescending order. The product of the numbers in this list is n. To use this, you must import math/prime.ast

    floor
    : RFIELD`a -> Integer
    : Rational -> Integer
    : Real -> Integer

    floor(x) is the largest integer that is not larger than x.

    Example: floor(-15.8) = -16

    fromPolar
    : (Real, Real) -> Complex

    fromPolar(r,theta) is z where z = r*e^(i*theta).

    gcd
    : (INTEGER`a,INTEGER`a) -> INTEGER`a
    : (Natural, Natural) -> Natural
    : (Integer, Integer) -> Integer

    gcd(m,n) is the greatest common divisor of m and n.

    Example: gcd(15, 35) = 5

    im
    : Complex -> Real

    im(z) is imaginary part of z.

    natconst
    : Natural -> NUMBER`a
    : Natural -> Natural
    : Natural -> Integer
    : Natural -> Rational
    : Natural -> Real
    : Natural -> Complex

    Function natconst allows you to use constants such as 1 as various kinds of numbers.

    numerator
    : Rational -> Integer

    numerator(x) is the numerator of the reduced form of fraction x. It has species Integer.

    Example: numerator(2//3) = 2

    odd?
    : INTEGER`a -> Boolean
    : Natural -> Boolean
    : Integer -> Boolean

    odd?(n) is true if n is odd.

    Example: odd?(12) = false

    prime?
    : Natural -> Boolean

    prime?(n) is true just when n is prime. To use this function, you must import math/prime.ast

    ratconst
    : Rational -> NFIELD`a
    : Rational -> Rational
    : Rational -> Real
    : Rational -> Complex

    Function ratconst allows you to use constants such as 1.0 as various kinds of numbers.

    re
    : Complex -> Real

    re(z) is real part of z.

    round
    : RFIELD`a -> Integer
    : Rational -> Integer
    : Real -> Integer

    round(x) the (or an) integer closest to x.

    Example: round(12.6) = 13

    sign
    : RRING`a -> Integer
    : Integer -> Integer
    : Rational -> Integer
    : Real -> Integer

    sign(n) is -1 if n < 0, 0 if n = 0, and 1 if n > 0.

    toPolar
    : Complex -> (Real, Real)

    toPolar(z) is (r,theta) where z = r*e^(i*theta).

    toSqPolar
    : Complex -> (Real, Real)

    toSqPolar(z) is (r,theta) where z = sqrt(r)*e^(i*theta).

    unit
    : SEMIRING`a
    : Natural
    : Integer
    : Rational
    : Real
    : Complex

    unit = 1.

    zero
    : SEMIGROUP`a
    : Natural
    : Integer
    : Rational
    : Real
    : Complex

    zero = 0.

    zero?
    : SEMIGROUP`a -> Boolean
    : Natural -> Boolean
    : Integer -> Boolean
    : Rational -> Boolean
    : Real -> Boolean
    : Complex -> Boolean

    zero?(n) is true if n = 0.

    Example: zero?(12) = false
