    • Contexts

    Suppose that you want to do several operations on a particular data structure called aDataStructure. For example, the data structure might contain boxes called a, b and c, and you might want to alter the contents of those boxes. You can write 
        Assign a(aDataStructure) := ...
    Assign b(aDataStructure) := ...
    Assign c(aDataStructure) := ...
    There is a provision to allow you to abbreviate this to 
        Context C aDataStructure =>
      Assign my a := ...
      Assign my b := ...
      Assign my c := ...
    %Context
    where context C tells how the contextual information (a, b and c) is determined.

    Creating contexts

    To create a context called C, use a declaration of the form 
        Context C 
      case ...
      case ...
    %Context

    Each case defines a context and tells how its value is determined.

    case x => e
    Inside an expression of the form 
       Context C t =>
     ...
   %Context
    replace my x by expression e, with expression target replaced by t. For example, if context Grape is defined by 
       Context Grape
     case skin => getskin(target)
     case pulp => extractPulp(target)
   %Context
    then expression 
       Context Grape seedless =>
     my skin + my pulp
   %Context
    is an abbreviation for 
       getskin(seedless) + extractPulp(seedless)

    case bring x, y
    This is the same as 
       case x => x target
   case y => y target
    you can define any number of cases in a single case of this form.

    Adding new cases to a context

    You can add new cases to an existing context by putting the word extension before the name of the context. For example, 
       Context extension Grape
     case bring stem
   %Context
    adds a new case to existing context Grape.

    Context inheritance

    You can copy all cases from another context into a context when the context is created. Use 
       Context Merlot isKindOf Grape
     case ...
   %Context
    to put all cases that are part of context Grape into context Merlot. Only those cases that are part of Grape at the time that point where Merlot is created are copied. Subsequent additions to Grape will not be added to Merlot.

    You can use more than one isKindOf phrase in the definition of a context. For example, 

       Context Merlot isKindOf Grape isKindOf Fruit
     case ...
   %Context
    copies cases from Grape and Fruit into context Merlot.

    You can add inherited contexts to an existing context. Declaration 

      Context extension Merlot isKindOf Wine.
    adds all cases of context Wine to existing context Merlot.

    Be careful not to do too much of this. There can only be one case for each identifier in a context, except in the case where the translations are identical.

    Context substitution

    Context substitution is done only for an identifier preceded by the reserved word my or by my- without any intervening space.

    When you write an expression of the form 

       Context C t =>
   ...
   %Context
    expression t is only computed once. If it involves any computation, then its value is given a name, and that name is used inside the expression. Also, each context value is only computed once, outside the expression. Using the context Grape from above, expression 
       Context Grape getGrape("cabernet") =>
     Let a = my skin + my skin.
     Value my pulp + a.
  %Context
    is an abbreviation for 
       (Let g = getGrape("cabernet").
    Let s = getskin(g).
    Let p = extractPulp(g).
    Let a = s + s.
    Value p + a.
   )
    where names g, s and p are chosen as new names that are not in use. You can also write 
       Context Grape getGrape("cabernet") =>
     Let a = my-skin + my-skin.
     Value my-pulp + a.
  %Context
    using the optional hyphenated form.

    Procedures and operators in contexts

    A context case can define a procedure or operator name. You use expression My P as a procedure, and my op as an operator of the same form as op. The optional hyphenated form can also be used. If you hyphenate a symbolic identifier, the hyphen is not part of the identifier. For example, the subtraction operator can be written my - or my--.

    For example, suppose context Cherry is defined as follows. 

       Context Cherry
     case Print => printCherry(target)
     case +     => addCherry(target)
   %Context
    Then expression 
       Context Cherry ch =>
     My Print
        z  my-+  n
     %My Print
   %Context
    is an abbreviation for 
       (Let t1 = printCherry(ch).
    Let t2 = addCherry(ch).
    (t1(t2(z,n))))
    You can just as well write 
       Context Cherry ch =>
     My Print
        z  my +  n
     %My Print
   %Context

    Context in definitions

    A context can be any identifier, even one that is being used for other purposes. Typically, a context is the name of a data abstraction or species.

    In the special case where context C is a data abstraction or species, you can write 

      Define Ask C a => b %Ask =  v %Define
    to abbreviate 
      Define Ask (?this _also_ a) => b %Ask =  
    Context C this =>
      v
    %Context  
  %Define
    where identifier this is a newly chosen name that is not in use for anything else. The same abbreviation can be used with Let.

    For example, if Grape is a species as well as a context, then 

      Define Ask Grape ?g => peel(). =
    Remove my skin.
  %Define
    abbreviates 
      Define Ask (?this _also_ ?g) => peel(). =
    Context Grape this =>
      Remove my skin.
    %Context
  %Define
    Often, the contextual information is all you want about a parameter, as is the case for the peel operation. (Identifier g is not used.) In that case, use an anonymous pattern variable. For example, write 
      Define Ask Grape ? => peel(). =
    Remove my skin.
  %Define
