Reading Files Using Pattern Matching

  • Getting the content of a file
  • Pattern matching in strings
  • Performing extracts in pattern matches
  • More general pattern matches
  • Example
    • This page assumes that you know the basics of Astarte. It uses pattern matching and contains an example that uses a loop. You should understand lists and strings.

    Getting the content of a file

    To read a file, you get the content of the file, as a string. Function infile does that. infile(fname) is the content of the file whose name is fname. fname is a string that gives a path to the file, either from the current directory or in absolute form. You can use either / or \ to separate directories.

    Function infile is lazy. It does not read anything from the file until something is needed. If you look at a few characters, a block of characters will be read, not necessarily the entire file.

    Function infile is also forgetful. The garbage collector will occasionally remove anything that it has read. If you need the same data again, the file will be reread. (The garbage collector must run a few times without the data being used before this will happen.) If you have a file that is expected to change while your program runs, this kind of thing will not work. For a file that might change, you must tell infile that the file is volatile. Use expression infile [volatileMode] fname to get the content of such a file.

    Pattern matching in strings

    You will need to import package collect/string.ast to use the features described here.

    Typically, you want to extract parts of the string that you get from a file. One way to do that is with pattern matching. First, put the string into a box. For example, to prepare to read from file "myfile", write 

          Var in! := infile("myfile").
    (The exclamation point is just part of the name, in!, of this box. By convention, names of boxes end on an exclamation point.) Now write 
          Extract[p1,p2,...,pn] from in!.
    to match patterns p1, p2, ..., pn successively against parts of the string that you have put into box in!. The prefix of that string that matched those patterns is removed from the string in box in!, so that subsequent matches will match what is left.

    Typically, you use $ in patterns to do the match. For example, to read two natural numbers from the front of the file, you might write 

          Assume n,m: Natural.
      Extract[$(?n), $(?m)] from in!.
    If you had put string " 23 456 999 222" in box in!, then this would bind n to 23 and m to 456, and would change box in! to hold " 999 222". Notice that this is similar to how you would write n and m, using Writeln. The only difference is the space between the values. For example, you might write 
          Writeln[$(n), " ", $(m)].

    If it is clear from context what kind of thing you are trying to read, then there is no need for the assumes. But to be sure that you get the right kind of thing, you might want to state what you want. For example, to read a list of integers, and call it x, you would write 

          Assume x: [Integer].
      Extract[$(?x)] from in!.
    If you try to read a string using a $ pattern, the pattern matcher will look for a string with double-quote marks around it. This is consistent with the fact that, if s is a string, $(s) puts quotes around s. If you want to read a string delimited by white-space, use pattern theString instead of $. For example, 
          Assume s: String.
      Extract[theString(?s)] from in!.
    first skips any leading white space, then consumes a string up to the next white space character, or to the end of what is available.

    There are many more patterns that you can use. For example, to get a line, use pattern theLine(?s). To get n characters, use pattern theN n ?s. So to read 5 characters, you might write 

          Assume s: String.
      Extract[theN 5 (?s)] from in!.
    You can use extract inside match expressions. Writing 
          Match extract[theN 5 (?s)] = in!.
    is equivalent to writing 
          Extract[theN 5 (?s)] from in!.
    More patterns are described in
  • Pattern matching in lists (for matching on general lists),
  • Reading from strings (for matching in strings, which are lists of characters).

    • Performing extracts in pattern matches

    Sometimes you want to do an extract in a context where you are only required to write a pattern. To do that, just write extract, in lower case. Pattern match 
          Match extract[p] = t.
    is equivalent to 
          Extract[p] from t.

    More general pattern matches

    You can use ++ in patterns to break apart a string (or other list). For example, 
          Match ?a ++ [3,4] ++ ?b = [1,2,3,4,5,6,7].
    solves equation
    a ++ [3,4] ++ b = [1,2,3,4,5,6,7]
    binding a = [1,2] and b = [5,6,7]. In general, all possible solutions are returned, using a stream.

    Example

    Here is function that takes a file name fname and a string s, and prints each line of file fname that contains string s as a substring. 
          Let PrintLines ?fname, ?s. =
        Var f! := infile fname.

        Loop first matching f!

          %%-------------------------------------
          %% Try to extract a line from the file.
          %%-------------------------------------

          loopcase extract[theLine(?thisLine)] =>

            %%-------------------------------------------------------
	    %% If we got a line, then check whether s is a substring.
	    %% If it is, then print the line.
            %%-------------------------------------------------------

	    Try
              Match ? ++ s ++ ? = thisLine.
            then
              Writeln[thisLine].
            %Try
 
            continue

          exitcase else => ()
        %Loop
      %Let
    Now you might add an execute to a package with this function, to make the package an application. Suppose, for simplicity, that you add 
          Execute
        PrintLines commandLine#3, commandLine#2.
      %Execute
    and that you put the entire thing into a file called grep.ast. Then running 
          astr grep case grep.ast
    prints the following. 
              loopcase extract[theLine(?line_l)] =>
          exitcase else => ()
