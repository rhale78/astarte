    • Tables

    A table stores keys and associated values. Two kinds of tables are provided: sparse tables, which are represented by binary search trees, and linear tables, which are represented by lists. Species SparseTable(K,V) is used for sparse tables whose keys have species K and whose values have species V. Species LinTable(K,V) is similar, but for tables represented as lists.

    Families SparseTable and LinTable are both members of community TABLE. Type TABLE`a(K,V) is a general kind of table that stores keys of species K and values of species V. Tables are described further in collect/table.ast.

    To use sparse tables, import collect/sparstbl.ast. To use linear tables, import collect/lintbl.ast. The following are abbreviated descriptions. See the containing file for details.

    Empty tables

    emptyTable
    : TABLE`*t(EQ`a,`b)

    emptyTable is an empty table. (collect/table.ast)

    emptySparseTable
    : SparseTable(ORDER`a,`b)

    emptySparseTable is an empty sparse table. (collect/sparsetbl.ast)

    emptyLinTable
    : LinTable(EQ`a,`b)

    emptyLinTable is an empty linear table. (collect/lintbl.ast)

    emptyTable?
    TABLE`t(EQ`a,`b)

    emptyTable?(t) is true if t is an empty table. (collect/table.ast)

    Conversion to and from lists and sets

    table
    : [(EQ`a,`b)] -> TABLE`*t(EQ`a,`b)

    table([(k1,v1), (k2,v2), ...]) is a table holding the given (key,value) pairs. (collect/table.ast)

    Example: table[("lion", "mammal"), ("lizard", "reptile")] produces a table with keys "lion" and "lizard", and associated values "mammal" and "reptile".

    sparseTable
    : [(ORDER`a,`b)] -> SparseTable(ORDER`a,`b)

    sparseTable(x) is the same as table(x), but it is sure to create a sparse table. (collect/sparstbl.ast)

    linTable
    : [(EQ`a,`b)] -> SparseTable(EQ`a,`b)

    linTable(x) is the same as table(x), but it is sure to create a linear table. (collect/lintbl.ast)

    listTable
    : [(EQ`a,`b)] -> TABLE`*t(EQ`a,[`b])

    listTable takes a list of ordered pairs, but associates with each key a list of the values that are given with that key in the list of pairs. (collect/table.ast)

    Example:

    listTable[("lion","mammal"), ("lion","cat"), ("lizard","reptile")]

    produces the same table as

    table[("lion",["mammal","cat"]), ("lizard",["reptile"])].

    toList
    : TABLE`t(EQ`a,`b) -> [(EQ`a,`b)]

    toList(t) produces a list of the (key,value) pairs in table t. (collect/table.ast)

    Example:

    toList(table[("lion","mammal"),("lizard","reptile")])

    = [("lion","mammal"),("lizard","reptile")].

    eagerToList
    : TABLE`t(EQ`a,`b) -> [(EQ`a,`b)]

    eagerToList(t) is similar to toList(t), but is eager, where toList is not necessarily eager.

    Looking in tables

    _at_
    : (TABLE`t(EQ`a,`b), EQ`a) -> `b

    t _at_ k is the value associated with key k in table t. (collect/table.ast)

    Example:

    table[("lion", "mammal"), ("lizard", "reptile")] _at_ "lion" = "mammal".

    _defaultAt_
    : (TABLE`t(EQ`a,`b), (EQ`a, `b)) -> `b

    t _defaultAt_ (k,v) is the value associated with key k in table t, if there is such a value. Otherwise, t _defaultAt_ (k,v) is v. (collect/table.ast)

    Examples:

    table[("lion", "mammal"), ("lizard", "reptile")] _defaultAt_ ("lion","cat") = "mammal".

    table[("lion", "mammal"), ("lizard", "reptile")] _defaultAt_ ("snake","legless") = "legless".

    _in_
    : ((EQ`a,EQ`b), TABLE`t(EQ`a,EQ`b)) -> Boolean

    (k,v) _in_ t is true if (key,value) pair (k,v) is in table t. (collect/table.ast)

    Examples:

    ("lion","mammal") _in_ table[("lion", "mammal"), ("lizard", "reptile")] is true.

    ("lion","cat") _in_ table[("lion", "mammal"), ("lizard", "reptile")]) is false.

    _keyMember?_
    : (EQ`a, TABLE`t(EQ`a,`b)) -> Boolean

    k _keyMember?_ t is true if k is a key in table t. (collect/table.ast)

    Examples:

    "lion" _keyMember?_ table[("lion", "mammal"), ("lizard", "reptile")] is true.

    "mammal" _keyMember?_ table[("lion", "mammal"), ("lizard", "reptile")] is false.

    memberWithKey
    : (EQ`a, TABLE`t(EQ`a,`b)) -> (EQ`a,`b)

    memberWithKey(k,t) is the (key,value) pair in table t that has key k. (collect/table.ast)

    Example:

    memberWithKey(table[("lion", "mammal"), ("lizard", "reptile")], "lion") = ("lion","mammal").

    tableKeys
    : TABLE`t(EQ`a,`b) -> [EQ`a]

    tableKeys(t) is a list of the keys in table t. (collect/table.ast)

    Example:

    tableKeys(table[("lion", "mammal"), ("lizard", "reptile")]) = ["lion", "lizard"].

    tableVals
    : TABLE`t(EQ`a,`b) -> [`b]

    tableKeys(t) is a list of the keys in table t. (collect/table.ast)

    Example:

    tableVals(table[("lion", "mammal"), ("lizard", "reptile")]) = ["mammal", "reptile"].

    Modifying tables

    +/
    : (TABLE`t(EQ`a,`b), (EQ`a, `b)) -> TABLE`t(EQ`a,`b)

    t +/ (k,v) is the result of adding (key,value) pair (k,v) to table t. However, if there is already a pair with key k, then the old pair is kept, and the result is the same as t. (collect/table.ast)

    Examples:

    table[("lion", "mammal"), ("lizard", "reptile")] +/ ("snake","reptile")

    = table[("lion", "mammal"), ("lizard", "reptile"), ("snake","reptile")].

    table[("lion", "mammal"), ("lizard", "reptile")] +/ ("lion","cat")

    = table[("lion", "mammal"), ("lizard", "reptile")].

    You can use +/ in patterns. Pattern ?t +/ (k,?v) matches a table tbl that contains key k. It binds v to the associated value, and t to the result of removing pair (k,v) from tbl.

    Pattern ?t +/ (?k,?v) binds (k,v) in all possible ways, in a stream, to the pairs in the target table, and binds t to the result of removing pair (k,v) from that table.

    +//
    : (TABLE`t(EQ`a,`b), (EQ`a, `b)) -> TABLE`t(EQ`a,`b)

    t +// (k,v) is the result of adding (key,value) pair (k,v) to table t. If there is already a pair with key k, then the new pair (k,v) replaces the old pair. (collect/table.ast)

    Examples:

    table[("lion", "mammal"), ("lizard", "reptile")] +// ("snake","reptile")

    = table[("lion", "mammal"), ("lizard", "reptile"), ("snake","reptile")].

    table[("lion", "mammal"), ("lizard", "reptile")] +// ("lion","cat")

    = table[("lion", "cat"), ("lizard", "reptile")].

    update
    : (TABLE`t(EQ`a,`b), (EQ`a, `b), `b -> `b) -> TABLE`t(EQ`a,`b)

    update(t,(k,v),f) returns the result of inserting pair (k,v) into table t, if key k is not present in t. If key k is already present in t, with associated value x, then the returned table associates f(x) with key k.

    _minusKey_
    : (TABLE`t(EQ`a,`b), EQ`a) -> TABLE`t(EQ`a,`b)

    t _minusKey_ k is the result of deleting any pair with key k from table t. If there is none, then the result is just t itself. (collect/table.ast)

    Example:

    table[("lion", "mammal"), ("lizard", "reptile")] _minusKey_ "lizard"

    = table[("lion", "mammal")].

    mapContent
    : (`b -> `c) -> TABLE`t(EQ`a,`b) -> TABLE`s(EQ`a,`c)

    mapContent f t is the table that results by replacing each content value v in t by f(v). (collect/table.ast)

    Example:

    mapContent tail (table[("lion", "mammal"), ("lizard", "reptile")])

    = [("lion", "ammal"), ("lizard","eptile")].

    Memoizing functions

    memoize
    : TABLE`t(EQ`a,`b) -> (EQ`a -> `b) -> EQ`a -> `b

    (memoize tbl f) is a function g that behaves like function f, except that each time a value g(x) is computed, pair (x, f(x)) is entered into a table. If g(x) is computed again with the same argument, the result will be looked up in the table, and function f will not be used.

    The intial table of values is tbl. Use emptyTable as an initial table to start with no entries. (collect/table.ast)

    sparseMemoize
    : (EQ`a -> `b) -> (EQ`a -> `b)

    (sparseMemoize f) = (memoize emptySparseTable f). (collect/sparstbl.ast)

    linMemoize
    : (EQ`a -> `b) -> (EQ`a -> `b)

    (linMemoize f) = (memoize emptyLinTable f). (collect/lintbl.ast)

    Miscellaneous operations

    size
    : TABLE`t(EQ`a,`b) -> Natural

    size(t) is the number of entries in table t. (collect/table.ast)

    Example:

    size(table[("lion", "mammal"), ("lizard", "reptile")]) = 2.

    extractMin
    : TABLE`t(ORDER`a,`b) -> ((ORDER`a,`b), TABLE`t(ORDER`a,`b))

    extractMin(t) = ((k,v),t') where (k,v) is the (key,value) pair in table t that has the smallest key, and t' is the result of deleting that pair from t. (collect/table.ast)

    Example:

    extractMin(table[("lion", "mammal"), ("lizard", "reptile")])

    = (("lion","mammal"), table[("lizard","reptile")]).

    extractMax
    : TABLE`t(ORDER`a,`b) -> ((ORDER`a,`b), TABLE`t(ORDER`a,`b))

    extractMax(t) = ((k,v),t') where (k,v) is the (key,value) pair in table t that has the largest key, and t' is the result of deleting that pair from t. (collect/table.ast)

    Example:

    extractMax(table[("lion", "mammal"), ("lizard", "reptile")])

    = (("lizard","reptile"), table[("lion","mammal")]).

    min
    : TABLE`t(ORDER`a,`b) -> (ORDER`a,`b)

    min(t) is the entry in table t with the smallest key. For example, min(table[(2,5),(3,4)]) = (2,5).

    max
    : TABLE`t(ORDER`a,`b) -> (ORDER`a,`b)

    max(t) is the entry in table t with the largest key. For example, max(table[(2,5),(3,4)]) = (3,4).

    Exceptions

    tableX
    : ExceptionSpecies

    When table operations fail, they generally fail with exception tableX. Operations that can fail are memberWithKey and _at_ (no such member); min, max, extractMin and extractMax (empty table).
