    • Modular numbers

    Species Modular contains pairs (n,m), where n is an integer and m is a modulus. The pair (n,m) is written (n _modulo_ m). Operations between values with the same modulus m are allowed, and they perform their arithmetic modulo m. If the moduli are different, then the operation fails with exception modulusX.

    Operations on modular numbers are as follows. You must import math/modular.ast to use species Modular. Some details that are not mentioned here can be found there.

    Basic arithmetic operations

    +
    : (Modular, Modular) -> Modular

    x + y is the sum of x and y.

    Example: (3 _modulo_ 5) + (4 _modulo_ 5) = 2 _modulo_ 5

    - (binary)
    : (Modular, Modular) -> Modular

    x - y is the difference of x and y.

    Example: (3 _modulo_ 5) - (4 _modulo_ 5) = 4 _modulo_ 5

    - (unary)
    : Modular -> Modular

    -x is the negation of x.

    Example: -(3 _modulo_ 5) = 2 _modulo_ 5

    *
    : (Modular, Modular) -> Modular

    x * y is the product of x and y.

    Example: (3 _modulo_ 5) * (4 _modulo_ 5) = 2 _modulo_ 5

    / (unary)
    : Modular -> Modular

    /x is the reciprocal of x. If x = (a _modulo_ m), then the reciprocal of x is a number y such that x*y = 1 _modulo_ m. The reciprocal only exists when gcd(a,m) = 1. When the reciprocal does not exist, the reciprocal operations fails. The exception is domainX("divide by zero") when taking the reciprocal of 0, and is domainX("no inverse") otherwise.

    Example: /(2 _modulo_ 5) = 3 _modulo_ 5

    / (binary)
    : (Modular, Modular) -> Modular

    x / y is the quotient of x and y. That is, it is x times the reciprocal of y. See the previous entry.

    ^
    : (Modular, Natural) -> Modular

    x ^ n is x to the power n. Here, n must be a natural number.

    Example: (3 _modulo_ 5)^2 = 4 _modulo_ 5

    ^/
    : (Modular, Integer) -> Modular

    x ^/ y is x to the power y. Here, y must be an integer. x^/(-y) is the reciprocal of x^/y.

    Example: (3 _modulo_ 5)^/(-2) = 4 _modulo_ 5

    Conversion to string

    $
    : Modular -> String

    $(x) is a string that is a decimal representation of x.

    Example: $(4 _modulo_ 5) = "(4 mod 5)"

    $'
    : Modular -> String

    $'(x) is the same as $(x), but the numbers have commas every three digits from the right.

    $$
    : (Modular, Natural) -> String

    x $$ n is a string that is a decimal representation of x, using at least n total characters.

    Example: (4 _modulo_ 5) $$ 11 = "  (4 mod 5)"

    $$'
    : (Modular, Natural) -> String

    x $$' n is the same as x $$ n, but the numbers have commas every three digits from the right.

    Unknown moduli

    It is allowed to create a modular number with modulus 0. This indicates an unknown modulus. When you perform an operation on a value with an unknown modulus and a value with a known modulus, the known modulus is taken. For example, (2 _modulo_ 0) + (3 _modulo_ 7) = (5 _modulo_ 7)

    You can only use an unknown modulus in situations where it is either possible to perform operations without knowing the modulus or where it is possible for the modulus to be determined. For example, you can add two values with unknown modulus ((2 _modulo_ 0) (3 _modulo_ 0) = (5 _modulo_ 0)) but you cannot divide them, since division requires knowing the modulus. Operations that cannot be performed due to an unknown modulus fail with exception modulusX.

    The remainder is stored as a natural number. Consequently, it is not allowed to have a negative number with an unknown modulus.

    Miscelleneous operations

    _as_
    : (INTEGER`a, Modular) -> Modular

    (n _as_ x) = (n _modulo_ (modulus x)).

    _modulo_
    : (INTEGER`a, Natural) -> Modular

    n _modulo_ m is number n with modulus m. The modulus m can be 0, indicating an unknown modulus, provided n is not negative.

    modulus
    : Modular -> Natural

    modulus(x) is the modulus of x.

    Example: modulus(3 _modulo_ 5) = 5

    remainder
    : Modular -> Natural

    remainder(x) is the numeric part of x, with the modulus removed. It is always a natural number that is less than the modulus of x.

    Example:
    remainder(3 _modulo_ 5) = 3
    remainder(-1 _modulo_ 5) = 4

    unit
    : Modular

    unit = (1 _modulo_ 0).

    zero
    : Modular

    zero = (0 _modulo_ 0).

    zero?
    : Modular -> Boolean

    zero?(x) is true if (the remainder of) x is 0.

    Example: zero?(12 _modulo_ 4) = true

    Exceptions

    modulusX
    Binary operations such as addition require values with the same modulus. When the moduli differ, these operations fail with exception modulusX.
