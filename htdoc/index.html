Astarte Reference Manual

Note: This documentation is also available from csci.ecu.edu/astarte/.

Astarte is a multiparadigm programming language. It supports multithreaded functional, logic, procedural and object-oriented programming paradigms, but of those it does functional programming best.

This is the top level to the Astarte reference manual. Astarte is under development by Karl Abrahamson (karl@cs.ecu.edu), and any comments about it or this documentation should be sent to him.

This manual is written for the typical undergraduate computer science student who has an understanding of data structures. Some features are necessarily more advanced than that, but most of what is here is accessible to such a student.

This manual is under construction, and if you navigate through it you might encounter missing pages. I apologize for them. They will be added over time.

You can use this manual either as a reference (using links, the table of contents and the index) or you can read through it using the next and prev links at the top and bottom of each page.

The following color codes are generally used in this manual.

  • black: normal text
  • brown: advanced text that you can probably skip
  • magenta: Astarte examples and code
  • dark orange: Examples that are incorrect

Using the implementation

Astarte is currently implemented for Unix. The implementation is written in C. At ECU, the Astarte implementation is available on the Unix machines in Austin 320. You can also get a copy for your own use, provided you have a computer that is running Unix. See "Installing Astarte", below, for installing your own copy.
