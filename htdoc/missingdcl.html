D6. Missing Declarations

  • Declaring definitions missing
  • Strongly missing types
  • Using definitions that are missing

    • Declaring definitions missing

    Occasionally you make an expectation that will have holes in it that you do not intend to fill. If you just fail to make the definitions, then you will get a warning that definitions are missing. You can tell the compiler to suppress those warnings using missing declarations.

    Declaration 

          Missing x: T.
    indicates that the definition of x might not be defined for species that belong to polymorphic type T. You are allowed to define them, but you will not receive any warnings about x not being defined for any type in T.

    Strongly missing types

    Declaration 

          Missing{strong} x: T.
    indicates that no declarations of x should be given for any species in polymorphic type T. If any are given, the compiler will warn you.

    Using definitions that are missing

    If you use an identifier of a species that has been declared missing, the compiler will look for a definition with that species. If it does not find one, it will warn you that you are using a missing definition.

    If you are doing a polymorphic definition, the compiler will warn you if any of the types would lead to use of a definition that has been declared missing, and that the compiler cannot see.

    You can suppress warnings about missing definitions being used by using modifier missing on the offending identifier. For example, use 

        Let x = (missing f)(y).
    to suppress warnings about f being missing.
