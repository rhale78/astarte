    • Unknowns

    An unknown is a value that has not yet been determined. It can occur in a data structure, representing a part of the data structure that has not been found.

    An unknown is different from a "lazy value", or promise, that is created as a result of lazy evaluation. There is no expression held with an unknown that tells how to compute its value. Instead, the value of an unknown is bound by the program, in a way that is similar to the binding of the content of a box.

    Like boxes, there are two flavors of unknowns: shared and nonshared. Bindings done to a shared unknown are visible to all threads. Bindings done to a nonshared unknown are only visible to the thread that does the binding, and to its child threads.

    There are two important differences between boxes and unknowns.

    1. An unknown can only be bound once. After it is bound, the unknown disappears: the value is no longer considered unknown, and the former presence of the unknown is not visible to the program. (But see nonshared unknowns and threads.)

    2. You never explicitly ask for the content of an unknown. A bound unknown has disappeared.

    There are two varieties of unknowns: protected and unprotected. An unprotected unknown can be bound by anyone. A protected unknown is created along with a key, and only somebody who holds the key for a given protected unknown can bind that unknown.

    There is no separate species of unknowns. An unknown can have any species, and any value can potentially be an unknown. If a thread tries to get the value of an unknown, then that thread is suspended until the unknown becomes bound. Be careful about that. If some other thread does not bind the unknown, then all threads that are waiting for that unknown are dead.

    Some functions, called active functions, are designed to work with unprotected unknowns. An active function tries to prevent itself from waiting for the value of an unknown, and also tries to prevent itself from failing. An active function that returns a Boolean result generally tries to make itself return true if possible.

    For example, activeHead(x) will check whether x is an unprotected unknown. If it is, activeHead will bind it to a list consisting of an unknown head and an unknown tail, and will return the unknown head. Evaluating activeNil?(x) on an unbound unprotected unknown x will bind that unknown to [], and return true.

    The following sections describe operations for use with unknowns. More detail is available in logic/unknown.ast.

    Creating unknowns

    UnknownKey
    UnknownKey is the species of a key for a protected unknown.

    newProtectedUnknown
    : BoxFlavor -> (`a, UnknownKey)

    newProtectedUnknown(flav) returns a pair (u,k) where u is a new unbound, protected unknown of flavor flav, and k is its key.

    newUnknown
    : BoxFlavor -> `a

    newUnkown(flav) produces a new (unprotected) unknown of flavor flav (either shared or nonshared).

    nonsharedUnknown
    : () -> `a

    nonsharedUnknown() returns a new unbound, nonshared, unprotected unknown.

    protectedNonsharedUnknown
    : () -> `a

    protectedNonsharedUnknown() returns a pair (u,k) where u is a new unbound, nonshared, protected unknown and k is its key.

    protectedSharedUnknown
    : () -> (`a, UnknownKey)

    protectedSharedUnknown() returns a pair (u,k) where u is a new unbound, shared, protected unknown and k is its key.

    sharedUnknown
    : () -> `a

    sharedUnknown() returns a new unbound, shared, unprotected unknown.

    unknown
    : () -> `a

    unknown() returns a new unbound, nonshared, unprotected unknown. You can use this function in patterns; unknown() matches any unknown at all.

    Var{unknown}
    Var{logic}
    Statement 
      Var{unknown} x.
    creates a new unprotected shared unknown and binds x to it. You can force a nonshared unknown by using the nonshared option, as in Var{nonshared,unknown} x. Statement 
      Var{logic} x.
    is equivalent to Var{nonshared,unknown} x.

    With one exception, all forms of var-expression can be used with unknowns. For example, you can create arrays of unknowns. The exception is that forms that use initialization cannot be used. It is not sensible to produce initialized unknowns, since then they are not unknown at all.

    Classifying, recognizing and comparing unknowns

    protectedUnknown?
    : `a -> Boolean

    protectedUnknown?(x) returns true just when x is an unbound protected unknown.

    sameUnknown?
    (`a,`a) -> Boolean

    sameUnknown?(x,y) is true if x and y are the same unbound unknown. Here, x and y must have the same species.

    sameUnknown??
    : (`a,`b) -> Boolean

    sameUnknown?? is the same as sameUnknown?, but it does not require its two parameters to have the same species.

    unknown?
    :`a -> Boolean

    unknown?(x) is true if x is any kind of unbound unknown.

    unprotectedUnknown?
    : `a -> Boolean

    unprotectedUnknown?(x) returns true just when x is an unbound unprotected unknown.

    Explicitly binding unknowns

    BindProtectedUnknown
    : ((`a,UnknownKey), `a) -> ()

    Statement 

      BindProtectedUnknown (x,k) := y.
    binds protected unknown x to y. x must be an unbound protected unknown and k must be the key for x, otherwise BindProtectedUnknown fails with exception bindUnknownX.

    BindUnknown
    : (`a,`a) -> ()

    Statement 

      BindUnknown x := y.
    binds unprotected unknown x to y. x must be an unbound unprotected unknown, otherwise BindUnknown fails with exception bindUnknownX.

    You can bind one unknown to another. The two become the same unknown, so that subsequent binding of one implicitly bind the other as well. If you try to bind an unknown to itself, nothing happens. The unknown is left unknown.

    It is possible to create cyclic structures by binding unknowns. For example, if you create a structure x that contains an unknown u, and you bind u to x, then you have a cyclic structure. Be careful about that.

    Active functions (which implicitly bind unknowns)

    Active functions behave like passive ones, but they try to avoid failing by binding unknowns where possible. For example, activeHead?(x) tries to take the head of x. If x is an unknown, then it normally would not be possible to take the head of x until x becomes bound. To avoid waiting, and also to avoid failure, activeHead(x) will bind x to a nonempty list with an unknown head and an unknown tail, and return the unknown head.

    activeHead
    : [`a] -> `a

    activeHead(x) is the head of x, but it will bind x to a nonempty list if x is an unprotected unknown.

    activeLazyHead
    : [`a] -> `a

    activeLazyHead(x) is similar to lazyHead. It produces the head of x, but it will bind x to a nonempty list if x is an unprotected unknown. This binding will done only be done when the promise created by activeLazyHead is forced.

    activeLazyTail
    : [`a] -> [`a]

    activeLazyTail(x) is similar to lazyTail. It produces the tail of x, but it will bind x to a nonempty list if x is an unprotected unknown. This binding will done only be done when the promise created by activeLazyTail is forced.

    activeListToPair
    : [`a] -> (`a, [`a])

    activeListToPair(x) is the pair (head(x), tail(x)), but it will bind x to a nonempty list if x is an unprotected unknown.

    activeNil?
    : [`a] -> Boolean

    activeNil?(x) is the true if x is [] and false otherwise, but it will bind x to [] if x is an unprotected unknown. (It tries to make itself return true.)

    activeTail
    : [`a] -> [`a]

    activeTail(x) is the tail of x, but it will bind x to a nonempty list if x is an unprotected unknown.

    Creating lists of unknowns

    The following functions are available if you import collect/boxes.ast. Alternatively, use var-expressions.

    infiniteUnknownArray
    : BoxFlavor -> [`a]

    infiniteUnknownArray(flav) is an infinite list of unbound, unprotected unknowns of flavor flav (either shared or nonshared).

    unknownArray
    : BoxFlavor -> Natural -> [`a]

    (unknownArray flav n) is a list of n unbound, unprotected unknowns of flavor flav (either shared or nonshared).
