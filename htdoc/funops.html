L5. Operations on Functions

This page describes some fairly generic operations on functions.

|
x | f = f(x). The bar is not a identifier, it is a special symbol, and can only be used as a binary operator.

||
: (`a -> `b, `b -> `c) -> (`a -> `c)

(f || g)(x) = g(f(x)).

__
: ((`a,`b) -> `c) -> `b -> `a -> `c

(__ f x) y = f(y,x). This symbol is two consecutive underscores. This can be used with binary operators. For example, __ + 2 is a function that, when run on parameter x, produces x + 2.

__/-
: (`a -> `b -> `c) -> (`b,`a) -> `c

__/- is the functional inverse of __. It takes a function that takes its parameters one at a time, and converts it to a function that takes its parameters as an ordered pair. However, it transposes the ordered pair. That is, (__/- f)(x,y) = f y x.

_bar_
: (`a, `a -> `b) -> `b

x _bar_ f = f(x). Function _bar_ differs from | in that _bar_ is a genuine function. | is part of the basic syntax of the language, and is not a function.

constf
: `a -> `b -> `a

(constf  k) is the function whose value on every parameter is k. That is, (constf  k)(x) = k.

curry
: ((`a,`b) -> `c) -> `a -> `b -> `c

If f is a function that takes an ordered pair as a parameter, then (curry f) is a function that takes its parameters one at a time. So (curry f) x y = f(x,y).

idf
: `a -> `a

idf is the identity function (idf(x) = x). It is the identity for function composition, and so is useful with higher-order functions.

_is_
: (`a,`a -> Boolean) -> Boolean

x _is_ f is the same as f(x), but it requires that f be a predicate. That is, f must be a function whose codomain is Boolean.

lazy
: (`a -> `b) -> (`a -> `b)

(lazy f) is the same function as f, but it is lazy. So evaluating (lazy f x) will not run f until its result is needed.

_pmap_
(`a -> `b, `c -> `d) -> (`a,`c) -> (`b,`d)

(f _pmap_ g)(x,y) = (f x, g y).

Example: (left _pmap_ right)((2,3), (4,5)) = (2,5).

_o_
:(`b -> `c, `a -> `b) -> (`a -> `c)

(f _o_ g)(x) = f(g(x)).

_o3_
: (`c -> `d, `b -> `c, `a -> `b) -> `a -> `d

(f _o3_ (g,h) )(x) = f(g(h(x))).

shortApply
: (`a -> `b, `a) -> `b)

shortApply(f,a) = f(a). It attempts to evaluate f(a) immediately, but if computation of f(a) takes longer than a preset time limit, that computation is cut off, and it becomes lazy.

succeedsf
: (`a -> `b) -> `a -> Boolean

(succeedsf f x) is true if f(x) does not fail.

_tee_
: (`a -> `b, `a -> `c) -> (`b,`c)

(f _tee_ g)(x) = (f x, g x).

Example: If double(x) = 2*x and triple(x) = 3*x then (double _tee_ triple)(8) = (16,24).

uncurry
: (`a -> `b -> `c) -> (`a,`b) -> `c

uncurry is the functional inverse of curry. It takes a function that takes parameters one at a time, and converts it to a function that takes an ordered pair. So (uncurry f)(x,y) = f x y.

xposef
:((`a,`b) -> `c) -> ((`b,`a) -> `c)

(xposef  f)(x,y) = f(y,x).
