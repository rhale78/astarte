    • Species of first class patterns

    Because patterns are handled by inline expansion, you cannot treat a pattern or pattern function as a value. For example, it is not possible to pass a pattern to a function, and let that function use it in a pattern match.

    To fix this problem, there are some parameterized species that can be used for almost that purpose. The species are as follows. You need to import pattern/pattern.ast to use them.

    PatternFunction(`a -> `b)
    A value of species PatternFunction(A -> B) can be used in a pattern as a function of species A -> B, provided the operator * is applied to it. For example, if x has species A, t has species B and f has species PatternFunction(A -> B), then 
        Match *f(?x) = t.
    is well-typed. It performs a match against target t, according to what function f does, and binds x.

    PatternConstant(`a)
    A value of species PatternConstant(A) can be used in a pattern as a constant of species A, provided operator * is applied to it. For example, if c has species PatternConstant(A) and t has species A, then 
        Match c = t.
    is well-typed. It performs a match according to the definition of c.

    ListPatternFunction(`a -> [`b])
    A value of species ListPatternFunction(A -> [B]) can be used as a function in a list pattern match, such as is done in Extract statements, provided operator * is applied to it. If x has species A, t has species <:[B]:> and f has species ListPatternFunction(A -> [B]), then 
        Extract *f(?x) from t.
    is well-typed. It performs a match against target t, according to what function f does, and binds x. Pattern match 
        Match *f(?x) ++ ?rest = tt.
    is also well-typed, provided tt has species [B], since the same mechanism is used for list patterns as for Extract.

    ListPatternConstant([`a])
    A value of species ListPatternConstant([A]) can be used as a constant in a list pattern match, such as is done in Extract statements, provided operator * is applied to it. If t has species <:[A]:> and c has species ListPatternConstant([A]), then 
        Extract *c from t.
    is well-typed. It performs a match against target t, according to what c does. Pattern match 
        Match *c ++ ?rest = tt.
    is also well-typed, provided tt has species [A], since the same mechanism is used for list patterns as for Extract.

    Creating first class patterns

    Pattern functions and constants are created as follows.

    patternFunction
    : (`a -> `b) -> PatternFunction(`b -> `a)

    If p is patternFunction(f), then 

        Match *p(?x) = t.
    is equivalent to 
        Match ?x = f(t).
    If f has type A -> B, then patternFunction(f) has type PatternFunction(B -> A).

    patternConstant
    : (`a -> ()) -> PatternFunction(`a)

    If p is patternConstant(F), then 

        Match *p(?x) = t.
    is equivalent to 
        F(t).
    If F has type A -> (), then patternConstant(F) has type PatternConstant(A).

    listPatternFunction
    : ([`a] -> (`b,[`a]) -> ListPatternFunction(`b -> [`a])

    If p is listPatternFunction(f), then 

        Match *p(?x) ++ ?rest = t.
    is equivalent to 
        Match (?x,?rest) = f(t).
    Similarly, statement 
        Extract *p(?x) from bx.
    is equivalent to 
        Match (?x,?rst) = f(@bx).
    Assign bx := rst.
    If f has type [A] -> (B,[A]), then listPatternFunction(f) has type ListPatternFunction(B -> [A]).

    listPatternConstant
    : ([`a] -> [`a]) -> ListPatternConstant([`a])

    If p is listPatternConstant(f), then 

        Match *p ++ ?rest = t.
    is equivalent to 
        Match ?rest = f(t).
    Similarly, statement 
        Extract *p from bx.
    is equivalent to 
        Assign bx := f(@bx).
    If f has type [A] -> [A], then listPatternFunction(f) has type ListPatternFunction([A]).

    First class patterns in the library

    The following are available by importing pattern/pattern.ast

    const_P
    : EQ`a -> PatternConstant(EQ`a)

    Pattern *(const_P c) is a pattern constant that matches only c.

    constList_P
    : [EQ`a] -> ListPatternConstant([EQ`a])

    Pattern *(constList_P c) is a list pattern constant that matches only c.

    span_P
    : COLLECTION`c(EQ`a) -> ListPatternFunction([EQ`a] -> [EQ`a])

    Pattern *(span_P c) ?x has the same effect as pattern span c ?x. It is a list pattern function that matches a maximal prefix of the target list that contains only members of collection c. For example, *(span_P "ab") ?s matches a prefix that contains only a's and b's, and binds s to that prefix.

    spanOthers_P
    : COLLECTION`c(EQ`a) -> ListPatternFunction([EQ`a] -> [EQ`a])

    Pattern *(spanOthers_P c) ?x has the same effect as pattern spanOthers c ?x. It is a list pattern function that matches a maximal prefix of the target list that contains only nonmembers of collection c. For example, *(span_P "ab") ?s matches a prefix that contains only characters other than a and b, and binds s to that prefix.

    theN_P
    : Natural -> ListPatternFunction([`a] -> [`a])

    Pattern *(theN_P n) ?x has the same effect as pattern (theN n) ?x. It is a list pattern function that matches exactly n members of the target list. It requires that the list contain at least n members.

    theUpToN_P
    : Natural -> ListPatternFunction([`a] -> [`a])

    Pattern *(theUpToN_P n) ?x has the same effect as pattern (theUpToN n) ?x.

    The following are available by importing pattern/stringp.ast

    dollar_P
    : ListPatternFunction(`a -> String)

    Pattern *dollar_P(?x) has the same effect as pattern $(?x).

    hex_P
    : ListPatternFunction(Natural -> String)

    Pattern *hex_P(?x) has the same effect as pattern hex(?x).

    white_P
    : ListPatternConstant(String)

    Pattern *white_P has the same effect as pattern white.

    theWS_P
    : ListPatternFunction(String -> String)

    Pattern *theWS_P(?x) has the same effect as pattern theWS(?x).

    theLine_P
    : ListPatternFunction(String -> String)

    Pattern *theLine_P(?x) has the same effect as pattern theLine(?x).

    theString_P
    : ListPatternFunction(String -> String)

    Pattern *theString_P(?x) has the same effect as pattern theString(?x).

    theId_P
    : ListPatternFunction(`a -> String)

    Pattern *theId_P(?x) has the same effect as pattern theId(?x).

    theBalanced_P
    : ListPatternFunction(String -> String)

    Pattern *theBalanced_P(?x) has the same effect as pattern theBalanced(?x).
