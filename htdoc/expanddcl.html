D16. Expand Declarations

  • Expand declarations
  • Suppressing expansion

    • Expand declarations

    Expand declarations are used to request inline expansions. Declaration 
        Expand double(?x) => x + x.
    indicates that the compiler should replace expression double(4) by expression 4+4.

    Rules for expand declarations are similar to rules for pattern declarations. However, there is no target, and the expansion does not define the pattern identifiers that occur on the left-hand side of =>.

    Expansions are only done when they do not restrict the polymorphism of a declaration.

    Suppressing expansion

    You can suppress expansion of a function by applying function noExpand to it. For example, the following expansion is defined for function max. 
          Expand max[?a,?b] => If a > b then a else b %If.
    To force the max function to be used, without being replaced by a conditional, you can write 
          Let x = (noExpand max)[y,z].
