E4. Sequencing and ()

  • Statements

  • The bar operator

    • Statements

    The unit species () has only one member, also called (). Value () is thought of as a void value, indicating no interesting result. A statement is a special case of an expression whose type is ().

    Performing actions in sequence

    If S is a statement and e is an expression, then expression S e indicates sequencing. First, statement S is evaluated, and then e is evaluated. The value of S e is the value of e.

    For example, statement Let x = 1 %Let makes x another name for the constant 1. Expression 

          Let x = 1.
      (x + x)
    has value 2.

    You can sequence together many statements. For example, expression 

          Let x = 2.
      Let y = x + x.
      (y*y)
    has value 16. Typically, when sequencing is used to give an expression whose type is not (), you use Value to indicate the final expression. Expression 
          Let x = 2.
      Let y = x + x.
      Value y*y.
    is exactly the same as the above expression.

    You can use () by itself as a null statement. It performs no action. For example, 

          If x == 0 then
        Assign y := 0.
      %If
    is defined to be the same as 
          If x == 0 then
        Assign y := 0.
      else 
        ()
      %If

    The bar operator

    Expression a | b has the same meaning as b a. This allows you to write a definition after it is used. For example, you can write a function that doubles its parameter as follows. 
         Define double(?x) = y |
       Let y = 2*x.
     %Define
    This has the same meaning as 
         Define double(?x) =
       Let y = 2*x.
       y
     %Define

    Ignoring the value of an expression

    Occasionally you want to compute the value of an expression only for the side effects that are obtained. Procedure Ignore can be used to do that. Statement 
         Ignore e.
    computes e and has value (), regardless of the value of e.

    Sequencing and lazy evaluation

    It is possible to create a value of species () that is not fully evaluated. If such a value is used in sequencing, it is not evaluated at all. See sequencing and lazy evaluation.
