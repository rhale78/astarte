    • Data tags

    In a typed language, data items can be thought of as carrying tags with them. An integer has a tag saying that it is an integer, a string says that it is a string, and so on.

    Astarte uses a somewhat unusual way of attaching species to data items. Each data item has two tags attached to it: the intrinsic tag and the extrinsic tag. Having two tags provides an extra level of flexibility.

    The intrinsic tag tells the species that an item thinks that it belongs to. An integer, for example, has intrinsic species Integer.

    The extrinsic tag tells the species that the compiler or interpreter assigns to a data item. This is typically the same as the intrinsic tag, but it can differ. A data item whose extrinsic and intrinsic tags differ can be thought of as gift wrapped. The label on the outside of the gift is the extrinsic tag, telling the item's externally visible species. Inside the gift package is the item itself, holding its intrinsic tag.

    Every data item has exactly one species. The species that it belongs to is its extrinsic species.

    Primary and secondary species

    There are two kinds of species, primary and secondary. The species that have been described so far are all primary. A data item of a primary species always has an intrinsic tag that is identical to its extrinsic tag; the system and the item agree on the species of the item.

    There is a secondary species for each data abstraction. For example, ORDER is a genus, and is also a secondary species. A data item can have an intrinsic tag of Integer, but an extrinsic tag of ORDER. This means that the system has forgotten that the item is an integer, and is only remembering that it is of a species that belongs to ORDER. Think of such an item as an integer wrapped in a package covered with wrapping paper that has ORDER written all over it. Until the package is opened, all you see is that the gift is something of the general kind ORDER. It is only after removing the wrapping that you see an integer.

    The intrinsic tag of an item is always a primary species. The extrinsic tag is either primary or secondary.

    Examples of uses for secondary species

    Astarte has a monomorphism restriction: polymorphism does not extend into data items. Each function that your program creates, for example, has a single species for its domain and a single species for its codomain.

    Secondary species provide a way around the monomorphism restriction. For example, suppose that you want to create a box that can hold any item at all. You cannot quite do that, but what you can do is create a box of species <:ANY:>, using the secondary species ANY. To put an item into the box, you must gift wrap it, giving it an extrinsic tag of ANY. When you get the content of the box, you can leave it gift wrapped, or you can unwrap it, to see what item was inside the gift wrapping.

    Secondary species also allow for lists of mixed species. Suppose, For example, that you create a list of species [EQ]. The items in this list are gift wrapped, with extrinsic tag EQ. Inside the gift wrapping is an item of a species that belongs to genus EQ.

    Suppose that you want to create a single function that will print any data item at all. Using name polymorphism, you can write 

        Let Print ?x. = 
      Writeln[$x].
    %Let
    yielding function Print: `a -> (). But this does not really create just one item, it creates infinitely many different items with the same name. Truly polymorphic functions can be created using secondary species. To create a single print function, you define a function of type ANY -> (). The definition of this function is give below.

    Creating items that belong to secondary species

    Notation <<x>> creates an item whose intrinsic species is the same as the intrinsic species of x, and whose extrinsic species is some secondary species, with the species determined by type inference. For example, 
          Let w2 = <<2:Natural>>:REAL.
    defines identifier w2 to name an item whose value is the number 2, whose extrinsic species is REAL and whose intrinsic species is Natural. If you now define 
          Let ww = <<w2>>:ANY.
    then ww is also the number 2, with intrinsic species Natural, but with extrinsic species ANY. What you have done is rewrapped the gift using new wrapping paper. Operator <<...>> is called the wrap operator, and is implemented by a primitive function called wrap.

    Removing the wrapping

    You can remove the wrapping by using <<...>> in a pattern. For example, here is a function of species ANY -> () that prints an arbitrary item by removing the wrapping. 
        Let Print << ?x >>. =
      Writeln[$x].
    %Let
    Now Print can be passed to another function and generally manipulated without losing its polymorphism.

    Primary and secondary variables

    Normally, a type variable can stand for either a primary or secondary species. The following special forms allow you to select primary or secondary.

    G``a
    Variable G``a can only be a secondary species.

    G`*a
    Variable G`*a can only be a primary species.
