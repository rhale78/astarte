T18. Creating and Defining New Parameterized Data Abstractions

  • Creating new communities
  • Adding a family to a community
  • Providing descriptions

    • Creating new communities

    Each community is either transparent (holding only transparent families) or opaque (holding only opaque families). Declaration 
           Abstraction MyOpaqueCommunity(`a).
    creates a new opaque community called MyOpaqueCommunity. Initially, MyCommunity has nothing in it. Declaration 
           Abstraction{transparent} MyTransparentCommunity(`a).
    creates a new, initially empty transparent community called MyTransparentCommunity. Always use a name that starts with an upper case letter for a community.

    Adding a family to a community

    Declaration 
           Relate F isKindOf C.
    adds family F or community to community C. You can also add a community to another community when the it is created. Declaration 
           Abstraction QUEUE(`a) isKindOf SEQUENCE.
    abbreviates 
           Abstraction QUEUE(`a).
       Relate QUEUE isKindOf SEQUENCE.

    Providing descriptions

    You can provide a description of a community in the declaration that defines it. The description is a string constant, and is put after the name of the community. For example 
        Abstraction{transparent} SET(`a)
      %" Community SET contains various forms of sets.
    %Abstraction
