T5. Polymorphism

  • Examples of polymorphism
  • Kinds of polymorphism in programming languages
  • The nature of polymorphism in Astarte
    • Astarte allows for polymorphic definitions. The types of such definitions involve type variables, such as `a and `b. A type variable stands for an arbitrary species.

    Examples of polymorphism

    `a -> `a
    This is the type of a function that can have species T -> T for any species T. For example, the identity function f(x) = x has this polymorphic type.

    (`a,`b) -> (`b,`a)
    An example of a function of this polymorphic type is the function f(x,y) = (y,x). For example, if the parameter of f has species (String, Boolean), then its result has species (Boolean, String).

    (`a -> `b, `a) -> `b
    An example of a function of this polymorphic type is the fuction apply(f,x) = f(x). It takes an ordered pair (f,x) where f is a function and x is some item. It returns the result of applying function f to item x.

    For example, if f has species (Boolean -> String), then x must have species Boolean, and the result of apply(f,x) must have species String. So in this case, apply has the species (Boolean -> String, Boolean) -> String. Since apply is polymorphic, it can have any species that looks like (A->B, A) -> B.

    Kinds of polymorphism in programming languages

    There are generally two forms of polymorphism that are used in programming languages:

    1. true polymorphism, in which the data items themselves are polymorphic, and

    2. ad-hoc polymorphism, or overloading, in which it is the names that are polymorphic.

    For example, under true polymorphism, there is a single item called nil (the empty list) that can be any kind to list. That is, nil is a polymorphic object of type [`a]. Under overloading, identifier nil names infinitely many different items, one for each species. So there is an empty list of species [Boolean], an empty list of species [String], etc., and all of them have the same name nil.

    The nature of polymorphism in Astarte

    Most functional programming languages use true polymorphism. Object-oriented languages use overloading. Astarte is in between; it uses overloading, not true polymorphism, but it looks more like a functional language than an object-oriented language. This allows Astarte to provide the basics of both functional and object-oriented programming.

    So when you create a polymorphic function, what you are really doing is creating many different definitions of the same name. When you use a name with a given species, the system finds the appropriate definition for it.
