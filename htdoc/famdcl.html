T16. Defining New Families

  • How to define new families
  • Restrictions on family declarations
  • Adding families to abstractions

    • How to define new families

    Parameterized species are defined in a way that is almost identical to the way species are defined. The difference is that the definition of a family must provide a parameter for the family, usually a variable or variables. For example, the definition of family Optional is as follows. 
         Extension
       Species{transparent} Optional(`a)  = noValue | value(`a).
     %Extension
    By default, a new family is     opaque. Use mode transparent, as in the definition of Optional, to make it transparent.

    When you expect a family, you must say the parameter, and you must give the same form of parameter as in the definition. For example, to expect Optional, you would write 

         Extension
       Expect{transparent} species Optional(`a).
     %Extension

    Restrictions on family declarations

    You are not allowed to have a variable on the left-hand side of a function type in the definition of a transparent family. This is required due to the handling of constraints for transparent families. For example, 
         Extension
       Species{transparent} Bad(`a) = `a -> `a.
     %Extension
    is not allowed.

    Adding families to abstractions

    When you create a parameterized species, you can add it to one or more abstractions. Just write isKindOf A,B,C after the name and parameter of the species to add it to abstractions A, B and C. For example, 
        Species Queue(`a) isKindOf SEQUENCE = [`a].
    This just abbreviates 
        Species Queue(`a) = [`a].
    Relate Queue isKindOf SEQUENCE.
