    • Scheme

    This page describes some aspects of Scheme, by relating them to similar aspects of Astarte. It does not attempt to cover the entire Scheme language. See "The Revised(5) Report on the Algorithm Language Scheme," edited by William Clinger and Jonathan Rees, for a complete description of Scheme.

    The line between data and programs is really quite fuzzy in Scheme. For example, data and programs are both lists. When a list (a b c d) is evaluated, first a is evaluated, and its value must be a function. Then that function is applied to argument (b c d). You must be careful to distinguish an expression from a constant list, keeping in mind that the first member of an expression is a function that computes the result of the expression.

    You might want to look at a couple of examples before reading the details of Scheme.

    Boolean operations and conditionals

    The boolean constants are #f and #t. However, in most contexts, you can use any value at all for "true", with only #f counting as false. In some Scheme implementations, #f is the same thing as ().

    (boolean? x)
    (boolean? x) returns #t just when x is either #t or #f.

    (not x)
    not x

    Since any object other than #f counts as true, (not (list 2 3)) has value #f.

    (and x y z ...)
    x _and_ y _and_ z _and_ ... (almost)

    Actually, (and x y z ...) evaluates its parameters x, y, ... from left to right. It stops as soon as one of those expressions is found to have value #f, and returns #f. If all evaluate to values that are different from #f, the last one is returned.

    (or x y z ...)
    x _or_ y _or_ z _or_ ... (almost)

    (or x y z ...) returns the first of its parameters whose value is not #f, or returns #f if all of them are #f.

    (equal? x y)
    x == y

    Equality testing is messy in Scheme. You have a choice of three different equality testing predicates.

    1. (eq? x y) returns #t if x and y are the same number or the same symbol or are internally represented by pointers to the same memory location. For example, (eq? (cons a b) (cons a b)) returns #f, since the two calls to cons put their results at different memory addresses.

    2. (eqv? x y) is similar to (eq? x y), but sometimes returns #t where (eq? x y) returns #f.

    3. (equal? x y) is the best approximation to a true equality test. It deals with lists and strings by going into the structure, and recursively comparing the parts.

    (> x y)
    x > y

    Other tests include <, <= and >=.

    (if a b c)
    If a then b else c %If

    (cond (t1 v1) (t2 v2) ... (else vn)) 
    Choose
  case t1 => v1
  case t2 => v2
  ...
  else => vn
%Choose

    Each condition t1, t2, ... is examined on the assumption that #f means false and anything else means true.

    Lists

    ()
    []

    (null? x)
    nil?(x)

    (pair? x)
    not nil?(x)

    This tells whether x is a nonempty list.

    (list? x)
    This returns true if x is a list, which is either a pair or ().

    (list a b c)
    [a,b,c]

    (cons h t)
    h::t

    h.t
    h::t

    You must use caution with dot notation. '(2.3.()) evaluates to the list (2 3), because it is quoted. But (2.3.()) is the list (2 3). If you try to evaluate it, you will be using 2 as a function, which will cause an error. Use the cons function to do a :: operation in a program. Since Scheme is typeless, you can use dot notation and cons to put any two things into a pair. For example, 14.20 (or, in a program, (cons 14 20)) is a pair containing 14 and 20. The "tail" of this "list" is not a list.

    (car x)
    head(x)

    (cdr x)
    tail(x)

    cdr is pronounced could-er. The names cdr and car are derived from the assembly language of the computer on which Lisp was first implemented.

    (cadr x)
    head(tail(x))

    (cdar x)
    tail(head(x)

    (cddr x)
    tail(tail(x)).

    You can form names using up to four a's and d's. For example, (caddr x) is defined to be (car (cdr (cdr x))).

    (length x)
    length(x)

    (append x y z ...)
    x ++ y ++ z ++ ...

    (reverse x)
    reverse(x)

    (member x y)
    x _in_ y (almost)

    This returns the longest suffix of list y whose first member is x. For example (member a '(b c a d a f)) = (a d a f). If x does not occur in list y, then member returns #f.

    Typically, this is used as a predicate. The conditional tests treat anything except #f as true.

    In order to do a membership test, equality tests must be done. There are three different ways to do equality tests, so there are three different membership testing functions. Function member uses equal?. A similar function memq uses eq?, and function memv uses eqv?.

    (list-tail x k)
    drop k x

    (list-ref x k)
    x # k

    (assoc x y)
    assoc x y (almost)

    List y must be a list of pairs, where each pair is in the form of a list of length 2. (assoc x y) finds the first pair in the list whose car is x, and returns that pair. For example, 

        (assoc 'a '((b c) (a d) (e f)))
    returns (a d). If no pair has car x, then it returns #f. Assoc must perform equality tests, so there are three versions. Function assoc uses equal? to do the equality tests, assq uses eq? and assv uses eqv?.

    (map f x)
    map f x

    You can use any number of list parameters. (map f x y) is roughly equivalent to the Astarte expression map f (x _zip_ y).

    (for-each f x)
    MapProc f, x.

    This applies f to each member of list x for its side-effects. Like map, for-each can take any number of list parameters.

    Characters

    #\a
    'a'

    #\space
    ' '

    #\newline
    '\n'

    (char? x)
    (char? x) returns #t if x is a character, and #f otherwise.

    (char->integer c)
    rank(c)

    (integer->char n)
    char(n)

    (char-=? x y)
    x == y

    There are other comparisons char-<?, char->?, char-<=? and char->=? for comparing characters by alphabetical order.

    (char-ci=? x y)
    toUpper(x) == toUpper(y)

    There are other comparisons char-ci?, char-ci<=? and char-ci>=? for comparing characters by alphabetical order, ignoring case.

    (char-alphabetic? x)
    letter?(x)

    There are other classification functions char-numeric?, char-whitespace?, char-upper-case? and char-lower-case?.

    (char-upcase x)
    toUpper(x)

    (char-downcase x)
    toLower(x)

    Strings

    In Astarte, a string is a special kind of list, whose members are characters. In Scheme, a string is not a list at all, but is a different kind of data value.

    "abc"
    "abc"

    (string? x)
    (string? x) is #t if x is a string, and #f otherwise.

    (string->list x)
    (string->list x) is a list of the characters in string x.

    (list->string x)
    (list->string x) converts a list of characters into a string.

    (string c d e,...)
    [c,d,e,...]

    (string-length x)
    length(x)

    (string-ref x k)
    x # k

    (string-append x y z ...)
    x ++ y ++ z ++ ...

    (substring a b s)
    s ## (a, b--a)

    (string-=? x y)
    x == y

    There are other comparisons string-<?, string->?, string-<=? and string->=?.

    (string-ci=? x y)
    toUpper(x) == toUpper(y)

    There are other comparisons string-ci<?, string-ci>?, string-ci<=? and string-ci>=? that ignore case.

    Symbols

    goat
    #"goat"

    This is a symbol , and represents itself. Symbols are used for variable names, or anything else that needs a name. Keep in mind that the distinction between data and programs is weak in Scheme. So it is natural that a variable name would be a data object.

    (symbol? x)
    (symbol? x) returns true if x is a symbol.

    (symbol->string x)
    symbolName(x)

    (string->symbol x)
    #x (the symbol whose name is x)

    Numbers

    Implementations are free to restrict the range of numbers, but are encouraged not to do so.

    35
    35

    (number? x)
    (number? x) is true if x is a number. Other classification predicates are integer?, rational?, real? and complex?, although not all implementations provide all of these types of numbers.

    (odd? x)
    odd?(x)

    (zero? x)
    zero?(x)

    (positive? x)
    x > 0

    (negative? x)
    x < 0

    (+ a b c ...)
    a + b + c + ...

    (- a)
    -a

    (- a b)
    a - b

    (* a b c ...)
    a * b * c * ...

    (/ a)
    /a

    (/ a b)
    a / b

    (quotient a b)
    a _div_ b

    (remainder a b)
    a _mod_ b

    (abs x)
    abs(x)

    (gcd a b)
    gcd(a,b)

    (max a b c ...)
    max[a,b,c,...]

    (min a b c ...)
    min[a,b,c,...]

    (sqrt x)
    sqrt(x)

    Other math functions include exp, log (natural logarithm), sin, cos, tan, asin, acos and atan.

    (floor x)
    floor(x)

    (ceiling x)
    ceiling(x)

    (number->string x)
    $x

    (string->number x)
    This converts to a number of a type determined by the string. It returns #f is string does not look like a number.

    Functions

    (procedure? f)
    (procedure? f) returns #t if x is a function.

    (lambda (x) e)
    (?x => e)

    (lambda (x y) e)
    ((?x,?y) => e)

    (apply f args)
    f args

    This form is used when args is a list of arguments. If args is list (a b c), then (apply f args) is the same as (f a b c).

    (call-with-current-continuation f)
    This is used for Scheme's version of exception handling, but is more general than exception handling. The parameter f must be a function of one argument. call-with-current-continuation calls (f r) where r is a function that, when called as (r val), abandons the current computation and returns val from call-with-current-continuation. This is a way to cut off computation of f early, and get out of this part of the program.

    Function f is free to do anything at all with the function r that is passed to it.

    Defining functions and giving things names

    (define x e)
    Let x = e.

    (define (f x) e)
    Define f(?x) = e.

    This can be used for recursive or nonrecursive definitions, and with any number of parameters. For example, (define (f a b) e) defines a function of two parameters.

    (let ((x e)) v)
    CloseScope Let x = e. (v) %CloseScope

    This construct allows you to do several bindings whose scope extends only over a single expression. For example, (let ((x (+ 2 3)) (y 5)) (* x y)) produces value 25. The bindings are done at the same time; you are not allowed to use one binding in the definition of the next one.

    (let* ((x a) (y b)) v)
    This is similar to (let ((x a) (y b)) v), but you are allowed to use each definition in the next one. For example, it would be allowed to write (let* ((x 2) (y (* x 3))) (+ x y))).

    (letrec ((x a) (y b)) v)
    Neither let nor let* can be used for recursive definitions. (letrec ((x a) (y b)) v) can be used for recursive, or even mutually recursive definitions. So the definition of x can refer to y, and the definition of y can refer to x.

    (set! 'x e)
    Relet x = e.

    Note the quote. It is the symbol x that you want to set to e, and you must quote x. It you write (set x e), you will set the variable that is the value of variable x to e. (Here, x would have a value that is a symbol, and that symbol is the variable that gets set.)

    Delayed evaluation

    (delay e)
    (:e:)

    This creates a lazy value, called a promise. Implementations are allowed, but not required, to do forces implicitly. Typically, you must force all promises yourself. Expression (force d) forces promise d, and returns the result.

    Sequencing

    (begin a b c)
    a b c

    Here, a and b are performed, and the value of this expression is c. You can have any number of items in the list.

    Input and output

    (call-with-input-file s f)
    s is the name of a file that is to be opened for input, and f is a function with one parameter. An input port p is tied to file f, and (f p) is evaluated. The result returned from (f p) is also returned from call-with-input-file. The file is closed when (f p) returns.

    (call-with-output-file s f)
    s is the name of a file that is to be opened for output, and that should not currently exist. Parameter f is a function with one parameter. An output port p is tied to file s, and (f p) is evaluated. The result returned from (f p) is also returned from call-with-output-file. The file is closed when (f p) returns.

    (input-port? x)
    This returns true if x is an input port.

    (output-port? x)
    This returns true if x is an output port.

    (current-input-port)
    Returns the standard input port.

    (current-output-port)
    Returns the standard output port.

    (read p)
    Returns the next object read from port p, and consumes that object, so that the next read will get one after this one. The input is presumed to hold an object written in the syntax of Scheme.

    If an end-of-file is encountered before the end of the current object, an end-of-file object is returned.

    (read)
    Same as (read (current-input-port)).

    (read-char p)
    Returns the next character read from port p, and consumes it.

    (read-char)
    Same as (read-char (current-input-port)).

    (peek-char p)
    Returns the next character read from port p, but does not consume it.

    (peek-char)
    Same as (peek-char (current-input-port)).

    (eof-object? x)
    This returns true if x is an end-of-file object.

    (write x p)
    Writes object x onto port p in Scheme syntax.

    (write x)
    This is the same as (write x (current-output-port)).

    (display x p)
    Writes object x onto port p. If x is a character or a string, it is printed without quotes.

    (display x)
    This is the same as (display x (current-output-port)).

    (newline p)
    This prints a newline on port p.

    (newline)
    This is the same as (newline (current-output-port)).

    (write-char c p)
    Writes character c onto port p.

    (write-char c)
    This is the same as (write-char c (current-output-port)).

    Loading Scheme files and keeping transcripts

    (load filename) loads the Scheme file called filename (which must be a string). All of the definitions in that file are read. If any of them are for functions that are already defined, then the new definitions replace the old ones.

    (transcript-on filename) causes a transcript of your session to be written to file filename. (transcript-off) turns off the transcript.

    Examples

    Here is a definition of a function that concatenates two lists. It is similar to the standard function append, but allows only two parameters. 
        (define (cat x y)
       (if (null? x) y
          (cons (car x) (cat (cdr x) y))
       )
    )

    Here is a definition of the member function. (member x y) returns the suffix of y that begins with the first occurrence of x in y, or returns the object #f if x does not occur in y. 

        (define (member x y) 
      (cond
         ((null? y)           #f)
         ((equal? x (car y))  y)
         (else                (member x (cdr y)))
      )
    )
