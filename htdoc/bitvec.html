    • Bit vectors

    A bit vector is a string of 0's and 1's that has a right end, but is infinite to the left. Package collect/bitvec.ast implements operations on bit vectors.

    For clarity, we use notation such as ...01101 to show a bit vector, where the leftmost digit shown is replicated to the left. That notation is not available in Astarte though.

    A bit vector has type Bitvec.

    Creating bit vectors

    bitvec
    : Integer -> Bitvec
    : (Boolean, [Boolean]) -> Bitvec

    If k is an integer then bitvec(k) is the bit vector that is the infinite two's complement representation of integer k. For example, bitvec(2) = ...0010 and bitvec(-2) = ...1110.

    If b is a boolean value and x is a list of boolean values, then bitvec(b,x) is a bit vector made by adding infinitely copies of b to the left of x. (false is replaced by 0 and true by 1.) For example, bitvec(false, [true,true,true,false]) = ...001110.

    You can use both forms of the bitvec function in patterns.

    Conversion to and from strings

    $
    : Bitvec -> String

    $(v) is the hexadecimal representation of v. The first digit is 0 for a vector that has infinitely many 0's and is F for a vector that has infinitely many 1's.

    Example: $(...001001011) = "04B".

    bitvecToBinary
    : (Bitvec,Natural) -> String

    bitvecToBinary(v,k) is the binary representation of the rightmost k bits of bit vector v.

    Example: bitvecToBinary(...001001011, 10) = "000001001011".

    bitvecToHex
    : (Bitvec,Natural) -> String

    bitvecToHex(v,k) is the hexadecimal representation of bit vector v, using a total of k characters, so that it shows the rightmost 4k bits of v.

    Example: bitvecToHex(...001001011, 5) = "0004B".

    binaryToBitvec
    : String -> Bitvec

    binaryToBitvec takes a string representing a bit vector in base 2, and produces the bit vector. If the leftmost digit is 0, then the bit vector is padded on the left with infinitely many 0's. If the leftmost digit is 1, the bit vector is padded on the left with infinitely many 1's.

    Examples:
    binaryToBitvec("01101") = ...0001101
    binaryToBitvec("101010101") = ...11101010101.

    hexToBitvec
    : String -> Bitvec

    hexToBitvec takes a string representing a bit vector in base 16, and produces the bit vector. If the leftmost digit is 7 or less, then the bit vector is padded on the left with infinitely many 0's. If the leftmost digit is 8 or more, the bit vector is padded on the left with infinitely many 1's.

    Examples:
    hexToBitvec("02B") = ...00101011
    hexToBitvec("92B") = ....11100100101011

    The null bit vector

    bvNull
    : Bitvec

    bvNull is bitvec(0), the vector of all 0's.

    bvNull?
    : Bitvec -> Boolean

    bvNull?(v) is true just when v is bvNull.

    Destroying a bit vector

    unbitvec
    : Bitvec -> Integer
    : Bitvec -> (Boolean, [Boolean])

    unbitvec(v) is the integer k such that bitvec(k) = v, or is a pair (b,x) such that bitvec(b,x) = v, depending on the desired type of the result.

    Operations on bit vectors

    bvNot
    : Bitvec -> Bitvec

    bvNot(v) is the bit vector obtained by replacing each 0 by a 1 and each 1 by a 0 in v.

    Example: bvNot(...01100) = ...10011.

    _bvAnd_
    : (Bitvec, Bitvec) -> Bitvec

    u _bvAnd_ v is the bitwise 'and' of bit vectors u and v.

    Example: ...00101 _bvAnd_ ...01110 = ...00100.

    _bvOr_
    : (Bitvec, Bitvec) -> Bitvec

    u _bvOr_ v is the bitwise 'or' of bit vectors u and v.

    Example: ...00101 _bvOr_ ...01110 = ...01111.

    _bvXor_
    : (Bitvec, Bitvec) -> Bitvec

    u _bvXor_ v is the bitwise exclusive 'or' of bit vectors u and v.

    Example: ...00101 _bvXor_ ...01110 = ...01011.

    _bvShl_
    : (Bitvec, Natural) -> Bitvec

    v _bvShl_ n is the result of shifting bit vector v to the left n bits, pulling in zeroes on the right.

    Example: ...00101 _bvShl_ 2 = ...0010100.

    _bvShr_
    : (Bitvec, Natural) -> Bitvec

    v _bvShr_ n is the result of shifting bit vector v to the right n bits, dropping the rightmost n bits.

    Example: ...00101 _bvShr_ 2 = ...001.

    bvMask
    : Natural -> Bitvec

    bvMask n is a vector of n 1's, preceded by infinitely many zeroes.

    Example: bvMask 2 = ...00011.

    bvMin
    : Bitvec -> (Natural,Bitvec)

    bvMin v is a pair (n,u). The values of n and u are chosen depending on whether v has infinitely many 0's or infinitely many 1's.

    If v has infinitely many 0's, then n is the index of the rightmost 1 in vector v, where the rightmost bit has index 0, and u is the vector that results by setting bit n in v to 0.

    If v has infinitely many 1's, then n is the index of the rightmost 0 in v, and u is the result of setting bit n to 1 in v.

    Examples:

    bvMin (...0010100) = (2,...0010000),

    bvMin (...11010111) = (3, ...11011111).

    If there is no sought bit, then bvMin fails with exception domainX("no 1 bit").

    bvMax
    : Bitvec -> (Natural,Bitvec)

    bvMax is similar to bvMin, but finds the leftmost 1 or 0.

    Example: bvMax(...001001101000) = (9, ...000001101000).

    bvField
    : (Bitvec, Natural, Natural) -> Bitvec

    bvField(v,n,m) is obtained from v by shifting to the right n bits and then selecting only the rightmost m bits, padding with 0's on the left. So bvField(v,n,m) = bvShl(v,n) _bvAnd_ bfMask(m).

    bvSetField
    : (Bitvec, Natural, Natural, Bitvec) -> Bitvec

    bvField(v,n,m,x) is obtained from bit vector v by replacing the m bits at indices n to n+m-1 by the righmost m bits of bit vector x.

    Example: bvSetField(...0110010111, 2, 3, 000) = ...0110000011.

    bvSetBits
    : (Bitvec, [Natural], Boolean) -> Bitvec

    bvSetBits(v,x,b) is obtained from v by replacing the bits whose indices are given by list x to 1 if b is true and 0 if b is false. The rightmost bit has index 0.

    Example: bvSetBits(...01101000, [0,2,3], true) = ...01101101
