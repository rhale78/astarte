D9. Exception Declarations

  • Creating new exceptions
  • Getting an exception description

    • Creating new exceptions

    You can create new exceptions. Each is a new member of species ExceptionSpecies. The forms are as follows.

    Exception id descr %Exception
    Create a new exception called id. It is not trapped, by default. String constant descr is a description of the exception. When this exception is trapped, string descr is printed to explain what happened. The description can be omitted, and defaults to an empty string.

    Example: 

          Exception myX.
    creates a new exception called myX. It also creates a predicate myX? that tests for myX. That is, myX?(e) is true exactly when e is myX.

    Example: 

          Exception gremlinX 
        %" A gremlin has sabotaged the computation.
      %Exception

    Exception trap id descr %Exception
    This is similar to the preceding case, but the new exception is trapped, by default.

    Example: 

          Exception trap bananaX
        %" I have slipped on a banana peel,
        %" and can't get up.
      %Exception

    Exception id(T) descr %Exception
    Create a new family of exceptions. They not trapped, by default. The members of the family have an associated item of species T. You can create them using id(x), and destroy them using unid(x). There is also a predicate id? and a pattern function id.

    Example: 

          Exception myX(Natural).
    Use myX(3) to create an exception with associated value 3. You can get the associated value using unmyX(myX(3)) = 3, or by using pattern myX(?n). Predicate myX?(x) yields true if x is myX(k) for some k.

    Exception trap id(T) %Exception
    Similar to the preceding case, but the exceptions in the family are trapped by default.

    Getting an exception description

    You can get the description of an exception while a program runs using function exceptionDescription. For example, exceptionDescription(testX) = "A test failed".
