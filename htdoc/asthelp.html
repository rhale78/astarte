M2. Getting Function Descriptions

  • Standard and library functions and constructs
  • Options for asthelp
  • Getting your own descriptions

    • Standard and library functions and constructs

    The manual entries for functions and other things contain more detail than is discussed in this documentation. Occasionally, you might want to look at them to find out more. You can get the manual entry by reading the .ast file that contains it, or by using the asthelp command.

    Command asthelp prints the manual entry or entries of a name. To find out about function map, you can use command 

        asthelp map
    The output will show where map is defined and the manual entry for it. You can ask for help about general constructs as well as about functions, species, etc. For example, 
        asthelp Loop
    tells you about the Loop construct.

    Options for asthelp

    The following options are available. They should be given before the name whose manual entry is sought. For example, you might type 

        asthelp -i mapproc
    to get a manual entry for MapProc.
    -i
    Ignore the case of letters. Without this, case is significant, and a search for writeln will not find Writeln.

    -s
    Report information about any name that has the given string as a substring. For example, 
        asthelp -s ite
    will tell you about Writeln, as well as many other functions.

    Getting your own descriptions

    If you write descriptions in your program, you can get them using asthelp. For example, if you have a function called foo, and you write a description of that function, then command 
       asthelp foo
    will show that description.

    When you compile a program myprog.ast, the compiler generates a file called myprog.asd containing information about what is defined in myprog.ast, including the descriptions. The compiler also writes a file called index.asx that contains an index of everything in the current directory.

    The asthelp command searches the current directory and the standard directory for index.asx files.

    If you include a file called indexloc.txt in a directory, containing in it the name of another directory (relative to the current directory) then index information will be written in the index.asx file in that directory instead. This allows you to consolidate information from several directories into a single directory.
