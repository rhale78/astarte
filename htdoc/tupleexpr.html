    • Ordered pairs

    An ordered pair is a kind of data item holding two other items. The pair holding a and b (in that order) is written (a,b). The parentheses are required.

    Expression (a,b,c) abbreviates (a,(b,c)), expression (a,b,c,d) abbreviates (a,(b,(c,d))), etc.

    Examples 

       (1,2,3)

   ("types", (3,true), "can be mixed")

    The := operator

    Expression A := B abbreviates (A,B). It is generally used for functions that put results into boxes. For example, you can write 
          Assign (x,a).
    but this reads better as 
          Assign x := a.

    Getting the components of an ordered pair

    You can use function left to take the left-hand member of an ordered pair, and function right to take the right-hand member. For example, 
         left(3,4)          = 3
     right(3,4)         = 4
     left(3,4,5)        = 3
     right(3,4,5)       = (4,5)
     left(right(3,4,5)) = 4
    A more pleasant way to get the parts of an ordered pair is to use pattern matching. For example, 
         Match (?a,?b,?c) = (3,4,5).
    binds a = 3, b = 4 and c = 5.

    There are also lazy operations to extract the components of ordered pairs. Functions lazyLeft and lazyRight behave like left and right, respectively, but they do not evaluate their arguments until their value is requested.

    Order of evaluation

    The order of evaluation of an ordered pair is undefined. The compiler might choose to evaluate the left hand part first, or it might choose to evaluate the right hand component first.

    This does not mean that the order of expressions in the pair is not important. It only says that, in order to compute (a,b), you can compute a and b in either order. After you compute them, you must put the results together with a first and b second.
