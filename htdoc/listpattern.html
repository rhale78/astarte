LC3. Pattern Matching in Lists

  • Pattern forms that extract from boxes
  • Pattern forms for use with Extract
  • Forms that do not use boxes
  • Functions for removing prefixes and suffixes of lists
  • Exceptions
    • This page describes pattern forms that can be used for many different species of lists, including strings. Forms that are specific to strings are described in reading from strings.

    The following are only available if you import package pattern/listpat.ast.

    Pattern forms that extract from boxes

    Extract
    Pattern match 
      Extract[p1,p2,...,pn] from b.
     requires that b is a box that contains a list. It matches each pattern p1,...,pn in turn against the content of box b and removes each from the front of the list before doing the next match in the sequence. When the match is finished, box b contains what is left over after the last match.

    Suppose that box b holds a list L, and suppose that p1 matches list x1, p2 matches x2, etc. Then after the match, box b holds the remainder r, where L = x1 ++ x2 ++ ... ++ xn ++ r. For example, if x and y each have species Natural and stdin! contains "29+347 abc" then 

      Extract[$(?x), "+", $(?y)] from stdin!.
    binds 
      x = 29
  y = 347
  stdin! := " abc"
    This example has used a constant pattern ("+") and the $ form described for strings.

    extract
    Pattern match 
      Match extract[p1,p2,...,pn] = b.
     is equivalent to 
      Extract[p1,p2,...,pn] from b.
     Use this form in situations where you need to put a pattern (for example, in a choose-matching expression) but want to do an extract.

    Pattern forms for use with Extract

    The following are some general pattern forms for use with extract. These can be used with any type of list that supports equality tests. Other forms that can only be used with lists of characters are described in pattern matching in strings.

    constants
    If c is a list, then Extract [c] from b. removes list c from the front of the content of box b. If c is not a prefix of @b, then this extraction fails with exception testX.

    Example: If b contains "brown fox" then 

      Extract ["brown"] from b.
    leaves b holding " fox".

    lists
    A pattern can be a list. For example, to extract a single member m, use 
      Extract [[?m]] from b.

    theN
    : Natural -> [`a] -> [`a]

    Statement 

      Extract[theN n p] from b.
    extracts a prefix of n members from the content of b, and matches pattern p against them. If there are fewer than n characters, the match fails with exception testX.

    theUpToN
    : Natural -> [`a] -> [`a]

    Statement 

      Extract[theUpToN n p] from b.
    extracts a prefix of n members from content of b, and matches pattern p against them. If there are fewer than n characters, all of them are extracted.

    span
    : COLLECTION`c(EQ`a) -> [EQ`a] -> [EQ`a]

    Pattern match 

      Extract[span c p] from b.
    matches a maximal prefix of the content of box b that contains only members of collection c.

    Example: if box stdin! holds string "aabdcar" before the following example match is performed, then x will be bound to string "aab" and stdin! will be set to hold "dcar".

    Extract[span "abc" ?x] from stdin!.

    spanOthers
    : COLLECTION`c(EQ`a) -> [EQ`a] -> [EQ`a]

    Pattern match 

      Extract[spanOthers c p] from b.
    matches a maximal prefix of the content of box b that contains only nonmembers of collection c.

    Example: If box stdin! holds "abaacdacn", then 

      Extract[spanOthers "dn" ?x] = stdin!.

    binds x to "abaac", and puts "dacn" in stdin!.

    lookingAt
    : EQ`a -> [EQ`a]

    Pattern match 

      Extract[lookingAt c] from b.
    succeeds if list (@b) is a nonempty list and the first member of list (@b) is c. It does not alter what is in box b. This is used to confirm that a particular member is present.

    lookingAtSatisfying
    : (`a -> Boolean) -> [`a]

    Pattern match 

      Extract[lookingAtSatisfying f] from b.
    succeeds if list (@b) is a nonempty list, and f(head(@b)) is true. It does not alter what is in box b.

    lookingAtNilOrSatisfying
    : (`a -> Boolean) -> [`a]

    Pattern match 

      Extract[lookingNilOrAtSatisfying f] from b.
    succeeds if either (@b) is an empty list or (@b) is a nonempty list and f(head(@b)) is true. It does not alter what is in box b.

    lookingAtMember
    : COLLECTION`c(EQ`a) -> [EQ`a]

    lookingAtMember is used to check that an item is a member of a given collection in a pattern match. 

      Extract[lookingAtMember c] from b.
    succeeds if the contents of box b (@b) is a nonempty list, and the head of (@b) is a member of collection c. This match does not alter what is in box b.

    lookingAtNonmember
    : COLLECTION`c(EQ`a) -> [EQ`a]

    lookingAtNonmember is used to check that an item is not a member of a given collection. 

      Extract[lookingAtNonmember c] from b.
    succeeds if (@b) is a nonempty list and head(@b) is not a member of collection c. It does not alter what is in box b.

    lookingAtNilOrMember
    : COLLECTION`c(EQ`a) -> [EQ`a]

    Pattern match 

      Extract[lookingAtNilOrMember c] from b.
    succeeds if either @b is an empty list or @b is a nonempty list and head(@b) is a member of collection c. It does not alter what is in box b.

    lookingAtNilOrNonmember
    : COLLECTION`c(EQ`a) -> [EQ`a]

    Pattern match 

      Extract(lookingAtNilOrNonmember c) from b.
    succeeds if either @b is an empty list or @b is a nonempty list and head(@b) is not a member of collection c. It does not alter what is in box b.

    lookingAtSublist
    : [EQ`a] -> [EQ`a]

    Pattern match 

      Extract[lookingAtSublist p] from b.
    succeeds if list (@b) has p as a prefix. It does not alter what is in box b. In general, pattern lookingAtSublist p matches list p, without modifying the box.

    Forms that do not use boxes

    Most forms for pattern matching in lists use ++ to allow one pattern to follow another. The forms that are used with extract (above) can also be used with ++. For example, 
      Match span "abc" ?x ++ ?r = "cabacrgca".
    binds x to "cabac" and r to "rgca". Match 
      Match ?r ++ span "abc" ?x = "cabacrgca".
    binds x to "ca" and binds r to "cabacrg". You can also use these forms without ++. Pattern match 
      Match span "abc" ?x = t.
    succeeds only if t contains only characters a, b and c, and it binds x to t when it succeeds.

    Operator ++ can also be used by itself in patterns. Here are descriptions of ++ and related operators.

    ++
    : ([`a], [`a]) -> [`a]

    Pattern match 

      Match ?x ++ ?y = t.
    binds x and y in every possibly way (in separate threads) such that x ++ y = t. It uses a stream, so that the pairs (x,y) are tried in increasing order by the length of x. That is, it backtracks over all solutions.

    Note. Handling of pattern p ++ q is overridden for some kinds of patterns so that it does not use backtracking. Instead, it uses special forms. For example, 

      Match "abc" ++ ?x = t.
    checks to see whether "abc" is a prefix of t, and if so removes it, and binds x to what is left. If n has species Natural, then 
      Match $(?n) ++ ?x = "345 444".
    binds n to 345 and x to " 444". It does not try all possible prefixes to see if one is a number. If you want to get backtracking regardless of the pattern form, use ++/ instead.

    Note. Pattern ?x ++ ?y assumes that x and y are examined independently. If a pair (x,y) is returned and that pair is rejected without examining all of x or any of y, then no further pairs are returned. The idea is that, if you have rejected pair (x,y) without examining all of x or any of y, then you will surely also reject all pairs (x',y') where x' is longer than x and y' is shorter than y. If this is not the case, use function ++! instead of ++ in the pattern.

    ++/
    : ([`a], [`a]) -> [`a]

    Pattern match 

      Match ?x ++/ ?y = t.
    is the same as Match ?x ++ ?y = t %Match, but has no special forms. It always uses a stream to get all pairs (x,y) such that x ++ y = t.

    ++//
    : ([`a], [`a]) -> [`a]

    Pattern match 

      Match ?x ++// ?y = t.
    is the similar to Match ?x ++/ ?y = t %Match, but it uses Mix rather than Stream, so the different options are explored in parallel.

    ++!
    : ([`a], [`a]) -> [`a]

    Pattern match 

      Match ?x ++! ?y = t.
    is the same as Match ?x ++/ ?y = t %Match, but it returns all pairs, regardless of how the results are examined. See the note above about assumptions on how results are examined.

    firstOccurrence
    : [EQ`a] -> [EQ`a]

    Pattern match 

      Match ?x ++ firstOccurrence(a) ++ ?y = t.
    finds the first occurrence of list a in list t, binds x to the part of t that occurs before that occurrence and binds y to the part that follows that occurrence. If a does not occur in t, then the match fails with exception testX. For example, 
      Match ?x ++ firstOccurrence("nip") ++ ?y = "manipulate".
    binds 
      x = "ma"
  y = "ulate"
    To use firstOccurrence, you must import collect/findlist.ast.

    Functions for removing prefixes and suffixes of lists

    pullN
    : Natural -> [`a] -> ([`a], [`a])

    (pullN n s) is a pair (w,r) where w is the first n members of s and r is the rest of s. So s = w ++ r. If there are fewer than n members in s, then pullN n s fails.

    Example: pullN 4 [6,2,8,9,2,4,6] = ([6,2,8,9],[2,4,6]).

    pullUpToN
    : Natural -> [`a] -> ([`a], [`a])

    (pullUpToN n s) is a pair (w,r) where w is the first n members of s and r is the rest of s. So s = w ++ r. If there are fewer than n things in the list x, then pullUpToN n x = (x,[]).

    Example:
    pullUpToN 4 [6,2,8,9,2,4,6] = ([6,2,8,9],[2,4,6])
    pullUpToN 4 "ab" = ("ab", "").

    pulls
    : ([`a] -> (`b, [`a])) -> [`a] -> [`b]

    (pulls puller s) repeatedly pulls things from list s using puller to do the pull. puller should be a function that acts like one of the pull functions described here or in string patterns.

    Example: Using pull:String -> (Natural, String), natural numbers are successively removed from the string.
    pulls (pull:String -> (Natural, String)) "12 234 99 " = [12, 234, 99].

    Note: If the puller fails with exception listExhaustedX, then pulls assumes the puller is saying there is no useful information left, and pulls returns an empty list.

    Example: pulls (pullN 3) "abcde" = ["abc"]. (The remaining two characters are not enough for the required 3; pullN 3 "de" fails with exception listExhaustedX.)

    Exceptions

    listExhaustedX
    : ExceptionSpecies

    List patterns fail with exception listExhaustedX when they attempt to get a value from an empty list.
