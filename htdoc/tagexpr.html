E20. Tags and Assumptions

  • Creating tags and assumptions
  • Examples

    • Creating tags and assumptions

    Although it is usually not necessary, you can indicate the (polymorphic) species of expressions explicitly if desired. This can be helpful in controlling how type inference is done, and in understanding type errors.

    e:T
    Writing :T after an expression tells the compiler that the expression has (polymorphic) type T, or some species that is included in T. The tag operator (:) has low precedence, so a tag applies to a large expression. For example, writing f x : T indicates that expression f x has type T.

    Assume x,y,z:T %Assume
    You can include an assumption such as this as a declaration or in any scope. It causes each subsequent occurrence of the listed identifiers to be tagged as indicated, and has the same scoping rules as a let-expression. See type inference for more.

    Examples

    The following examples use a global assumption and a local assumption. They both implement the assoc function, but, through tags, have had their types limited to Natural -> [(Natural,String)] -> String. 
    Assume x:Natural.

Let assoc1 ?x ?l =
  Assume r,l: [(Natural, String)].
  Loop ?r = l first matching r
    exitcase []             => fail testX
    exitcase [(x,?y) & ?]   => y
    loopcase else           => continue tail r
  %Loop
%Let

Define assoc2 by first
  do Assume x:Natural; y:String.
  case  assoc2 ?  []              = fail testX
  case  assoc2 ?x ((?x,?y) :: ?)  = y
  case  assoc2 ?x (? :: ?t)       = assoc2 x t
%Define
