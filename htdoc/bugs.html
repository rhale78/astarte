List of Known Bugs

  • Semantic bugs: compiler
  • Cosmetic bugs: compiler
  • Semantic bugs: interpreter
  • Cosmetic bugs: interpreter
    • These are most of the known bugs. This does not include things that have not been completed and are not documented, or things that appear to work according to the documentation, but need to be improved, except for a few cases.

    Semantic bugs: compiler

    These are bugs in the compiler that have an effect on the semantics of the programming language.

    1. Advise declarations are imported sideways. That is, if package A imports package B and then imports package C, any advise declarations made in B are visible in C.

    2. There is a mistake in the standard definition of downcast that shows up when compiling collect/linset.ast.

    3. The standard definition of equality for new families contains a mistake. The definition works, but its polymorphic type is not quite right.

    4. In some contexts, the scope of local assumptions does not agree with the scope of identifiers.

    5. A nonrestrictable variable is currently allowed to accept a new constraint.

    6. Asthelp shows some types as too general. (This is probably a compiler bug since the compiler writes the information that asthelp uses.)

    Cosmetic bugs: compiler

    These bugs result in unpleasant and possibly incorrect messages, but do not affect the language implementation.

    1. The exhaustiveness checker for pattern matching is not implemented correctly. It does not catch missing cases that it should catch.

    2. Circular light constraints `a >>= `a are shown when they should not be.

    3. There is a mistake in the type-constraint checker regarding handling of fictitious species. It does not affect most programs.

    4. A nonrestrictable variable is currently allowed to get a new constraint on it. That should not be allowed.

    5. A pattern match rule that uses an irregular function in a bad context is not reported until the rule is used.

    6. When the results of type inference are shown, an expectation that is generated as a result of processing the current definition is shown as a prior expectation.

    7. Overloading a function name where one definition is regular and the other is irregular gives a message that there is a bad overlap in types, but does not explain the problem.

    Semantic bugs: interpreter

    1. Function outerBinding currently allows access to things that should be inaccessible because they are private or protected. It also disallows access to some things to which it should allow access.

    2. The implementation of threads is not finished. Functions for detaching and managing threads have not been implemented.

    3. RawShow only prints on external files.

    Cosmetic bugs: interpreter

    These are unpleasant bugs, but they do not affect the correctness of the language implementation.

    1. The debugger is incomplete and undocumented.

    2. If a trap occurs inside a lazy primitive, you can't see what the lazy primitive was doing by inspecting ast.rts.
