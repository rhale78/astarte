L9. Input and Output

  • Input
  • Output
  • File modes
  • Internal files

    • Input

    Input is done by getting the contents of a file as a string. The operation that does this is lazy, so the file is not actually read until it is needed. You can use the usual list and string operations to get things from this string. You can, for example, put the string into a box, and use the string-box read operations (reading) to read prefixes.

    infile
    : [FileMode] -> String -> String
    : [FileMode] -> File -> String
    : String -> String
    : File -> String

    infile mode name returns the contents of file name. Parameter mode tells how the file is opened, and must be a list of FileMode values. See file modes, below.

    Expression infile(name) is equivalent to infile [] name.

    An alternative form is (infile mode f) or (infile f), where f is an object of species File rather than a string. See File.

    Examples:
    Let s = infile [volatileMode] "data.txt".
    Let r = infile "data.txt".

    stdin!
    : <:String:>

    stdin! is the standard input. It is a box whose contents is a string, which is the remaining (unconsumed) input. To redirect standard input, put another string in this box.

    trueStdin!
    : <:String:>

    It is possible to redirect standard input on the command line. If the standard input is redirected, then box trueStdin! is created, and is set to hold the unredirected standard input. If input is not redirected, then stdin! and trueStdin! are the same box.

    Output

    To do output, either write to one of the standard outfiles, or tie an outfile to a system file and write to that. Each time you write, what you write is appended to what was written by the previous write to the same outfile.

    Outfile(`a)
    An object of species Outfile(`a) is a file that has been opened for output, and whose members have species `a. Typically, you use Outfile(Char).

    AppendToFile
    : String -> String -> ()
    : File -> String -> ()

    AppendToFile f, s %AppendToFile is the same as WriteFile [appendMode], f, s %WriteFile. It appends string s to the end of file f. Here, f can be either the name of the file or an object of type File.

    Do not overuse this. If you have a lot of data to write to a file, open it using outfile, write to it using FWrite and FWriteln, and then close the file.

    Example: AppendToFile "out.txt", "Add this to the end of the file".

    CloseFile
    : Outfile(`a) -> ()

    CloseFile f %CloseFile closes an Outfile f. After closing, the outfile cannot be written to any more. Any attempt to write to it will cause a failure with exception closedFileX (for a system file) or exception closedPipeX (for a file created by function pipe).

    Example: CloseFile f.

    FlushFile
    : Outfile(`a) -> ()

    Output to files is accumulated in buffers until enough is available to send to the file at once. FlushFile f %FlushFile forces the buffer for outfile f to be sent to the file, without waiting for more data.

    Example: FlushFile @stdout!.

    FPut
    : Outfile(`a) -> `a -> ()

    FPut f, c %FPut writes character c to outfile f. It is possible for f to be of species Outfile(T), where T is not Char. Then FPut f, c %FPut writes value c (of species T) to file f.

    Example: Fput f, 'a'.

    FWrite
    : Outfile(`a) -> [[`a]] -> ()
    : [Outfile(`a)] -> [[`a]] -> ()

    FWrite f, x %FWrite writes each string in list x to outfile f. FWrite [f,g,h], x %FWrite writes each string in list x to each of the outfiles f, g and h. In the case where f has species Outfile(T), x must be a list of lists of T's. Each member of each list is written to file f.

    Example: FWrite f, ["I have ", $n, " cats"].

    FWriteln
    : Outfile(Char) -> [String] -> ()
    : [Outfile(Char)] -> [String] -> ()

    FWriteln f, x %FWriteln writes each string in list x to outfile f, and then writes a newline character to file f. FWriteln [f,g,h], x %FWriteln writes each string in list x to each of the outfiles f, g and h, followed by a newline.

    Example: FWriteln f, ["I have ", $n, " cats"].

    outfile
    : [FileMode] -> String -> Outfile(Char)
    : [FileMode] -> File -> Outfile(Char)
    : String -> Outfile(Char)
    : File -> Outfile(Char)

    outfile mode name returns an outfile (of species Outfile(Char)) that writes to file name. Parameter mode is a list of modes. See file modes, below, for the modes.

    outfile(name) is equivalent to (outfile [] name). Parameter name can be a string or a File item.

    An alternative form is (outfile mode f) or (outfile f), where f has species File. See File.

    Example: Let f = outfile [appendMode] "out.txt".

    Show
    : Outfile(Char) -> `a -> ()

    Show f, x %Show prints view(x) on file f. view(x) is a string that describes x, but that replaces lazy parts by string such as "(lazy)". See the description of view.

    stderr!
    : <:Outfile(Char):>

    Box stderr! holds the standard error output file. You can put another outfile in this box to redirect standard error. Note that stderr! is not an outfile -- it is a box whose contents is an outfile.

    stdout!
    : <:Outfile(Char):>

    Box stdout! holds the standard output file. You can put another outfile in this box to redirect standard output. Note that stdout! is not an outfile -- it is a box whose contents is an outfile.

    Write
    : [String] -> ()

    Write x %Write writes each string in list x to the standard output.

    Example: Write["I have ", $n, " cats"].

    WriteFile
    : [FileMode] -> String -> String -> ()
    : [FileMode] -> File -> String -> ()
    : String -> String -> ()
    : File -> String -> ()

    WriteFile mode, filename, s %WriteFile opens file filename with the given mode, writes string s into that file, and then closes the file. Parameter filename can be a string, but it can also be an item of species File.

    WriteFile f, s %WriteFile is the same as WriteFile [], f, s %WriteFile. That is, it uses the default mode, which is to empty the file before writing.

    Example: WriteFile "out.txt", "This is my output". causes file out.txt to hold exactly This is my output.

    Writeln
    : [String] -> ()

    Writeln x %Writeln writes each string in list x to the standard output, and then writes a newline character to the standard output.
    Example: Writeln["I have ", $n, " cats"].

    WriteTo
    : Outfile(Char) -> ()

    WriteTo(f). redirects the standard output to file f. Here, f must have species Outfile(Char).

    File modes

    FileMode
    FileMode is a species that contains appendMode, binaryMode and volatileMode. A file mode is used to control how a file is opened.

    appendMode
    : FileMode

    Mode appendMode is used only for outfiles. When a file is opened with this mode, new characters are added to the end of the file. Without this mode, the file is emptied when it is opened for output.

    binaryMode
    : FileMode

    Mode binaryMode indicates that the file is binary. When this mode is absent, the file contains text.

    volatileMode
    : FileMode

    Mode volatileMode is used only for infiles. It indicates that the file's contents might change during the course of the program's execution. When this mode is not used, infiles are presumed to remain constant. This allows the run-time system to reuse memory that is occupied by the file's contents, and to reread the file if necessary to get that information back.

    Internal files

    An internal file allows one thread to communicate with another (or with itself if you are careful) by writing into one end of a pipe, and reading from the other. A pipe can have species Outfile(T) for any species T. Use FWrite and FPut to write into it, and use CloseFile to close it. Be sure to close it, since otherwise the receiving end will never see an end-of-list marker.

    pipe
    : () -> ([`a],Outfile(`a))

    pipe() = (x,f) where f is an outfile and x is a list. Anything that is written to f shows up in list x. Any thread that tries to look at more of x than has been written waits until something is written (by another thread).

    Example: 

      Match (?pipeout, ?pipein) = pipe().
  FWrite pipein, ["test"].
  CloseFile pipein.
    causes list pipeout to hold "test".
