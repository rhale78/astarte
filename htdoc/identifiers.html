E8. Identifiers

  • Normal identifiers
  • Symbolic identifiers
  • Reserved words and symbols
  • Suppressing special handling
    • An identifier is a name for something. Identifiers have the following forms.

    Normal identifiers

    A normal identifier begins with a letter or the special symbol _ (underscore), and can contain letters, digits and the special symbols
       _ ! ? '
    For example, each of the following is a normal identifier. 
        NormalIdentifier
    normalIdentifier_with_underscores
    normal'
    normal?
    normal!
    Normal identifiers are used to name values, species, etc.

    Symbolic identifiers

    A symbolic identifier is any sequence of the special symbols 
      #  +  -  *  /  ^ 
  !  =  <  >  @  &
  ~  $  _  \  ?  '
    that does not begin with symbol ? or '. Additionally, there is one more symbolic identifier, ::. The following are all symbolic identifiers. 
        +
    ++/
    $_$@'

    Reserved words and symbols

    Some words and symbols are reserved, and cannot be used as identifiers. They are as follows. Each is reserved in lower case form (such as abbrev) and in upper case form (such as Abbrev). Reserved words that start with underscore are only reserved in the exact form shown.

    abbrev abstraction advise all _and_ anticipate
    ask assume atomic await bring _butIfFail_
    by case catchingEachThread catchingTermination choose class
    const constraint constructor context continue cutHere
    default define description directory do else
    exception execute exitcase expand expect export
    extension extract first for from holding
    if import implementation implements _implies_ interface
    isKindOf let loop loopcase match matching
    missing mix mixed my one open
    operator _or_ package pattern perhaps relate
    relet species speciesAsValue stream subcases target
    team then trap try unique untrap
    var when with _where_ while

    Any sequence of four or more consecutive = signs or < signs is reserved (for comments). The following symbolic identifiers are reserved. 

      =     <<     ->      <--?
  ?     >>     =>      <<<<....
  |     ?<<    =>>     ====....
  &     >>?    <--

    Suppressing special handling

    Some identifiers are handled in special ways in the sytax. For example, + is a binary operator. If you want to suppress this special handling, enclose the identifier in parentheses. For example, (+) is the addition function, and is not treated as a binary operator. See special handling for details on this.
